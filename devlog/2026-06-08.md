# 开发日志 — 2026-06-08

## 功能：连杀系统 — 5秒窗口连续击杀触发飘字提示 + 金币1.2x加成

### 需求

1. 玩家连续击杀敌人（间隔不超过5秒）时触发连杀计数
2. 从2连杀开始显示 "N连杀!" 飘字提示，不限次数
3. 连杀状态下击杀金币掉落 1.2x 加成

### 影响范围

- 击杀流程（`HealthSystem` → `onEnemyKilled` 回调）新增连杀逻辑
- 金币发放（`EconomySystem.addGold`）通过倍率控制
- 新增飘字实体类型 `ComboFloatingText`，独立于伤害飘字系统

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/core/components.ts` | 新增 `ComboFloatingText` 组件（comboCount, lifetime, maxLifetime, velocityY, scale, alpha）+ `comboTextQuery` 查询 |
| `src/systems/ComboKillSystem.ts` | **新建** — 连杀系统核心：comboCount 追踪、5秒窗口超时重置、`notifyEnemyKilled()` 返回金币倍率、`spawnComboText()` 生成飘字实体、`renderAll()` Canvas 2D 渲染 "N连杀!" 文字 |
| `src/systems/ComboKillSystem.test.ts` | **新建** — 11个测试用例覆盖：单杀/连杀计数、金币倍率、5秒窗口超时、飘字实体生命周期、reset |
| `src/main.ts` | ① 导入 `ComboKillSystem`；② `onEnemyKilled` 回调改用 `comboKillSystem.notifyEnemyKilled()` 获取倍率；③ `economy.addGold(Math.floor(baseGold * comboMultiplier))` 替代 `economy.rewardForEnemy()`；④ `onPostRender` 中新增 `comboKillSystem.renderAll()` 调用；⑤ 系统注册顺序：healthSystem → comboKillSystem → damageNumberSystem |

### 关键技术决策

- **飘字独立组件**：`ComboFloatingText` 与 `DamageNumber` 分离，避免在 bitecs typed arrays 中混入字符串。飘字文本由渲染时拼接 `${count}连杀!`。
- **Canvas 2D 渲染**：沿袭 `DamageNumberSystem` 的 `renderAll()` 模式，通过 `onPostRender` 在 UI 层之上绘制。
- **连杀视觉分级**：普通连杀（2-4）使用金色，高连杀（≥5）使用炽热橙红色 + 外圈光晕。
- **金币倍率**：`notifyEnemyKilled()` 返回 `1.2`（连杀≥2）或 `1.0`，在 `main.ts` 回调中直接乘算后入账。

## 重构：移除全屏倒计时遮罩，改为顶部HUD常驻显示

### 需求

1. 去掉波次刷新前"敌军即将来袭"全屏倒计时遮罩面板
2. 顶部HUD常驻显示下一波敌人出现倒计时（⏱Xs）
3. 最后10秒可点击"立即开始"按钮跳过倒计时

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/systems/UISystem.ts` | ① 删除 `buildCountdownOverlay()` 方法及调用（~107行）；② 增强 `buildTopHUD()` 倒计时区域：倒计时≤10s时按钮显示"立即开始"（90px宽、绿色加亮边框），>10s时显示"▶"（50px宽、暗色）；③ 清理不再使用的 `ENEMY_DISPLAY_NAMES` 常量、`getWavePreview` 回调和 `setWavePreviewCallback` 方法 |
| `src/main.ts` | 移除 `uiSystem.setWavePreviewCallback()` 接线（波次敌人预览已随遮罩移除） |
| `src/systems/UISystem.test.ts` | 更新测试：按钮label从 `'▶'` 改为 `'立即开始'`（cd=5s ≤ 10s） |

### 关键技术决策

- **按钮动态宽度**：通过 `cd <= 10` 判断，切换 90px（"立即开始"）和 50px（"▶"）两种宽度，右边缘位置固定（与 speedBtn 保持 12px 间距），左边缘随宽度自适应
- **视觉区分**：≤10s 时按钮用绿色 `#2e7d32` + 金色边框 `#ffd54f`，强调可点击；>10s 时用暗色 `#37474f` + 白色边框
- **保留 `modalBackdropAlpha`/`hasFullscreenOverlay`**：这两个字段还被暂停/胜利/失败/抽卡/Buff遮罩使用，不能删除
