| 文件 | 改动 |

| `src/systems/UISystem.ts` | 使用右侧槽位分配方式布局 HUD 按钮；新增卡牌图鉴 HUD 小按钮；移除暂停弹窗内的卡牌图鉴按钮并恢复弹窗高度 |
| `src/systems/UISystem.test.ts` | 增加卡牌/敌人图鉴相邻布局、暂停与倍速左移、暂停弹窗无卡牌图鉴入口的单测 |

## 功能：手牌 hover 预览动画与卡面描述常驻

### 需求变化

鼠标移动到手牌区域的卡牌上时，当前卡牌需要略微放大并向上弹出手牌区；鼠标离开卡牌范围后恢复。取消原有卡牌介绍 tips 面板，卡牌介绍信息直接显示在卡牌 UI 内。

### 改动文件

| 文件 | 改动 |
|------|------|
| `design/02-gameplay.md` | 更新卡牌信息展示与拖卡交互说明，明确取消 tips 面板并加入 hover 预览动画 |
| `src/systems/UISystem.ts` | 增加手牌卡牌 hover 插值状态；手牌卡牌绘制支持放大、上移和更高层级；卡牌描述直接绘制在卡面；停止调用旧 tooltip 渲染 |
| `src/systems/UISystem.test.ts` | 增加 hover 后卡牌上移放大、卡面描述常驻、无 tooltip 面板的单测 |

### 验证结果

- `npm run typecheck` 通过。
- `npx vitest run src/systems/UISystem.test.ts` 通过，11 个测试。
- `npm test` 通过，76 个测试文件，1314 个测试。
- `npm run build` 通过；保留 Vite 既有大 chunk 体积警告。
- Playwright 打开 `http://127.0.0.1:3000/ai-tower-defender/`，进入第一关后截图确认手牌 hover 卡牌上弹放大、描述显示在卡面且没有额外 tooltip。
