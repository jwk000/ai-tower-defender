## 功能：重新生成单位四态 2 帧动作素材

### 需求变化

上一版攻击/死亡动作主要由基准图缩放、拉伸和局部变形派生，不满足动作帧质量要求。本次要求以当前单位图片为参考，重新生成所有单位动作素材；每个单位至少包含待机、移动、攻击、死亡 4 个状态，每个状态 2 帧。`idle` 可以轻量呼吸，`move`、`attack`、`death` 必须是新姿势图片，不能只靠缩放拉伸。

### 改动文件

| 文件 | 改动 |
|------|------|
| `design/09-ai-art-redesign.md` | 将单位图片接入基线更新为 `idle/move/attack/death` 四态 8 帧，并明确非 idle 状态不得用缩放拉伸替代 |
| `scripts/generate_unit_action_frames.py` | 改为通过 `animal-mediakit` 参考图生成 4×2 动作表，再本地去纯绿背景并切片为 256×256 RGBA 运行时资源 |
| `public/art/units/` | 47 个单位的 `idle/attack/death` 帧重新生成；新增所有单位 `move_0/move_1` 帧，共满足 376 张必需动作图 |
| `src/systems/RenderSystem.ts` | 场景单位新增 `move` 状态选择；移动中使用 `move` 帧，不再用移动呼吸缩放替代动作帧 |

### 验证结果

- `animal-mediakit` 初始化通过，认证有效，animal-gateway 连通正常。
- 批量生成通过：47 个单位 × 4 状态 × 2 帧。
- 独立资源检查通过：376 张必需 PNG 全部存在，均为 `256x256 RGBA`，无空 alpha。
- 抽样缩略图检查通过：移动、攻击、死亡均为新姿势，不是简单缩放拉伸。
- `npm run typecheck` 通过。
- `npm test` 通过，76 个测试文件、1314 个用例。
- `npm run build` 通过；仅保留 Vite chunk size 常规警告。

## 修复：敌人精灵朝向与移动方向相反

### 问题

哥布林、野猪、大象、沙漠黑虫、钻地甲虫等敌人资源图默认面朝左，但运行时 `Visual.facing=1` 语义是面朝右。此前位图精灵在 `facing=1` 时不翻转，导致大多数敌人图片看起来背向移动方向。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/systems/RenderSystem.ts` | 增加单位精灵 `scaleX` 计算 helper；敌人位图资源按“默认面朝左”处理，向右移动时水平翻转，向左移动时保持原图 |
| `src/systems/__tests__/MovementSystem.pathEquivalence.test.ts` | 增加资源朝向与运行时 `facing` 映射测试，避免后续再次写反 |

### 验证结果

- `npm test -- src/systems/__tests__/MovementSystem.pathEquivalence.test.ts` 通过，9 个测试。
- `npm run typecheck` 通过。
