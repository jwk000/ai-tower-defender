# 开发日志 — 2026-06-07

## 功能：炮塔视觉改造 — 黑色圆形炮弹 + 可旋转炮管 + 抛物线轨迹

### 需求

1. 所有炮塔的子弹改为黑色圆形炮弹
2. 给炮塔增加炮管，炮管可以平滑旋转追踪目标
3. 攻击时炮弹从炮管尖端发射出来
4. 炮弹具有抛物线下落轨迹（非直线飞行）

### 影响范围

所有发射可见弹道的塔类型：Arrow (0), Cannon (1), Ice (2), Fire (7), Poison (8), Ballista (9)。Missile (6) 已有抛物线，也增加了炮管支持。Laser (4) / Lightning (3) / Bat (5) 不受影响。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/core/components.ts` | 新增 `Barrel` 组件（angle, targetAngle, length, width） |
| `src/systems/UnitFactory.ts` | `createTower()` 创建塔时添加 `Barrel` 组件 |
| `src/systems/AttackSystem.ts` | ① PROJ_VISUAL 中 0/1/2/7/8/9 改为黑色圆形 ShapeVal.Circle；② update() 中增加炮管平滑旋转逻辑（8 rad/s）；③ spawnProjectile() 从炮管尖端计算发射起点 + 锁目标位置计算抛物线参数；④ spawnMissileProjectile() 同步从炮管尖发射 |
| `src/systems/ProjectileSystem.ts` | update() 中对 `totalTime > 0` 的非导弹投射物使用抛物线轨迹（复用 `computeMissileParabola` 物理，重力 400px/s²） |
| `src/systems/RenderSystem.ts` | ① 渲染塔的炮管（深灰矩形 + 旋转）；② 修改 pushCmd：不再强制所有投射物为 arrow 形状，根据 Visual.shape 渲染（circle → 圆形） |

### 关键技术决策

- **抛物线统一物理**：非导弹塔的投射物复用导弹塔的 `computeMissileParabola` 函数，重力常数 400px/s² 保持不变
- **目标位置锁定**：投射物 spawn 时锁定目标的当前坐标作为落点，避免飞行期追移动目标的复杂行为
- **炮管平滑旋转**：炮管旋转速度 8 rad/s，角度差归一化到 [-π, π] 取最短路径

## 功能：关卡内敌人图鉴

### 需求

1. 关卡内增加敌人图鉴功能，图鉴按钮在 HUD 最右侧（原暂停按钮位置）
2. 暂停按钮向左移动腾出空间
3. 点击弹出类似卡牌图鉴的卡牌列表，显示当前关卡内的所有敌方单位信息
4. 卡牌仅用于展示，不进入手牌
5. 每种敌方单位展示独立图标、名字、描述等信息
6. 普通、精英、Boss 用卡牌颜色区分（银灰/金色/红色）

### 影响范围

- 新建 `src/ui/EnemyCodexUI.ts`：Canvas 2D 渲染的敌人图鉴弹窗
- 修改 `src/systems/UISystem.ts`：HUD 按钮布局、回调参数
- 修改 `src/main.ts`：EnemyCodexUI 实例化、数据注入、输入事件绑定

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/ui/EnemyCodexUI.ts` | **新建**。Canvas 2D 图鉴弹窗，1400×850 面板，7 列卡片网格，支持滚轮和拖拽滚动。卡片展示矢量图标、名称、类型标签、属性摘要、描述。三级颜色区分（normal=#90a4ae, elite=#ffd700, boss=#ef5350） |
| `src/systems/UISystem.ts` | 新增 `onOpenEnemyCodex` 字段和 `setEnemyCodexCallback()` setter；`buildTopHUD()` 中添加 📖 图鉴按钮（最右侧 `rightControlsEdgeD - 29`），暂停按钮左移到 `rightControlsEdgeD - 70`，速度按钮左移到 `rightControlsEdgeD - 112`，跳过按钮左移到 `rightControlsEdgeD - 174` |
| `src/main.ts` | 导入 `EnemyCodexUI` + `EnemyCodexEntry` + `ENEMY_CONFIGS`；构造函数中创建 enemyCodex 实例绑定 pause 回调；关卡选择和战斗界面的 postRender / pointerDown / pointerMove / pointerUp 中添加敌图鉴优先处理；`initBattle()` 中调用 `setupEnemyCodex(config)` 从波次配置收集唯一敌人类型生成卡片数据，分类规则：isBoss→boss，bossWave 中→elite，其余→normal |

### 关键技术决策

- **分类逻辑**：Boss 直接由 `EnemyConfig.isBoss` 判断；精英由所在波次的 `isBossWave` 标记推断（boss 波次中的敌人通常都是较强单位）
- **数据来源**：敌人信息从 `ENEMY_CONFIGS`（硬编码配置）读取，关卡敌人列表从 `LevelConfig.waves[].enemies[].enemyType` 去重收集
- **图标设计**：使用盾形 + 三角角饰 + 双眼的通用矢量图标，每个敌人的 `enemy.color` 作为图标主色，保持视觉上的个体辨识度
