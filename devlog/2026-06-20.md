# 2026-06-20 开发日志

## Boss 战补充金币波次

- 需求：Boss 出现后定时补充新的敌人波次，补充波次以让玩家获得金币为目的，避免场上只剩一个 Boss。
- 设计：更新关卡设计原则，明确 Boss 存活期间由 `WaveSystem` 定时补充非 Boss 敌人；默认复用该 Boss 波内已有的非 Boss 敌人，也支持 `bossReinforcements.interval / maxAliveNonBoss / groups` 精细调参。
- 修复：`WaveSystem` 增加 Boss 补怪计时器和场上非 Boss 存活数上限；补怪只在 Boss 波、首轮出怪结束、Boss 仍存活时触发，Boss 死亡后停止补怪并保持最后一波胜利结算可达。
- 配置：补齐 `WaveConfig`、编辑器 `levelModel`、`modelToLevelConfig` 的 `bossReinforcements` 类型、解析、序列化和运行时桥接。
- 回归：新增测试覆盖显式补怪、默认复用 Boss 波非 Boss 敌人、Boss 死亡后不再补怪并进入胜利；`npm run typecheck`、定向 vitest、`npm run build` 通过。
