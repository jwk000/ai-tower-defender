- 配置：补齐 `WaveConfig`、编辑器 `levelModel`、`modelToLevelConfig` 的 `bossReinforcements` 类型、解析、序列化和运行时桥接。
- 回归：新增测试覆盖显式补怪、默认复用 Boss 波非 Boss 敌人、Boss 死亡后不再补怪并进入胜利；`npm run typecheck`、定向 vitest、`npm run build` 通过。

## 第 5 关深渊领主未出场提前胜利

- 问题：第 5 关最终波配置 `spawnDelay: 30`，波次系统固定间隔到点后只看场上是否有活敌人；最终波等待出怪期间场上为空时会直接进入胜利，导致深渊领主尚未生成。
- 修复：`WaveSystem` 最终波结算必须同时满足本波出怪队列已完成、出怪数量达到配置总数、场上无存活敌人。
- 回归：新增“最后一波还在等待出怪时空场不能胜利”测试，覆盖深渊领主等待生成场景；`npm test -- WaveSystem.spawn.test.ts`、`npm run typecheck` 通过。

## 放置失败不再消耗手牌

- 问题：手牌拖拽放置塔或士兵时，偶发出现单位没有创建成功，但金币和卡牌已经被消耗。
- 设计：补充卡牌消耗验收规则，明确只有实体或技能效果真实创建成功后才扣金币、移除手牌；非法落点、工厂创建失败、金币不足、手牌槽位变化都视为出牌失败。
- 修复：手牌拖拽状态记录起拖卡牌 ID；`HandSystem.playCard` 支持 expected card 校验，避免槽位被自动补牌或调试替换后误删其它卡；士兵部署改为实体创建成功后由外层统一扣金币和消费卡牌。
- 回归：新增 expected card mismatch 不消费测试；`npm test -- --run src/systems/__tests__/HandSystem.test.ts src/systems/__tests__/BuildSystem.cost-decoupling.test.ts`、`npm run typecheck` 通过。

## 胜利条件改为击杀最终 Boss

- 需求：关卡胜利条件改为击杀本关 Boss，避免第 5 关深渊领主仍存活时直接胜利。
- 设计：同步玩法文档和 Boss 通用规则，明确胜利由最后一波配置的最终 Boss 真正死亡触发；非 Boss 敌人是否存活不影响胜利。
- 修复：`WaveSystem` 在最终波记录本波配置的 Boss 类型和是否已生成，最终波不再按固定时间或清场判胜，只在最终 Boss 已生成且真正死亡后结算胜利；巨型史莱姆待分裂状态不算真正死亡。
- 回归：补充最终 Boss 存活不胜利、最终 Boss 未生成不胜利、非 Boss 死亡但 Boss 存活不胜利、Boss 死亡且非 Boss 存活仍胜利、巨型史莱姆待分裂不胜利等测试；`npm test -- WaveSystem.spawn.test.ts`、`npm run typecheck` 通过。
