## 修复：精英死亡不触发抽卡

### Bug 分析

精英死亡→3选1抽卡的检测原本在 `WaveSystem.update()` 中通过轮询 `Health.current[eliteEid]` 实现，但存在三个致命缺陷：

1. **系统执行顺序漏检**：`TrapSystem`(930) 在 `WaveSystem`(929) 之后运行，陷阱造成的最后一击被 WaveSystem 错过，下一帧实体已被清理 → 不触发抽卡。

2. **v5.0 波次到期不可达**：30秒波次计时器到期后 `finishWave()` 设 `waveActive=false`，下一帧 `update()` 提前 return，elite death check 完全不可达。

3. **精英 eid 跨波丢失**：存活到下波的旧精英，被 `startWave()` → `resetEliteTracking()` 清掉 eid → 永远不触发。

### 修复方案

将精英死亡检测从 WaveSystem 的位置依赖轮询，改为在 `HealthSystem.onEnemyKilled` 回调中通过 `UnitTag.isElite[eid] === 1` 判断。覆盖所有死因，不受管线顺序影响。同时加入 `!this.cardDraftSystem.isActive()` 守卫防重复触发。

### 改动文件
| 文件 | 改动 |
|------|------|
| `src/systems/WaveSystem.ts` | 删除 `onEliteKilled` 参数和 `update()` 中的 elite 死亡轮询（-13行） |
| `src/main.ts` | HealthSystem 回调中新增 `UnitTag.isElite` 检测 + 抽卡触发；WaveSystem 构造移除第8参数 |
| `src/systems/__tests__/WaveSystem.spawn.test.ts` | 两个旧测试改为验证 Elite 组件属性（适配 API 变更） |
