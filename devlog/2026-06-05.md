## 修复：精英死亡不触发抽卡

### Bug 分析

精英死亡→3选1抽卡的检测原本在 `WaveSystem.update()` 中通过轮询 `Health.current[eliteEid]` 实现，但存在三个致命缺陷：

1. **系统执行顺序漏检** ：`TrapSystem`(930) 在 `WaveSystem`(929) 之后运行，陷阱造成的最后一击被 WaveSystem 错过，下一帧实体已被清理 → 不触发抽卡。

2. **v5.0 波次到期不可达** ：30秒波次计时器到期后 `finishWave()` 设 `waveActive=false`，下一帧 `update()` 提前 return，elite death check 完全不可达。

3. **精英 eid 跨波丢失** ：存活到下波的旧精英，被 `startWave()` → `resetEliteTracking()` 清掉 eid → 永远不触发。

### 修复方案

将精英死亡检测从 WaveSystem 的位置依赖轮询，改为在 `HealthSystem.onEnemyKilled` 回调中通过 `UnitTag.isElite[eid] === 1` 判断。覆盖所有死因，不受管线顺序影响。同时加入 `!this.cardDraftSystem.isActive()` 守卫防重复触发。

### 改动文件
| 文件 | 改动 |
|------|------|
| `src/systems/WaveSystem.ts` | 删除 `onEliteKilled` 参数和 `update()` 中的 elite 死亡轮询（-13行） |
| `src/main.ts` | HealthSystem 回调中新增 `UnitTag.isElite` 检测 + 抽卡触发；WaveSystem 构造移除第8参数 |
| `src/systems/__tests__/WaveSystem.spawn.test.ts` | 两个旧测试改为验证 Elite 组件属性（适配 API 变更） |

### Bug 修复：图鉴界面空白

- **症状** ：点击图鉴按钮后界面全黑（只有遮罩，无面板内容）
- **根因** ：`ctx.roundRect()` 在 `onPostRender` 阶段的 Canvas 2D 上下文中可能不被支持或行为异常；emoji 字符（🗼⚔⛓✦◇）在 Canvas 上渲染不可靠

  2. Emoji 符号替换为 ASCII 字母（`T/S/X/*/A`），确保跨平台渲染一致

  4. 渲染前后 `ctx.save()/restore()` 隔离上下文状态

### Bug 修复 #2：主菜单图鉴仍空白 — 渲染时序错误

- **根因** ：`Game.loop()` 中 `paused=true` 时 `onUpdate` 被跳过。图鉴在主菜单的渲染写在 `onUpdate` 里，图鉴打开时 game.paused=true → 渲染被跳过 → 空白。
- **游戏循环实际顺序** ：`beginFrame() → input.flush() → [!paused ? onUpdate : world.update(0)] → endFrame() → onPostRender()`
- **修复** ：主菜单的图鉴渲染从 `onUpdate` 移至 `onPostRender`（`onPostRender` 始终执行，不受 pause 影响）

## 修复：GitHub Pages 构建失败

### 问题分析

GitHub Actions 在 `npm run build` 的 `vite build` 阶段失败，esbuild 报错：

`Top-level await is not available in the configured target environment`

源头是 `src/main.ts` 入口文件直接使用了顶层 `await import(...)` 和 `await loadAllUnitConfigs()`。`tsconfig.json` 的 TypeScript target 是 `ES2022`，但 Vite 生产构建默认面向更老的浏览器目标（如 `chrome87`、`safari14`、`es2020`），这些目标不支持顶层 await，因此本地与 GitHub Pages 生产构建都会失败。

### 修复方案

将入口初始化逻辑收进 `async function bootstrapGame()`，保留原初始化顺序：

1. 加载 YAML 单位配置
2. 注入敌人/塔/士兵/技能/陷阱配置
3. 创建并启动 `TowerDefenderGame`
4. 开发模式下动态加载关卡编辑器
5. 绑定 resize、window 调试对象

这样生产产物不再包含顶层 await，同时不放宽浏览器兼容目标。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/main.ts` | 将入口顶层 await 启动逻辑封装为 `bootstrapGame()` 并用 `void bootstrapGame()` 启动 |

### 验证结果

- `npm run typecheck` ✅ 通过
- `npm test` ✅ 62 suites / 1234 tests passed
- `npm run build` ✅ 通过，原 esbuild 顶层 await 错误消失
