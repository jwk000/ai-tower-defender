### 改动详情

#### 1. 预览标记半径修复（AttackSystem.ts）
- **问题**：TargetingMark 的 `blastRadius` 硬编码为 `120`，但实际导弹溅射半径从塔配置读取为 `80`（gameData.ts `splashRadius: 80`）
- **修复**：改为从 `Attack.splashRadius[eid]` 读取塔的实际溅射半径，与 `spawnMissileProjectile` 保持一致
- **视觉效果**：红色靶心环半径从 120px 缩小到 80px，与爆炸伤害范围完全一致

#### 2. 圆形弹坑效果（TileDamageSystem + RenderSystem）
- **问题**：旧实现遍历所有格子，在每个被爆炸覆盖的格子内绘制独立裂纹——弹坑与格子对齐，不自然
- **修复**：改为以命中点 `(hitX, hitY)` 为中心，半径为 `blastRadius` 的圆形弹坑实体
- **渲染效果**（双通道）：
  - Canvas 2D（`TileDamageSystem.render`）：径向渐变暗色填充 + 边缘环 + 锯齿状辐射裂纹 + 碎片散射
  - PixiJS（`RenderSystem.drawTileDamageMarks`）：大圆形填充 + 环 + 旋转矩形裂纹 + 碎片小圆点
- **去重**：同一弹坑位置重复命中时刷新计时器而非创建新实体
- **Fallback**：`craterRadius === 0` 时保留旧的逐格模式（向后兼容）

## 任务：敌人攻击动画 — 脉冲波形 + 冲撞缩放 + 受击反馈

### 需求
敌人到达基地时没有任何攻击动作，完全看不出来。敌人的视觉表现都是球形，需要设计明显的攻击动画。

### 设计方案（Phase 1+2 已实现）
- **受击反馈**：基地触发 `hitFlashTimer`（白色闪烁 120ms）+ `ScreenShakeSystem.triggerShake`（微震 3px/0.25s）
- **近战冲撞动画（squash-stretch）**：敌人缩进（0-35%）→ 弹向目标方向 + 放大（35-60%）→ 回弹（60-100%），持续 0.4s
- **脉冲波形**：3 层同心圆从敌人位置向外扩散并衰减，增强冲击感

### 改动文件
| 文件 | 改动 |
|---|---|
| `src/systems/MovementSystem.ts` | 新增攻击动画生命周期：首帧到达→触发攻击动画+伤害→0.4s后销毁；移除旧 `onReachEnd` 方法；导入 `ScreenShakeSystem` |
| `src/systems/RenderSystem.ts` | `drawEntities()` 新增敌人攻击动画渲染：squash-stretch 变形 + 脉冲波同心圆；区分简单形状敌人和有 `partsId` 的复合敌人 |
| `src/systems/MovementSystem.test.ts` | 更新"到达终点的敌人被销毁"测试：改为验证攻击动画生命周期（3帧流程：到达→动画中→销毁） |

### 实现细节

#### MovementSystem — 攻击动画状态机
- **首帧到达**：`attackDur === 0` → 进入 Case 2 → 造成伤害 + 设置 `attackAnimTimer/attackAnimDuration = 0.4` + 受击闪光 + 屏幕震动
- **动画进行中**：`attackAnimTimer > 0` → 暂停移动帧衰减计时器 → `continue`
- **动画结束**：`attackAnimTimer <= 0 && attackDur > 0` → Case 1 → `destroyEntity`
- **设计要点**：`attackAnimTimer` 在首帧到达后设置（不在顶部检查中衰减），第 2 帧开始衰减

#### RenderSystem — 双轨渲染
- 简单形状敌人（`partsId === 0`）：直接渲染带动画参数的形状
- 复合部件敌人（`partsId !== 0`）：传递给 `drawSoldierComposite`
- 动画参数通过 `attackAnimPosX/Y/Size` 局部变量传递，避免修改原始的 `const posX/posY`
- 脉冲波与实体同层（z+1），白色半透明描边圆
