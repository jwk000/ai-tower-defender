| `src/core/RuleHandlers.ts` | `play_sound` 规则播放前先归一化音效 key，无法识别时跳过 |
| `src/main.ts` | 胜利音频播放兼容 `bgm/*.ogg` / `sfx/*.ogg` 资源路径配置 |
| `src/core/RuleHandlers.test.ts` | 增加旧常量别名和路径形式音效配置的回归测试 |

### 验证结果

## 审计：敌方单位与 Boss 音效覆盖

### 需求

检查敌方单位音效是否齐全，重点关注 Boss 音效还需要补充哪些。

### 检查结果

- 当前 5 个已配置关卡实际会用到的敌方单位均可命中已有音效 key，未发现资源文件缺失。
- 普通敌人主要共用 `enemy_spawn`、`enemy_death`、`enemy_attack`、`mage_attack`、`poison_hit`、`stun_apply`、`soldier_heal` 等通用音效。
- 当前 Boss 均有出场和死亡音效链路：`SFX_BOSS_SPAWN -> wave_boss`，`SFX_BOSS_DIE -> victory`。
- Boss 技能已有 4 类通用音效：召唤 `boss_summon`、阶段/控制 `boss_phase2`、导弹/扭曲 `boss_missile`、吞噬/AOE `boss_devour`，巨型史莱姆分裂另有 `boss_split`。

### 缺口判断

- 普通敌人资源齐全但粒度偏粗，重装、机械、飞行、自爆、治疗/复活类敌人缺少专属事件音色。
- Boss 资源不缺文件，但表现层不够“Boss 专属”：14 个 Boss 共用同一出场和死亡音效，多个技能复用 `boss_summon` / `boss_phase2` / `boss_missile` / `boss_devour`。
- 最高优先级应补：Boss 专属出场、Boss 专属死亡、阶段转换、召唤、导弹预警/落地、吞噬、分裂、狂暴倒计时。

## 排查：第4关敌人半路停住

### 现象

第4关战斗截图中，大批敌人在中路交战区域聚集，看起来没有继续向水晶推进。

### 排查结果

- 第4关 `pathGraph` 三条出生路径均有完整后继，`spawn_0`、`spawn_1`、`spawn_2` 都连接到 `crystal_anchor`，未发现路径断点。
- MovementSystem 在敌人 FollowPath 移动前会先检查攻击：只要敌人有 `Attack` 组件且范围内存在士兵或塔，就执行攻击并把 `Movement.currentSpeed` 置为 0，本帧不推进路径。
- 攻击动画期间 `Visual.attackAnimTimer > 0` 时同样会停止移动。截图中的弓手位于中路交战点附近，第4关敌人配置包含机器狗、无人机、飞机、油罐、坦克等带攻击范围的单位，因此会被附近玩家单位吸住反复攻击。

### 结论

当前表现不是第4关路径配置卡死，而是敌人攻击玩家单位时主动停下。若产品目标是普通敌人持续冲水晶，需要先调整设计：例如仅特定敌人可停下拆塔/打士兵，或给普通敌人增加“忽略玩家单位继续推进”的行为标记。

## 修复：第4关无人机群停在半路

### 现象

第4关无人机/飞行敌人在路径中段或合流点附近没有攻击动作，但停在原地不继续前进。

### 根因

`drone` 属于 `EnemyFlockMember` 鸟群/无人机群敌人，移动走 `processFlockEnemy`。该逻辑会给成员施加 `anchorOffsetX/Y` 编队偏移，让成员追随“路径点 + 偏移”的目标点；但路径推进判定只检查实体是否靠近原始路径点。当偏移距离大于 `FLOCK_PATH_ADVANCE_RADIUS` 时，成员会稳定停在偏移目标附近，却永远不满足“靠近原始 waypoint”的推进条件。

### 改动

- `MovementSystem.processFlockEnemy` 的路径推进判定同时接受原始 waypoint 与带编队偏移的目标点。
- 增加回归测试：LowAir 鸟群敌人靠近带偏移的编队目标点时必须推进 `pathIndex` 并能结算到达水晶。

### 验证

- `npm test -- src/systems/MovementSystem.test.ts` 通过，16 个用例。
- `npm run typecheck` 通过。

## 修复：弓手/法师在场时敌人原地停住

### 现象

场上存在弓手、法师等远程士兵后，敌人会停在路径中段，不继续前进，也没有明显攻击我方单位。

### 根因

敌人被位移到路径段外后会触发 `MovementSystem.tryPathRecovery`。旧逻辑恢复到最近 waypoint 后本帧直接 `continue`，把 `Movement.currentSpeed` 置为 0。远程士兵追击/站位会让敌人与路径恢复持续交替出现，导致敌人每帧都只恢复、不推进，视觉上表现为全体原地停住且没有攻击动作。

### 改动

- 路径恢复后刷新 `pathIndex` 和当前位置，并在同一帧继续执行路径推进。
- 更新 path recovery 回归测试：敌人被拉回路径后应立即继续移动，`currentSpeed` 保持大于 0。

### 验证

- `npm test -- src/systems/__tests__/MovementSystem.soldier.test.ts src/systems/MovementSystem.test.ts src/systems/SoldierAISystem.test.ts` 通过，73 个用例。
- `npm run typecheck` 通过。

## 修复：弓手攻击不播放攻击动作

### 现象

弓手进入战斗并发射远程投射物时，角色仍持续显示移动动作，没有切到 `attack_0/1` 攻击帧。

### 根因

弓手攻击走 `SoldierAISystem` 的士兵战斗逻辑。该逻辑在攻击冷却完成后只生成投射物或近战刀光，没有刷新 `Visual.attackAnimTimer`。`RenderSystem.getUnitSpriteState()` 只有在 `Visual.attackAnimTimer > 0` 时才会优先选择 `attack` 状态，因此弓手实际开火后仍按移动状态取图。

### 改动

- `SoldierAISystem` 新增士兵普攻动画触发逻辑，远程投射物、近战攻击和处决攻击都会设置 `Visual.attackAnimTimer/attackAnimDuration`。
- `SoldierAISystem.test.ts` 增加弓手远程攻击触发攻击动作计时器的回归测试。

### 验证

- `npm test -- src/systems/SoldierAISystem.test.ts` 通过，40 个用例。
- `npm run typecheck` 未通过，失败点为工作区既有 `src/systems/EnemySkillSystem.ts` 音效 key 字面量类型错误：`boss_phase_ice`、`boss_missile_impact`、`boss_devour` 不能赋给当前收窄到 `boss_devour_impact` 的类型。
