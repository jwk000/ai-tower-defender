## 修复：音效设计资源调用链路校验

### 需求

检查代码是否真正调用了设计中的音效资源。

### 检查结果

- `Sound.ts` 声明的 70 个 SFX key 与代码/YAML/关卡音频配置交叉比对后，资源文件均存在。
- 发现 YAML 规则里仍使用 `SFX_ENEMY_DIE`、`SFX_BOSS_SPAWN` 等旧设计常量，原逻辑会直接传给 `Sound.play()` 并被静默丢弃。
- 发现关卡胜利配置使用 `bgm/victory_xxx.ogg`、`sfx/victory_xxx.ogg` 路径形式，原胜利播放逻辑直接传给 `Music.play()` / `Sound.play()`，不会命中 key。

### 改动

| 文件 | 改动 |
|------|------|
| `src/utils/Sound.ts` | 新增 `normalizeSfxKey()`，兼容真实 key、`sfx/*.ogg` 路径和旧 `SFX_*` 常量别名 |
| `src/core/RuleHandlers.ts` | `play_sound` 规则播放前先归一化音效 key，无法识别时跳过 |
| `src/main.ts` | 胜利音频播放兼容 `bgm/*.ogg` / `sfx/*.ogg` 资源路径配置 |
| `src/core/RuleHandlers.test.ts` | 增加旧常量别名和路径形式音效配置的回归测试 |

### 验证结果

- 集合比对确认：`unknownYaml=[]`，`unknownDirectSfx=[]`，`unknownLevelSfx=[]`，`unknownDirectBgm=[]`，`unknownLevelBgm=[]`。
- `npm test -- src/core/RuleHandlers.test.ts` 通过，59 个用例。
- `npm run typecheck` 通过。

## 审计：敌方单位与 Boss 音效覆盖

### 需求

检查敌方单位音效是否齐全，重点关注 Boss 音效还需要补充哪些。

### 检查结果

- 当前 5 个已配置关卡实际会用到的敌方单位均可命中已有音效 key，未发现资源文件缺失。
- 普通敌人主要共用 `enemy_spawn`、`enemy_death`、`enemy_attack`、`mage_attack`、`poison_hit`、`stun_apply`、`soldier_heal` 等通用音效。
- 当前 Boss 均有出场和死亡音效链路：`SFX_BOSS_SPAWN -> wave_boss`，`SFX_BOSS_DIE -> victory`。
- Boss 技能已有 4 类通用音效：召唤 `boss_summon`、阶段/控制 `boss_phase2`、导弹/扭曲 `boss_missile`、吞噬/AOE `boss_devour`，巨型史莱姆分裂另有 `boss_split`。

### 缺口判断

- 普通敌人资源齐全但粒度偏粗，重装、机械、飞行、自爆、治疗/复活类敌人缺少专属事件音色。
- Boss 资源不缺文件，但表现层不够“Boss 专属”：14 个 Boss 共用同一出场和死亡音效，多个技能复用 `boss_summon` / `boss_phase2` / `boss_missile` / `boss_devour`。
- 最高优先级应补：Boss 专属出场、Boss 专属死亡、阶段转换、召唤、导弹预警/落地、吞噬、分裂、狂暴倒计时。
