## 修复：敌人偶发从半路出现

### 现象

敌人有时不是从出生口出现，而是直接出现在路径中段；该现象不是巨型史莱姆分裂导致。

### 根因

敌人生成时只给 `Movement` 写入了 `speed`、`moveMode`、`spawnIdx`。bitecs 的组件字段是按实体 id 存储的数组，实体 id 被复用时，旧实体残留的 `pathIndex`、`progress`、`targetX/Y` 等运动字段可能被新敌人继承。新敌人虽然 `Position` 写在出生点，但第一帧移动系统会按旧路径进度接管，表现为从半路出现。

### 改动

| 文件 | 改动 |
|------|------|
| `src/systems/WaveSystem.ts` | 生成敌人时完整初始化 `Movement` 字段，确保 `pathIndex/progress/currentSpeed` 等出生状态归零，并同步出生点到 `targetX/Y`、`homeX/Y` |
| `src/systems/__tests__/WaveSystem.spawn.test.ts` | 增加实体 id 复用场景回归测试，验证新生敌人不会继承旧路径进度 |

- `npm run typecheck` 通过。
