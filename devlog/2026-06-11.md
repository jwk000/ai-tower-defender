# 2026-06-11 开发日志

## 修复：AI 美术资源运行时未加载

### 问题现象

运行游戏时看不到已生成的 AI 美术资源替换效果，画面仍显示旧的程序化几何外观。

### 根因

Vite 配置的部署路径为 `base: '/ai-tower-defender/'`，但新接入的大部分图片资源使用了根路径 `/art/...`。在 `http://127.0.0.1:3000/ai-tower-defender/` 下运行时，浏览器会请求错误的 `http://127.0.0.1:3000/art/...`，资源返回 404，渲染系统因此回退到旧几何兜底。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/utils/artAssets.ts` | 新增 `assetUrl()`，统一把 `public/art` 资源路径解析到当前 Vite `BASE_URL` 下 |
| `src/utils/imageCache.ts` | 图片懒加载缓存改为使用 `assetUrl()` 后的真实 URL |
| `src/systems/RenderSystem.ts` | 地格纹理缓存改为使用 `assetUrl()`，修复路径地格图片 404 |
| `src/systems/DecorationSystem.ts` | 背景图加载改为复用 `assetUrl()`，统一资源路径策略 |
| `src/utils/artAssets.test.ts` | 新增资源路径解析测试，防止部署 base 回归 |

### 验证结果

- `curl http://127.0.0.1:3000/art/cards/card_arrow_tower.png` 返回 404，确认旧路径错误。
- `curl http://127.0.0.1:3000/ai-tower-defender/art/cards/card_arrow_tower.png` 返回 200，确认修复后的路径可访问。
- `npm run typecheck` 通过。
- `npm test` 通过，75 个测试文件、1298 个测试全部通过。

## 修复：UI 图片假透明与拉伸变形

### 问题现象

AI 生成的 UI 图片标称透明背景，但实际文件是 RGB 图片，透明区域被生成为灰白棋盘格像素。运行时直接绘制后，按钮、面板、卡框周围出现棋盘格背景。同时，部分 UI 图片源尺寸与运行时目标尺寸不一致，直接拉伸会导致按钮、HUD 和面板外观变形。

### 根因

1. `public/art/ui/*.png` 没有 alpha 通道，假透明棋盘格已经写入图片像素。
2. `UISystem` 对按钮、HUD、面板使用整图拉伸绘制，没有区分固定比例图片和可伸缩 UI 皮肤。
3. 横向按钮皮肤被套用到所有按钮，包括速度、暂停、图鉴等近方形小按钮。

### 改动文件

| 文件 | 改动 |
|------|------|
| `public/art/ui/*.png` | 将边缘连通的灰白棋盘格背景转为真实透明 alpha，并裁掉无效透明边界 |
| `src/utils/imageCache.ts` | 新增 `drawLoadedImage9Slice()`，支持九宫格绘制可伸缩 UI 图片 |
| `src/systems/UISystem.ts` | 面板、HUD、横向按钮改用 9-slice 绘制；小方形按钮不再套用横向按钮皮肤 |

### 验证结果

- `public/art/ui/*.png` 均为 RGBA，且 alpha 通道包含透明像素。
- `npm run typecheck` 通过。
- `npm test` 通过，75 个测试文件、1298 个测试全部通过。

## 修复：按钮九宫格、图鉴图片与敌人图鉴素材

### 问题现象

按钮使用 UI 皮肤后看起来像缩小贴图，没有按九宫格正确拉伸。手牌区底板边缘和空槽存在感过强，遮挡战斗画面的轻量感。卡牌图鉴仍显示旧矢量图标，没有使用已生成的卡牌插画；敌人图鉴没有独立图片资源，只能显示程序化图标。

### 根因

1. `ui_button_green.png` 去底裁边后的实际尺寸变化，原九宫格切片上下边距过大，在运行时 28px 高按钮上被压缩。
2. 手牌区同时叠加 UI 面板图、实心底板、描边和空槽描边，视觉权重过高。
3. `CardEncyclopediaUI` 和 `EnemyCodexUI` 是独立 Canvas 渲染路径，未复用战斗 HUD 中的图片加载逻辑。
4. 生成资产清单未包含敌人图鉴图片。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/systems/UISystem.ts` | 调整按钮九宫格切片，横向按钮优先绘制图片皮肤；降低手牌底板和空槽透明度、描边强度 |
| `src/ui/CardEncyclopediaUI.ts` | 卡牌图鉴优先加载 `public/art/cards/card_<id>.png`，失败时回退矢量图标 |
| `src/ui/EnemyCodexUI.ts` | 敌人图鉴优先加载 `public/art/enemies/enemy_<id>.png`，失败时回退矢量图标 |
| `src/utils/artAssets.ts` | 新增 `enemyArtPath()` |
| `scripts/generate-ai-art-assets.mjs` | 将 24 个敌人图鉴素材加入统一模型生成清单 |
| `public/art/enemies/*.png` | 新增 24 张敌人图鉴素材，统一使用 `azure/gpt-image-2` 生成并本地去底裁边 |
| `art-generation-progress.json` | 记录 24 个敌人图鉴素材生成进度，全部 `done` |
| `design/09-ai-art-redesign.md` | 补充敌人图鉴素材路径和尺寸规范 |

### 验证结果

- 24 张 `public/art/enemies/*.png` 均为 RGBA，且 alpha 通道包含透明像素。
- `art-generation-progress.json` 中敌人素材全部 `done`，`modelLock` 保持 `azure/gpt-image-2`。

## 修复：场景单位美术资源接入

### 问题现象

战斗场景中的塔、士兵、敌人、机关等单位仍然全部使用代码生成的几何图形，没有替换成图片资源。

### 处理策略

1. 保留原有 `visualParts` / `drawUnitComposite` / 陷阱 switch 绘制逻辑，作为资源缺失或加载失败时的兜底。
2. 运行时优先加载 `public/art/units/unit_<unit_id>_idle_<frame>.png`，通过呼吸相位在 0/1 两帧之间切换。
3. 继续复用现有代码动画和状态叠加，包括攻击缩放、移动朝向、受击闪白、冰冻、眩晕、中毒、精英描边和 Boss 阶段效果。
4. 场景单位 AI 直接生成在当前网关两次读超时，因此本轮先用已经由 `azure/gpt-image-2` 生成的卡牌图和敌人图派生 94 张透明场景精灵帧；生成脚本和进度清单仍保留直接生成任务，后续接口稳定后可继续覆盖。

### 改动文件

| 文件 | 改动 |
|------|------|
| `src/systems/RenderSystem.ts` | 新增场景单位图片缓存和绘制路径；塔/士兵/敌人/机关优先使用图片，失败回退程序化绘制 |
| `src/render/Renderer.ts` | 支持图片渲染时按 `scaleX` 水平翻转，保留单位朝向 |
| `src/systems/WaveSystem.ts` | 敌人生成时写入类型索引，保证场景敌人能匹配到正确素材 |
| `src/utils/artAssets.ts` | 新增 `unitArtPath()` |
| `scripts/generate-ai-art-assets.mjs` | 补充 47 个场景单位、每个 2 帧的直接生成清单 |
| `public/art/units/*.png` | 新增 94 张场景单位 idle 透明帧 |
| `art-generation-progress.json` | 记录 94 张场景单位帧为 `done`，并标注当前由同模型既有素材本地派生 |
| `design/09-ai-art-redesign.md` | 补充场景单位运行时接入和兜底规则 |

### 验证结果

- `npm run typecheck` 通过。
- `npm test` 通过，75 个测试文件、1300 个测试。
- 94 张 `public/art/units/*.png` 均存在，资源进度清单单位项全部 `done`。

## 修复：法术释放特效假透明背景

### 问题现象

火球、箭雨、暴风雪、炸弹等法术卡释放后，投射物拖尾路径和命中特效图片带有灰底或棋盘格背景，叠加到场景上会出现明显方形底色。

### 根因

`public/art/fx/` 下法术释放链路实际引用的 FX 图片由 AI 生成后仍是 RGB PNG，提示词里的透明背景没有变成真实 alpha 通道。运行时 `SpellProjectileSystem` 正常按图片绘制，但资源本身没有透明区域。

### 改动文件

| 文件 | 改动 |
|------|------|
| `public/art/fx/fx_fire_explosion_charge_0.png` | 本地去底，输出 RGBA 真透明 PNG |
| `public/art/fx/fx_fire_explosion_impact_1.png` | 本地去底，输出 RGBA 真透明 PNG |
| `public/art/fx/fx_arrow_rain_impact_1.png` | 本地去底，输出 RGBA 真透明 PNG |
| `public/art/fx/fx_ice_burst_impact_1.png` | 本地去底，输出 RGBA 真透明 PNG |
| `public/art/fx/fx_bomb_explosion_impact_1.png` | 本地去底，输出 RGBA 真透明 PNG |
| `src/utils/artAssets.test.ts` | 补充法术投射物和命中特效路径契约测试 |
| `design/09-ai-art-redesign.md` | 补充法术投射物、拖尾、释放效果必须为真实 alpha PNG 的验收规则 |

### 验证结果

- 5 张法术释放链路 FX 图片均为 RGBA，且 alpha 通道包含透明像素。
- `npm run typecheck` 通过。

| 文件 | 改动 |
|------|------|
| `src/utils/pathTileTexture.ts` | 移除路径方向邻接判断，`TileType.Path` 固定返回 `tile_<theme>_path.png` |
| `src/utils/pathTileTexture.test.ts` | 改为验证直线、拐角、三通、十字布局都使用同一张无方向路径贴图 |
| `scripts/generate-ai-art-assets.mjs` | 生成清单不再追加直线、拐角、三通、十字路径件，仅保留基础路径和端点图 |

### 验证结果

- `design/art-path-tile-connectors.json` 可正常解析。
