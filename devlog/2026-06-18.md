## 修复弩塔穿透弹误伤自身

- 问题：弩塔攻击敌人时自身会掉血。
- 原因：`ProjectileSystem` 的弩塔穿透弹分支只按 `Position + Health + UnitTag` 扫描沿途实体，并只检查层级可命中，没有校验发射源与目标阵营是否敌对；源塔和友方塔也可能被穿透线段命中。
- 修复：在公共 `canProjectileHitTarget` 中加入来源实体与目标实体的 `Faction` 敌对关系校验，确保有来源的弹道只能命中敌对单位。
- 回归：新增弩箭穿透测试，覆盖弩箭经过源塔、友方塔、敌人时只伤害敌人；同时让 ProjectileSystem 测试夹具补齐 `Faction`，更贴近真实运行实体。
- 验证：`npm test -- ProjectileSystem.test.ts` 通过；`npm run typecheck` 通过。

## 修复进关手牌分类 includes 崩溃

- 问题：点击进入第一关后第一帧崩溃，浏览器 fatal overlay 显示 `TypeError: Cannot read properties of undefined (reading 'includes')`。

- 修复：`main.ts` 的 UI registry 改为保存完整 `CardConfig`，优先使用 YAML 静态卡牌配置，找不到时补齐 fallback 配置必填字段；`UISystem.getHandCardCategory` 增加字符串守卫，避免旧形态半截配置再次触发 fatal。
- 调试：自定义 `onUpdate` 系统循环补齐逐系统异常捕获和 `systemCrashed` 日志，后续系统崩溃会标出具体系统名和堆栈。
- 回归：新增 `UISystem` 用例覆盖 registry 返回旧形态半截配置时不因缺少 `id` 崩溃。
- 验证：浏览器刷新后点击第一关，可正常进入战斗画面且无新 fatal；`npm test -- --run src/systems/UISystem.test.ts`、`npm run typecheck`、`npm run build` 通过。
