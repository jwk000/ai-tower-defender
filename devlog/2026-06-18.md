# 2026-06-18 开发日志

## 修复弩塔穿透弹误伤自身

- 问题：弩塔攻击敌人时自身会掉血。
- 原因：`ProjectileSystem` 的弩塔穿透弹分支只按 `Position + Health + UnitTag` 扫描沿途实体，并只检查层级可命中，没有校验发射源与目标阵营是否敌对；源塔和友方塔也可能被穿透线段命中。
- 修复：在公共 `canProjectileHitTarget` 中加入来源实体与目标实体的 `Faction` 敌对关系校验，确保有来源的弹道只能命中敌对单位。
- 回归：新增弩箭穿透测试，覆盖弩箭经过源塔、友方塔、敌人时只伤害敌人；同时让 ProjectileSystem 测试夹具补齐 `Faction`，更贴近真实运行实体。
- 验证：`npm test -- ProjectileSystem.test.ts` 通过；`npm run typecheck` 通过。

## 优化抽卡连续重复规则

- 问题：出牌后的自动补牌仍可能立刻补回刚打出的同一张卡，玩家感知为连续重复抽牌。
- 修复：`HandSystem.drawRandomCard` 支持传入本次需要排除的卡牌 ID；`playCard` 自动补牌时优先排除刚打出的卡，再尽量排除上一张成功抽入手牌的卡。
- 兜底：若排除后没有其他合法候选，仍允许抽取被限制卡，避免小卡池或手牌类别限制导致补牌失败。
- 回归：新增出牌后不会立刻补回刚打出卡的单测，并保留随机补牌不连续重复与唯一候选兜底测试。
- 验证：`npm test -- HandSystem.test.ts` 通过；`npm run typecheck` 通过。

- 需求：虫族女王遇到塔必须攻击塔直到杀死塔，遇到我方兵也必须攻击兵直到杀死兵。

- 回归：新增塔锁定与士兵锁定两个用例，覆盖目标存活不切换、目标死亡后释放锁定。
- 验证：`npm test -- MovementSystem.test.ts` 通过；`npm run typecheck` 通过。

## 重制超级机器人平稳移动帧

- 问题：超级机器人 `move_0` / `move_1` 的整机姿态明显倾斜，移动循环看起来像身体斜倒，而不是重型机械平稳行进。

- 修复：以旧 `move_0` / `move_1` 作为基准图重新编写提示词生成两帧朝左移动动作表；替换 `public/art/units/unit_enemy_super_robot_move_0.png` 与 `unit_enemy_super_robot_move_1.png`，两帧保留基准形象和明显迈步动作，同时躯干、头肩和红色核心不再明显倾斜。
- 接入：重新生成美术图集，更新 `public/art/atlases/` 中引用超级机器人移动帧的关卡图集图片；帧路径和坐标未变化，不需要改动图集索引。
- 验证：检查两张移动帧仍为 256×256 RGBA 透明 PNG；生成 `tmp/ai-action-sheets/super_robot_move_candidates_final_preview.png` 对比确认新帧沿用基准超级机器人形象、腿部保持朝左、不是站立动作，且躯干不再明显倾斜。

## 关卡入口阻塞预加载本关资源

- 问题：发布到 GitHub Pages 后，首次打开关卡会边进关边请求图集和图片，冷缓存下容易出现加载慢、首帧缺图或进入战斗后卡顿。

- 修复：新增 `levelAssetPreloader` 按 `LevelConfig` 推导本关 atlas id；`imageCache` 支持按 atlas id 阻塞式预加载并复用现有缓存。
- 接入：`startLevel` 改为异步加载流程，点击关卡后显示加载进度，资源加载失败时警告并继续走懒加载兜底，避免单个 404 卡死进关。
- 验证：新增 `imageCache` 与 `levelAssetPreloader` 单测覆盖按 id 预加载和第五关资源清单。
