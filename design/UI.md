# UI / UX 设计

> 设计分辨率：1920×1080 横屏 | 锚点定位系统 | 卡牌驱动交互 | 单 Run 闭环无 meta
> 当前界面规则版本：**v3.6（2026-05-19）**

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **符号优于文字** | 场景中不显示单位名称，形状/动画/特效让玩家一眼识别 |
| **卡牌即交互锚** | 战斗中所有部署操作从手牌发起，手牌区是视觉重心 |
| **按需显示** | 选中实体时才弹出详情面板；HUD 只展示必要数字 |
| **拖拽优先** | 出卡部署用拖拽，给玩家"打出去"的物理感 |
| **关间正式化** | 关间节点（商店/秘境/过关奖励）使用全屏模态面板，与战斗 UI 隔离 |
| **抽卡是主动决策** | UI 必须明确展示"可抽 / 不可抽 / 本次是否还能重抽"，避免误解为自动补牌 |

### 2.1 区域划分

| 区域 | 内容 | 尺寸 | 位置 |
|------|------|------|------|
| 顶部 HUD | 水晶能量/恢复倒计时/金币/波次/敌军/水晶HP/Run进度/抽卡按钮/倍速/暂停 | 1920×36 | y=0 |
| 全屏背景 | 天空渐变+远景+云+天气粒子 | 1920×1080 | 全屏 |
| 地图区域 | 21×9格战场 | 1344×576 | 居中 (x=288) |
| **手牌区** | 4张手牌（横向排列），固定上限4张 | 800×180 | bottom-center, offset(0,-130) |
| **抽卡操作区** | 抽卡按钮 / 当前是否还能重抽 / 满手提示 | 240×96 | 手牌区右侧 |
| **卡池状态区** | 当前卡池张数 / 已删卡次数 / 下次删卡费用提示（只读） | 220×80 | bottom-right, offset(-40,-170) |
| **塔图鉴提示区** | 选中塔卡时显示其流派标签（减速/灼烧/中毒/聚焦/穿透） | 260×96 | 手牌区左侧 |
| 选中信息面板 | 选中地图实体时浮现 | ~300×100 | 手牌区上方动态浮出 |

### 2.2 UI 死区规范

死区由 UI 区域实际位置动态计算（不硬编码）：

```typescript
const UI_DEAD_ZONES: UIDeadZone[] = [
  { region: 'top_hud',   hitTest: (x, y) => y < scaleY(HUD_HEIGHT) },
  { region: 'hand_area', hitTest: (x, y) => y > viewportH - scaleY(HAND_HEIGHT) },
  { region: 'draw_area', hitTest: (x, y) => DrawPanel.contains(x, y) },
  { region: 'tooltip_overlay', hitTest: (x, y) => SelectedInfoPanel.contains(x, y) },
];
```

死区优先级：模态覆盖层 > 手牌区 > 抽卡操作区 > 顶部HUD > 选中面板

## 3. 顶部 HUD

| 显示项 | 符号 | 格式 | 说明 |
|--------|------|------|------|
| 水晶能量 | 蓝色菱形 ◇ | `◇ 7/10` | 当前/上限 |
| 回能节奏 | 环形倒计时 ⟳ | `⟳ 2s` | 距离下 1 点能量恢复剩余时间 |
| 金币 | 金色圆点 ● | `● 120` | 本 Run 持有 |
| 波次 | 旗标 ⚑ | `⚑ 3/8` | 当前/总波次 |

## 4. 手牌与抽卡系统

### 4.1 手牌区布局

```
       ┌──────────────────────────────────────────────────────────────┐
       │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐      [抽卡]               │
       │  │箭塔 │ │剑士 │ │火球 │ │金矿 │      可重抽 1 次          │
       │  │ 🏹  │ │ ⚔  │ │ 💥  │ │ ⛏  │      或：手牌已满 / 冷却中   │
       │  │ ◇3  │ │ ◇2  │ │ ◇3  │ │ ◇4  │                            │
       │  └─────┘ └─────┘ └─────┘ └─────┘                            │
       └──────────────────────────────────────────────────────────────┘
```

- 单卡尺寸：120×168，横向间距 16px
- 固定上限 4 张
- **不再存在“打出立即补牌”**
- 手牌区右侧常驻展示当前抽卡状态
- 塔卡名称与图标需明确区分：**冰塔 / 火塔 / 毒塔 / 激光塔 / 弩塔**，避免旧“元素塔”误读

### 4.2 单卡视觉规范

```
┌────────────────┐
│ ╭────────────╮ │   ← 边框颜色 = 当前卡牌等级
│ │  单位图形  │ │       L1 = 🟢绿色 / L2 = 🔵蓝色 / L3 = 🟡金色
│ │            │ │
│ ╰────────────╯ │
│  箭塔  Lv.2    │
│  核心输出卡    │   ← 可选短标签 / 关键词
│  ◇3            │   ← 水晶能量消耗
└────────────────┘
   120 × 168
```

| 等级 | 边框颜色 | 触发条件 |
|------|---------|---------|
| L1（基础） | 🟢 绿色 | 入手默认 |
| L2（进阶） | 🔵 蓝色 | 关间花金币升级后 |
| L3（满级） | 🟡 金色 | 若该卡支持 Lv.3，则关间升满后 |

### 4.3 抽卡交互流程（v3.6）

| 阶段 | 操作 | UI 反馈 |
|------|------|---------|
| 可抽 | 距上次抽卡已满 5 秒，且手牌未满 | `[抽卡]` 亮起，边框发光 |
| 第一次抽取 | 点击抽卡按钮 | 抽出 1 张并弹出 `[保留] [丢弃并重抽]` |
| 重抽机会 | 玩家选择丢弃 | 原卡飞入弃牌动画区，立刻补出第 2 张 |
| 第二次抽取 | 重抽结果出现 | 显示 `必须保留` 标签，不再出现丢弃按钮 |
| 不可抽 | 手牌已满或 CD 未到 | 按钮置灰，并显示 `手牌已满` / `3s后可抽` |

### 4.4 出卡交互流程

| 操作 | 视觉反馈 |
|------|---------|
| 悬停 | 卡片上移 12px + 缩放 1.05× + 详情弹窗 |
| 拖出 | 卡片消失，光标跟随幽灵预览 |
| 合法目标 | 地图格绿色高亮 + 卡片轮廓变绿 |
| 非法目标 | 红色高亮 + 卡片轮廓变红 |
| 松开（合法） | 部署/释放，扣水晶能量，卡离开手牌 |
| 松开（非法） | 取消，卡回手牌 |
| 能量不足 | 卡片暗淡 + 红色`能量不足`标识 |

### 4.5 卡池状态区

```
卡池：12 张
已删卡：2 次
下次删卡：80G
```

- 战斗内仅作只读提示，不提供删卡操作
- 让玩家记得：当前抽卡质量来自关间构筑结果

## 6. 响应式布局系统

### 6.1 核心架构

```
游戏世界（地图/实体）: 设计空间(1920×1080) → scaleFactor → 视口空间
UI 元素: 锚点系统 → 视口空间绝对定位
```

- 内部逻辑坐标系 1920×1080
- `scaleFactor = viewportH / 1080`（以高度为基准等比缩放）

### 6.2 锚点系统

```
┌───────────────┬───────────────┬───────────────┐
│  TOP_LEFT     │  TOP_CENTER   │  TOP_RIGHT    │
├───────────────┼───────────────┼───────────────┤
│  MIDDLE_LEFT  │ MIDDLE_CENTER │ MIDDLE_RIGHT  │
├───────────────┼───────────────┼───────────────┤
│  BOTTOM_LEFT  │ BOTTOM_CENTER │ BOTTOM_RIGHT  │
└───────────────┴───────────────┴───────────────┘
```

### 6.3 LayoutManager API

```typescript
class LayoutManager {
  static viewportW: number;
  static viewportH: number;
  static scale: number;        // viewportH / 1080
  static anchor(config: AnchorConfig): { x: number; y: number };
  static scaleX(designPx: number): number;
  static scaleY(designPx: number): number;
}
```

### 6.4 主要锚点配置

| 元素 | 锚点 | 尺寸 |
|------|------|------|
| HUD 背景条 | `top-left, offset(0,0)` | fullWidth × 36 |
| 手牌区背景 | `bottom-center, offset(0,-130)` | 800 × 180 |
| 抽卡操作区 | `bottom-center, offset(470,-145)` | 240 × 96 |
| 卡池状态区 | `bottom-right, offset(-40,-170)` | 220 × 80 |

## 7. 关间界面

### 7.1 关间结构

v3.6 关间成长来源统一为：
- **过关奖励**
- **商店**
- **秘境**

因此 UI 不再表达旧版"单一三选一路线图节点"，而是表达为：

```
关卡结算
  ↓
过关奖励面板
  ↓
商店 / 秘境（若触发）
  ↓
进入下一关
```

### 7.2 商店界面目标

商店必须突出三种核心操作：
- **买新卡**
- **升级已有卡**
- **删除卡**（显示本次费用与下次翻倍后的费用）

推荐信息结构：
- 左侧：当前卡池与等级分布
- 中间：可购买新卡
- 右侧：升级区 / 删除区 / 刷新区

### 7.4 卡池界面

卡池界面是 **关外构筑总览面板**，用于让玩家查看本 Run 当前已经拥有的全部卡、卡牌等级分布、删卡成本趋势，以及哪些卡会进入关内抽卡池。

**打开时机**：
- 过关奖励结算后
- 商店内点击 `查看卡池`
- 秘境结算后若奖励改变了卡池，自动弹出或高亮入口

**禁止时机**：
- 战斗进行中不可打开
- 抽卡、拖卡、暂停菜单遮罩期间不可叠加打开

#### 7.4.1 界面目标

卡池界面要回答玩家 4 个问题：
1. **我现在有哪些卡？**
2. **哪些卡已经升级，哪些还没升级？**
3. **当前卡池厚度是否会影响关内抽卡质量？**
4. **如果继续删卡，下次要花多少钱？**

#### 7.4.2 推荐布局

```text
┌─────────────────────────────────────────────────────────────────────┐
│ 卡池总览                           当前金币 120G        [关闭]      │
├──────────────────────┬──────────────────────────────────────────────┤
│ 左侧：构筑摘要       │ 右侧：卡牌列表 / 详情                         │
│                                                          │
│ 卡池张数：12         │ [全部] [塔] [士兵] [法术] [建筑] [功能]       │
│ 已删卡：2 次         │                                              │
│ 下次删卡：80G        │ ┌────┐ ┌────┐ ┌────┐ ┌────┐                  │
│ 平均费用：◇3.1       │ │箭塔│ │剑士│ │火球│ │金矿│ ...              │
│ 满级卡：4            │ │L2 │ │L1 │ │L3 │ │L1 │                  │
│ 未满级卡：8          │ └────┘ └────┘ └────┘ └────┘                  │
│                      │                                              │
│ 说明：关内抽卡只会   │ 选中卡后右侧下方面板：                         │
│ 从这些卡中抽出       │ 名称 / 类型 / 当前等级 / 最高等级 / 能量费用   │
│                      │ 定位标签 / 说明 / 可否在商店升级               │
└──────────────────────┴──────────────────────────────────────────────┘
```

#### 7.4.3 左侧摘要区

| 字段 | 含义 |
|------|------|
| 卡池张数 | 当前 Run 可进入关内抽卡池的唯一卡数量 |
| 已删卡次数 | 本 Run 已执行删卡的累计次数 |
| 下次删卡费用 | 下一次在商店删除卡牌所需金币 |
| 平均费用 | 当前卡池中卡牌平均水晶能量消耗 |
| 满级卡数量 | 已达到各自等级上限的卡数 |
| 未满级卡数量 | 仍可通过商店继续升级的卡数 |

#### 7.4.4 卡牌列表规范

- 默认按：**类型 → 等级 → 名称** 排序
- 支持筛选：全部 / 塔 / 士兵 / 法术 / 建筑 / 功能
- 每张卡应显示：
  - 名称
  - 当前等级
  - 水晶能量消耗
  - 类型标签
  - 核心职责标签
- 对已获得卡明确标注：`已纳入卡池，不会重复获得`

#### 7.4.5 单卡详情区

展示内容：
- 当前等级
- 最高等级（1-3）
- 当前能量消耗
- 职责说明
- 下一等级预览（若可升级）
- 是否已满级

#### 7.4.6 与商店界面的关系

- 卡池界面负责：**查看 / 规划 / 理解抽卡池**
- 商店界面负责：**买卡 / 升卡 / 删除**
- 因此卡池界面默认不直接执行删卡，只提供跳转或提示

#### 7.4.7 结果反馈要求

- 若秘境事件改变卡池：结算页必须明确写出 `卡池变化：+1 新卡 / -1 删除 / 某卡升至 Lv.2`
- 若过关奖励拿到新卡：卡池界面首次打开时高亮新卡

## 8. 关卡路线图 / 关间流程页

### 8.1 页面定位

路线图页用于承接战斗结束后的流程导航，让玩家知道：
- 本关结果如何
- 当前即将进入的是哪一关 / 哪一种节点
- 这一关的地形主题、节奏提示、敌人构成
- 下一次点击「进入挑战」前，自己是否需要先查看卡池或直接开战

### 8.2 重新设计目标（验收修正）

当前版本问题：
- 路线节点过小，主视觉不聚焦
- 全屏留白过多，地图氛围感弱
- 关卡描述过短且不换行，信息难读
- 玩家在进入关卡前看不到敌人压力来源

因此路线图页改为 **「当前关卡大入口 + 底部路线缩略带 + 右侧敌情情报」** 结构，目标是：
1. **当前关卡入口明显放大**，进入挑战动作成为视觉焦点
2. **补足背景层与场景氛围**，避免大面积纯色空白
3. **关卡描述支持两到三行换行**，明确主题与风险
4. **增加敌人介绍区**，让玩家开战前知道本关主要敌方构成
5. **保留整体路线进度表达**，但不再让所有节点抢主视觉

### 8.4 布局规则

#### 8.4.1 主视觉层级

- **当前关卡主卡**为页面第一视觉中心，占据中上区域
- 当前关卡入口尺寸必须显著大于底部路线节点，建议至少为普通节点的 **2.2~2.8 倍面积**
- 底部路线节点改为 **进度缩略带**，负责表达推进顺序，不承担主要信息阅读任务
- Boss 节点允许再额外放大，并使用不同描边/底色强调终局压力

#### 8.4.2 背景层

路线图背景至少分 3 层：
1. **底色渐变层**：根据关卡主题切换冷暖色
2. **装饰剪影层**：山体 / 树林 / 遗迹 / 洞窟 / 菌毯等低对比背景元素
3. **路径引导层**：连接主卡与底部路线带的光路 / 虚线 / 地图线条

要求：
- 背景是信息补足，不得盖过当前关卡主卡与按钮可读性
- 页面空白可被背景层填充，但不能堆满小装饰造成噪声

#### 8.4.3 文本换行

- 关卡名称：1 行，必要时缩小字号，不截断
- 关卡描述：**允许 2~3 行自动换行**
- 超过 3 行时才截断为 `…`
- 描述文本必须优先表达：
  - 地图节奏（长线 / 多拐点 / 双线压力）
  - 关键威胁（快兵 / 召唤 / 爆炸 / Boss）
  - 建议动作（先补前排 / 注意清群 / 推荐查看卡池）

#### 8.4.4 敌人介绍区

敌情情报区放在主卡右侧或右下侧，至少展示：
- 敌人名称
- 数量（可按整关累计）
- 最多展示 5 项主要敌人

展示优先级：
1. Boss / 精英
2. 数量最多的杂兵
3. 机制特殊怪（快兵 / 爆炸怪 / 召唤怪等）

若某关没有有效敌情数据，则显示：
- `敌情待侦察`
- `进入后根据波次逐步揭示`

#### 8.4.5 路线节点缩略带

- 所有关卡仍按顺序展示在底部
- 已完成：低亮 + 对勾
- 当前：放大、加高亮描边、与主卡颜色同步
- 未解锁：灰化
- 普通节点尺寸建议在 140~180 区间；当前缩略节点可提升到 190~220
- 节点内仅保留最关键摘要：节点类型 / 关卡名 / 波数，不再塞入长描述

### 8.5 交互与状态表达

| 状态 | 表现 |
|------|------|
| 当前关卡主卡 | 高亮描边 + 明显 CTA 按钮 |
| 已完成缩略节点 | 低亮底色 + `✓` |
| 当前缩略节点 | 与主卡同色系高亮 |
| 未解锁缩略节点 | 灰化 + 隐藏详情 |
| Boss 节点 | 特殊红金/紫金主题 |

### 8.6 关卡路线图面板锚点

| 元素 | 锚点 | 尺寸 |
|------|------|------|
| 顶栏标题 | `top-left, offset(36, 24)` | 自适应 |
| 顶栏资源 | `top-center / top-right` | 自适应 |
| 当前关卡主卡 | `middle-center, offset(-140, -120)` | **900 × 380** |
| 敌情情报卡 | `middle-right, offset(-120, -100)` | **300 × 360** |
| 进度缩略带 | `bottom-center, offset(0, -170)` | **1680 × 180** |
| 进入挑战按钮 | 主卡内底部居中 | **360 × 76** |
| 卡池按钮 | `bottom-right, offset(-220, -56)` | 140 × 52 |
| ESC 退出 | `top-right, offset(-40, 24)` | 140 × 44 |

### 8.7 实现约束

- 路线图必须优先保证 **当前关卡入口可读性**，而不是平均铺满所有节点
- 当前关卡描述、敌情情报、节点缩略带三者必须同时存在
- 视觉允许使用纯程序绘制的大色块背景，不要求先接入真实美术资源
- 若代码侧暂无完整敌人图鉴数据，可先基于关卡波次统计生成 `敌人名称 + 数量` 文本列表

## 9. 商店界面

### 9.1 界面目标（v3.6）

商店是 **关外可控成长界面**，核心只做三件事：
- **买新卡**
- **升级已有卡**
- **删除卡**

设计上必须让玩家一眼回答以下问题：
1. 我现在有多少钱？
2. 这次应该扩卡池、升核心卡，还是删废卡？
3. 删掉一张卡现在多少钱，删完下次会涨到多少？
4. 哪些卡已经满级，哪些还能继续升级？

> 旧版商店中的“功能卡橱窗 / 水晶恢复商品 / 商店内科技树升级”不再作为 v3.6 核心表达，统一收束为围绕卡池管理的构筑界面。

### 9.2 推荐布局（全屏 1920×1080）

```text
┌─────────────────────────────────────────────────────────────────────────────┐
│ 🏪 商店                    当前金币 240G                      [查看卡池] [离开] │
├──────────────────────┬──────────────────────────────┬───────────────────────┤
│ 左：卡池摘要         │ 中：可购买新卡                │ 右：成长操作区         │
│                      │                              │                       │
│ 卡池张数：12         │ ┌────┐ ┌────┐ ┌────┐        │ 升级已有卡             │
│ 满级卡：4            │ │箭塔│ │火球│ │牧师│        │ ┌───────────────┐      │
│ 未满级卡：8          │ │60G │ │85G │ │70G │        │ │ 箭塔 Lv.1 → Lv.2│      │
│ 已删卡：2 次         │ └────┘ └────┘ └────┘        │ │ 花费：45G       │      │
│ 下次删卡：80G        │                              │ └───────────────┘      │
│ 平均费用：◇3.1       │ 刷新：30G                     │                       │
│                      │                              │ 删除卡                 │
│ 当前构筑倾向：       │ 新卡均为唯一获得               │ 选择 1 张已有卡删除     │
│ 清兵 / 控场 / 运营    │ 已拥有的卡不再出现             │ 本次：80G              │
│                      │                              │ 下次：160G             │
│                      │                              │ [确认删除]             │
└──────────────────────┴──────────────────────────────┴───────────────────────┘

### 9.3 三栏信息职责

| 区域 | 职责 | 必显信息 |
|------|------|---------|
| 左栏：卡池摘要 | 帮玩家快速判断构筑状态 | 卡池张数 / 满级数 / 未满级数 / 已删卡次数 / 下次删卡费用 |
| 中栏：买新卡 | 提供扩池入口 | 候选新卡、价格、类型、等级上限、职责标签 |
| 右栏：升级/删卡 | 提供定向优化入口 | 当前选中卡、升级花费、当前等级、上限等级、删卡费用与翻倍预告 |

### 9.4 卡池界面与商店联动

- 从商店右上角点击 **[查看卡池]** 打开卡池界面
- 卡池界面只负责**查看与规划**：看当前卡池厚度、卡牌等级、删卡成本、构筑倾向
- 商店界面负责**执行操作**：购买新卡、升级已有卡、删除已有卡
- 从卡池界面点选某张卡后，返回商店时应保留该卡为当前选中对象，便于直接执行升级或删除
- 若某张卡已满级，商店右侧成长操作区应显示 `已满级`，隐藏升级按钮，仅保留详情与删除入口
- 若玩家金币不足，商店保留按钮但置灰，并明确显示 `还差 XXG`

### 9.5 买新卡区

- 仅展示**当前尚未拥有**的卡
- 所有新卡遵守 **唯一获得** 规则
- 每张商品卡显示：
  - 名称
  - 类型（塔 / 士兵 / 法术 / 建筑 / 功能）
  - 当前获得后等级（默认 Lv.1）
  - 最高等级（1-3）
  - 水晶能量消耗
  - 金币价格
  - 职责标签（清兵 / 单体 / 运营 / 治疗 / 控制 等）
- 若该卡已拥有，则商品位不应再刷出同名卡

### 9.5 升级区

升级区用于强化**已拥有卡**，不负责买新卡。

#### 9.5.1 展示规则

- 先从卡池中选中一张卡，再在右栏显示其升级信息
- 需明确显示：
  - 当前等级
  - 最高等级
  - 下一等级效果预览
  - 升级所需金币
- 若卡已满级：
  - 显示 `已满级`
  - 升级按钮置灰

#### 9.5.2 交互优先级

- 默认优先推荐：
  1. 未满级核心卡
  2. 当前构筑高频抽到/高价值卡
- 但推荐仅作视觉弱提示，**不替玩家自动决策**

### 9.6 删除卡区

删除卡是商店中的高价值构筑操作，必须做强提示。

#### 9.6.1 展示内容

- 当前已删卡次数
- **本次删除费用**
- **下次删除费用（翻倍后）**
- 当前被选中用于删除的卡
- 删除后的提示文案：`删除后，该卡将不再进入本 Run 关内抽卡池`

#### 9.6.2 交互限制

- 只能删除**当前已拥有**的卡
- 不能删除不存在于当前卡池的卡
- 删除需二次确认
- 删除完成后：
  - 卡从卡池移除
  - 下次删卡费用立即刷新为翻倍后的值
  - 若玩家正在查看该卡详情，右栏自动关闭或切换到空态

### 9.7 空态与禁用态

| 场景 | UI 表现 |
|------|---------|
| 金币不足买卡 | 商品卡价格标红，购买按钮置灰 |
| 金币不足升级 | 升级按钮置灰，并显示 `金币不足` |
| 金币不足删卡 | 删除按钮置灰，并显示 `本次需 80G` |
| 全卡已拥有 | 买新卡区显示 `当前卡池已收满可获得卡` |
| 无可升级卡 | 升级区显示 `当前无可升级卡` |
| 无可删卡目标 | 删除区显示 `请选择一张卡` |

### 9.8 商店与卡池界面的关系

- **商店**：负责执行操作（买 / 升 / 删）
- **卡池界面**：负责总览与规划
- 商店内点击 `查看卡池` 可打开卡池总览，但关闭后返回商店原位置

### 9.9 商店锚点建议

| 元素 | 锚点 | 尺寸 |
|------|------|------|
| 顶栏 | `top-left, offset(0,0)` | fullWidth × 72 |
| 左栏摘要 | `top-left, offset(40,96)` | 360 × 820 |
| 中栏买卡区 | `top-left, offset(440,96)` | 720 × 820 |
| 右栏成长区 | `top-right, offset(-40,96)` | 320 × 820 |
| 查看卡池按钮 | 顶栏右侧 | 140 × 44 |
| 离开按钮 | 顶栏最右 | 120 × 44 |

## 10. 秘境界面

### 10.1 界面目标（v3.6）

秘境是 **高波动事件界面**，职责不是卖货，而是让玩家在风险与收益之间做一次明确选择。

秘境 UI 必须清楚表达：
- 事件背景是什么
- 每个选项会付出什么
- 每个选项可能得到什么
- 该结果是否会影响卡池 / 金币 / 水晶 / 当前构筑方向

### 10.2 结构模板

```text
┌─────────────────────────────────────────────────────────────────────┐
│ 🌀 秘境：古老祭坛                                         [离开]    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  你来到一座被遗忘的祭坛，石板上刻着古老的咒文。                     │
│  中央的晶柱仍在微弱发光，似乎愿意回应一次交换。                     │
│                                                                     │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ ① 触碰晶柱                                                    │  │
│  │    获得 1 张稀有卡；失去 100 水晶 HP                            │  │
│  └───────────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ ② 献祭金币                                                    │  │
│  │    支付 80G；随机将 1 张卡升至下一等级                         │  │
│  └───────────────────────────────────────────────────────────────┘  │
│  ┌───────────────────────────────────────────────────────────────┐  │
│  │ ③ 离开                                                        │  │
│  │    什么都不发生                                               │  │
│  └───────────────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────────────┘
```

### 10.3 选项信息规范

每个选项必须直接写清：
- **代价**：扣金币 / 扣水晶 HP / 删卡 / 随机失去某资源
- **收益**：得卡 / 升卡 / 删卡机会 / 恢复 / 特殊增益
- **确定性**：
  - 完全确定 → 直接写结果
  - 含随机结果 → 明确写 `随机获得 / 随机失去 / 从 X 中随机`

### 10.4 风险等级表达

| 风险级别 | 颜色 | 用法 |
|---------|------|------|
| 低风险 | 蓝绿 | 明确正收益或轻代价 |
| 中风险 | 金黄 | 有明显机会成本 |
| 高风险 | 红色 | 扣血 / 重大随机 / 可能破坏构筑 |

高风险选项应加：
- 红色边框
- 警示图标
- 简短风险关键词（如 `高风险` / `可能失去核心卡`）

### 10.5 结果结算页

秘境选项执行后，不要直接黑屏切走，必须进入**结果页**。

结果页至少显示：
- 本次选择
- 资源变化
- 卡池变化
- 升级变化
- 是否可继续前往下一关

示例：

```text
秘境结果
+ 获得：火球术
- 失去：100 水晶 HP
卡池变化：+1 新卡（火球术）
```

若秘境事件改变卡池：
- 结果页必须明确写出 `卡池变化：+1 新卡 / -1 删除 / 某卡升至 Lv.2`
- 必要时提供 `[查看卡池]`

### 10.6 按钮与流程

| 按钮 | 行为 |
|------|------|
| 选项按钮 | 执行本次秘境选择 |
| 离开 | 放弃事件，返回关间流程 |
| 查看卡池 | 在结果页中查看变化后的卡池 |
| 继续 | 关闭秘境，进入下一流程 |

### 10.7 秘境锚点建议

| 元素 | 锚点 | 尺寸 |
|------|------|------|
| 主面板 | `middle-center, offset(0,0)` | 1280 × 760 |
| 标题 | 面板内 `top-center, offset(0,44)` | 自适应 |
| 描述区 | 面板内 `top-center, offset(0,140)` | 1040 × 120 |
| 选项列表 | 面板内 `offset(0,320)` | 1040 × 360 |
| 结果页摘要 | 面板内 `offset(0,300)` | 900 × 220 |

## 12. Run 结算界面

### 12.1 胜利结算

```
全屏 1920×1080 模态：
