# UI / UX 设计

> 设计分辨率：1920×1080 横屏 | 锚点定位系统 | 卡牌驱动交互 | 单 Run 闭环无 meta

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **符号优于文字** | 场景中不显示单位名称，形状/动画/特效让玩家一眼识别 |
| **卡牌即交互锚** | 战斗中所有部署操作从手牌发起，手牌区是视觉重心 |
| **按需显示** | 选中实体时才弹出详情面板；HUD 只展示必要数字 |
| **拖拽优先** | 出卡部署用拖拽，给玩家"打出去"的物理感 |
| **关间正式化** | 关间节点（商店/秘境）使用全屏模态面板，与战斗 UI 隔离 |

### 2.1 区域划分

| 区域 | 内容 | 尺寸 | 位置 |

### 2.2 UI 死区规范

死区由 UI 区域实际位置动态计算（不硬编码）：

```typescript
const UI_DEAD_ZONES: UIDeadZone[] = [
  { region: 'top_hud',   hitTest: (x, y) => y < scaleY(HUD_HEIGHT) },
  { region: 'hand_area', hitTest: (x, y) => y > viewportH - scaleY(HAND_HEIGHT) },
  { region: 'tooltip_overlay', hitTest: (x, y) => SelectedInfoPanel.contains(x, y) },
];
```

死区优先级：模态覆盖层 > 手牌区 > 顶部HUD > 选中面板

## 4. 手牌系统

### 4.1 手牌区布局

```
       ┌───────────────────────────────────────────┐
       │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐          │
       │  │箭塔 │ │剑士✦│ │火球 │ │金矿 │          │  📚牌组:8/12
       │  │ 🏹  │ │ ⚔  │ │ 💥  │ │ ⛏  │          │  🗑弃牌:0
       │  │ ◇3  │ │ ◇2  │ │ ◇3  │ │ ◇4  │          │
       │  └─────┘ └─────┘ └─────┘ └─────┘          │
       └───────────────────────────────────────────┘
```

- 单卡尺寸：120×168，横向间距 16px
- 固定上限 4 张（打出立即从牌库补抽 1 张）

### 4.2 单卡视觉规范

```
┌────────────────┐
│ ╭────────────╮ │   ← 边框颜色 = 卡牌等级
│ │  单位图形  │ │       L1 = 🟢绿色 / L2 = 🔵蓝色 / L3 = 🟡金色
│ │            │ │
│ ╰────────────╯ │
│  箭塔 ✦✦       │   ← 名字 + 等级菱形（L1无/L2=✦/L3=✦✦）
│  ❤120  ⚔20    │   ← 关键属性
│  ◇3            │   ← 能量消耗
└────────────────┘
   120 × 168
```

| 等级 | 边框颜色 | 触发条件 |
|------|---------|---------|
| L1（基础） | 🟢 绿色 | 初始状态 |
| L2（进阶） | 🔵 蓝色 | 科技树路径 depth=2 节点已解锁 |
| L3（满级） | 🟡 金色 | 科技树路径 depth=3 节点已解锁 |

### 4.3 卡牌交互流程

| 操作 | 视觉反馈 |
|------|---------|
| 悬停 | 卡片上移 12px + 缩放 1.05× + 详情弹窗 |
| 拖出 | 卡片消失，光标跟随幽灵预览 |
| 合法目标 | 地图格绿色高亮 + 卡片轮廓变绿 |
| 非法目标 | 红色高亮 + 卡片轮廓变红 |
| 松开（合法） | 部署/释放，扣能量，卡进弃牌堆 |
| 松开（非法） | 取消，卡回手牌 |
| 能量不足 | 卡片暗淡 + 红色"能量不足"标识 |

### 4.4 牌组/弃牌堆指示器

```
📚 牌组: 8/12     ← 剩余可抽/总卡组
🗑 弃牌: 4        ← 弃牌数量
```

- 点击图标弹出全览面板（半透明）
- 牌组用尽时弃牌堆自动洗回，有动画反馈

### 4.5 跨波保留标识

`persistAcrossWaves=true` 的法术卡：
- 右上角显示 ✦（金色星）
- 波次结束时不进入弃牌堆

## 6. 响应式布局系统

### 6.1 核心架构

```
游戏世界（地图/实体）: 设计空间(1920×1080) → scaleFactor → 视口空间
UI 元素: 锚点系统 → 视口空间绝对定位
```

- 内部逻辑坐标系 1920×1080
- `scaleFactor = viewportH / 1080`（以高度为基准等比缩放）

### 6.2 锚点系统

```
┌───────────────┬───────────────┬───────────────┐
│  TOP_LEFT     │  TOP_CENTER   │  TOP_RIGHT    │
├───────────────┼───────────────┼───────────────┤
│  MIDDLE_LEFT  │ MIDDLE_CENTER │ MIDDLE_RIGHT  │
├───────────────┼───────────────┼───────────────┤
│  BOTTOM_LEFT  │ BOTTOM_CENTER │ BOTTOM_RIGHT  │
└───────────────┴───────────────┴───────────────┘
```

### 6.3 LayoutManager API

```typescript
class LayoutManager {
  static viewportW: number;
  static viewportH: number;
  static scale: number;        // viewportH / 1080
  static anchor(config: AnchorConfig): { x: number; y: number };
  static scaleX(designPx: number): number;
  static scaleY(designPx: number): number;
}
```

### 6.4 主要锚点配置

| 元素 | 锚点 | 尺寸 |
|------|------|------|
| HUD 背景条 | `top-left, offset(0,0)` | fullWidth × 36 |
| 手牌区背景 | `bottom-center, offset(0,-130)` | 800 × 180 |
| 牌堆图标 | `bottom-right, offset(-200,-160)` | 50 × 70 |

### 8.1 触发与特例

- **触发时机**：关 N（N∈1..7）战斗胜利（最后一波清空+水晶存活）

### 8.2 战绩字段

| 字段 | 来源 |
|------|------|
| 通关用时 | RunManager 本关计时，`mm:ss` |
| 本关击杀 | UnitSystem 本关累计 |
| 最大单波击杀 | UnitSystem 单波峰值 |
| 出卡次数 | HandSystem 本关累计 |
| 水晶损失 | `crystalHpStart - crystalHpEnd`，负数显红 |
| 金币奖励 | RunManager `addGold()` 本关净增量 |

### 8.3 三选项行为

| 选项 | 边框色 | 行为 |
|------|--------|------|
| 🏪 商店 | 金黄 | 进入商店子面板 |
| 🌀 秘境 | 紫蓝 | 进入秘境事件 |
| ⏭ 跳过 | 灰白 | 零代价直入下关 |

**跳过**：无资源变化，立即 `RunManager.advanceToNextLevel()`

### 8.4 子面板关闭返回流程

| 子面板 | 关闭触发 | 关闭后 |
|--------|---------|--------|
| 商店 | 点"离开商店"或 ESC | 关闭 → 节点推进 → 返回路线图 |
| 秘境 | 选择任一选项后 | 同上 |
| 跳过 | 点"直接前进" | 跳过子面板直接推进 |

**节点推进动画**（1000ms）：
1. 面板淡出 200ms
2. 旧节点切换到✓ 300ms
3. 路径段金色实线 同步
4. 新节点亮起 300ms
5. 主按钮更新 200ms
