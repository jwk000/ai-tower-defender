# UI / UX 设计

> 设计分辨率：1920×1080 横屏 | 锚点定位系统 | 卡牌驱动交互 | 单 Run 闭环无 meta
> 当前界面规则版本：**v3.6（2026-05-19）**

---

## 1. 设计原则

| 原则 | 说明 |
|------|------|
| **符号优于文字** | 场景中不显示单位名称，形状/动画/特效让玩家一眼识别 |
| **卡牌即交互锚** | 战斗中所有部署操作从手牌发起，手牌区是视觉重心 |
| **按需显示** | 选中实体时才弹出详情面板；HUD 只展示必要数字 |
| **拖拽优先** | 出卡部署用拖拽，给玩家"打出去"的物理感 |
| **关间正式化** | 关间节点（商店/秘境/过关奖励）使用全屏模态面板，与战斗 UI 隔离 |
| **抽卡是主动决策** | UI 必须明确展示"可抽 / 不可抽 / 本次是否还能重抽"，避免误解为自动补牌 |

### 2.2 UI 死区规范

死区由 UI 区域实际位置动态计算（不硬编码）：

```typescript
const UI_DEAD_ZONES: UIDeadZone[] = [
  { region: 'top_hud',   hitTest: (x, y) => y < scaleY(HUD_HEIGHT) },
  { region: 'hand_area', hitTest: (x, y) => y > viewportH - scaleY(HAND_HEIGHT) },
  { region: 'draw_area', hitTest: (x, y) => DrawPanel.contains(x, y) },
  { region: 'tooltip_overlay', hitTest: (x, y) => SelectedInfoPanel.contains(x, y) },
];
```

死区优先级：模态覆盖层 > 手牌区 > 抽卡操作区 > 顶部HUD > 选中面板

## 4. 手牌与抽卡系统

### 4.1 手牌区布局

```
       ┌──────────────────────────────────────────────────────────────┐
       │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐      [抽卡]               │
       │  │箭塔 │ │剑士 │ │火球 │ │金矿 │      可重抽 1 次          │
       │  │ 🏹  │ │ ⚔  │ │ 💥  │ │ ⛏  │      或：手牌已满 / 冷却中   │
       │  │ ◇3  │ │ ◇2  │ │ ◇3  │ │ ◇4  │                            │
       │  └─────┘ └─────┘ └─────┘ └─────┘                            │
       └──────────────────────────────────────────────────────────────┘
```

- 单卡尺寸：120×168，横向间距 16px
- 固定上限 4 张
- **不再存在“打出立即补牌”**
- 手牌区右侧常驻展示当前抽卡状态

### 4.2 单卡视觉规范

```
┌────────────────┐
│ ╭────────────╮ │   ← 边框颜色 = 当前卡牌等级
│ │  单位图形  │ │       L1 = 🟢绿色 / L2 = 🔵蓝色 / L3 = 🟡金色
│ │            │ │
│ ╰────────────╯ │
│  箭塔  Lv.2    │
│  核心输出卡    │   ← 可选短标签 / 关键词
│  ◇3            │   ← 水晶能量消耗
└────────────────┘
   120 × 168
```

| 等级 | 边框颜色 | 触发条件 |
|------|---------|---------|
| L1（基础） | 🟢 绿色 | 入手默认 |
| L2（进阶） | 🔵 蓝色 | 关间花金币升级后 |
| L3（满级） | 🟡 金色 | 若该卡支持 Lv.3，则关间升满后 |

### 4.3 抽卡交互流程（v3.6）

| 阶段 | 操作 | UI 反馈 |
|------|------|---------|
| 可抽 | 距上次抽卡已满 5 秒，且手牌未满 | `[抽卡]` 亮起，边框发光 |
| 第一次抽取 | 点击抽卡按钮 | 抽出 1 张并弹出 `[保留] [丢弃并重抽]` |
| 重抽机会 | 玩家选择丢弃 | 原卡飞入弃牌动画区，立刻补出第 2 张 |
| 第二次抽取 | 重抽结果出现 | 显示 `必须保留` 标签，不再出现丢弃按钮 |
| 不可抽 | 手牌已满或 CD 未到 | 按钮置灰，并显示 `手牌已满` / `3s后可抽` |

### 4.4 出卡交互流程

| 操作 | 视觉反馈 |
|------|---------|
| 悬停 | 卡片上移 12px + 缩放 1.05× + 详情弹窗 |
| 拖出 | 卡片消失，光标跟随幽灵预览 |
| 合法目标 | 地图格绿色高亮 + 卡片轮廓变绿 |
| 非法目标 | 红色高亮 + 卡片轮廓变红 |
| 松开（合法） | 部署/释放，扣水晶能量，卡离开手牌 |
| 松开（非法） | 取消，卡回手牌 |
| 能量不足 | 卡片暗淡 + 红色`能量不足`标识 |

### 4.5 卡池状态区

```
卡池：12 张
已删卡：2 次
下次删卡：80G
```

- 战斗内仅作只读提示，不提供删卡操作
- 让玩家记得：当前抽卡质量来自关间构筑结果

## 6. 响应式布局系统

### 6.1 核心架构

```
游戏世界（地图/实体）: 设计空间(1920×1080) → scaleFactor → 视口空间
UI 元素: 锚点系统 → 视口空间绝对定位
```

- 内部逻辑坐标系 1920×1080
- `scaleFactor = viewportH / 1080`（以高度为基准等比缩放）

### 6.2 锚点系统

```
┌───────────────┬───────────────┬───────────────┐
│  TOP_LEFT     │  TOP_CENTER   │  TOP_RIGHT    │
├───────────────┼───────────────┼───────────────┤
│  MIDDLE_LEFT  │ MIDDLE_CENTER │ MIDDLE_RIGHT  │
├───────────────┼───────────────┼───────────────┤
│  BOTTOM_LEFT  │ BOTTOM_CENTER │ BOTTOM_RIGHT  │
└───────────────┴───────────────┴───────────────┘
```

### 6.3 LayoutManager API

```typescript
class LayoutManager {
  static viewportW: number;
  static viewportH: number;
  static scale: number;        // viewportH / 1080
  static anchor(config: AnchorConfig): { x: number; y: number };
  static scaleX(designPx: number): number;
  static scaleY(designPx: number): number;
}
```

## 7. 关间界面

### 7.1 关间结构

v3.6 关间成长来源统一为：
- **过关奖励**
- **商店**
- **秘境**

因此 UI 不再表达旧版"单一三选一路线图节点"，而是表达为：

```
关卡结算
  ↓
过关奖励面板
  ↓
商店 / 秘境（若触发）
  ↓
进入下一关
```

### 7.2 商店界面目标

商店必须突出三种核心操作：
- **买新卡**
- **升级已有卡**
- **删除卡**（显示本次费用与下次翻倍后的费用）

推荐信息结构：
- 左侧：当前卡池与等级分布
- 中间：可购买新卡
- 右侧：升级区 / 删除区 / 刷新区

### 7.3 秘境界面目标

秘境必须突出：
- 风险与收益并列展示
- 奖励可能为金币、卡、删卡机会、治疗、代价交换
- 若结果会影响卡池质量，要在结算时明确提示

## 8. UI 不变式（v3.6）

1. 手牌区永远只展示 **最多 4 张**。
2. 抽卡按钮必须明确区分 **可抽 / CD中 / 满手不可抽**。
3. 第一次抽卡后必须给出 **保留 / 丢弃并重抽** 明确操作。
4. 第二次抽卡结果必须显著标识 **不能丢弃**。
5. HUD 必须持续展示 **水晶能量当前值、上限、恢复节奏**。
6. 关间界面必须把 **买卡 / 升卡 / 删卡** 作为核心成长操作呈现。
7. 删除卡 UI 必须展示 **当前费用** 与 **下次费用翻倍** 的预期。
8. 不允许 UI 继续暗示旧版“自动补牌”“人口主导节奏”“Crystal 决定卡等级上限”等已废弃机制。

| 终战 | 王座 W23 | 圆形+外圈光环+皇冠 | 赤金 |

| 选项 | 边框色 | 行为 |

| 🏪 商店 | 金黄 | 进入商店子面板 |
| 🌀 秘境 | 紫蓝 | 进入秘境事件 |
| ⏭ 跳过 | 灰白 | 零代价直入下关 |

### 8.4 子面板关闭返回流程

| 子面板 | 关闭触发 | 关闭后 |
|--------|---------|--------|
| 商店 | 点"离开商店"或 ESC | 关闭 → 节点推进 → 返回路线图 |
| 秘境 | 选择任一选项后 | 同上 |
| 跳过 | 点"直接前进" | 跳过子面板直接推进 |

**节点推进动画**（1000ms）：
1. 面板淡出 200ms
2. 旧节点切换到✓ 300ms
3. 路径段金色实线 同步
4. 新节点亮起 300ms
5. 主按钮更新 200ms
