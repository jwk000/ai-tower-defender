## 1.1 低空单位与防空对策总览

低空敌人统一为 `layer: LowAir`，不会被炮塔、导弹塔、地刺、捕兽夹、焦油坑等地面爆炸/控制稳定命中。关卡出现 LowAir 敌人时，卡池必须提供至少一种可攻击 LowAir 的单位。

可攻击 LowAir 的稳定对策为：箭塔、弩塔、激光塔、蝙蝠塔、电塔、冰塔、火塔、毒塔、弓手、法师。导弹塔虽然是全图战略塔，但设计定位为地面爆炸，不处理低空敌人。

| 关卡 | LowAir敌人 | 出现波次 | 初始卡池中的LowAir对策 | 后续/精英卡池补充 | 设计结论 |
|------|------------|----------|--------------------------|-------------------|----------|
| 1 绿野仙踪 | 无 | - | 箭塔、弓手、法师 | 弩塔等更多地面/低空混合解法 | 无低空压力，教学不引入防空负担 |
| 2 沙漠虫潮 | 吸血蝗虫 | 3-5波 | 弓手 | 电塔、弩塔、箭塔 | 第3波开始教学低空，弓手是初始兜底，电塔/弩塔是成长解 |
| 3 黑暗古堡 | 吸血蝙蝠 | 1-7波 | 箭塔、激光塔、蝙蝠塔 | 弩塔、电塔、弓手 | 本关从第1波即有低空压力，初始卡池提供三种防空塔 |
| 4 末日废土 | 无人机、飞机 | 1-8波 | 电塔、弓手 | 导弹塔不算防空；可通过前置永久池获得箭塔/弩塔/激光塔/蝙蝠塔 | 低空密度最高，必须优先部署电塔/弓手或继承的对空塔 |
| 5 深渊裂隙 | 吸血蝗虫、吸血蝙蝠、无人机、飞机 | 2-12波 | 全部已解锁卡池随机，需保证抽牌保底至少1张LowAir对策 | 全部卡池 | 终局混编默认包含低空压力，抽牌系统必须保底防空 |

## 1.2 关卡卡池继承规则

关卡卡池采用单向累积规则：前面关卡在 `cardPool` 或 `draftPool` 中出现过的卡牌，后续所有关卡的 `cardPool` 与 `draftPool` 都必须继续包含。新关卡只允许在继承前置卡牌的基础上追加本关主题卡，不允许移除旧卡。

该规则保证玩家已经见过、学习过的卡牌不会在后续关卡从初始抽牌或精英3选1中消失；如果因为玩法重构需要禁用某张卡，必须先更新设计文档并明确写入 `banPool` 的产品原因。

## 1.3 MDA数值重设计计划（2026-06-15）

### 设计目标

- **Mechanics**：以配置真实运行规则为预算基准。每波除 YAML 敌人外，`WaveSystem` 会额外生成1只随机精英；精英按 `HP×2、ATK×1.5、金币×2`，并继续叠加阶段难度倍率 `0.8 / 1.0 / 1.3 / 1.5`。资源预算按最坏掉落下限计算：普通敌人击杀金币取 `reward.gold × 0.8` 向下取整，精英击杀金币取 `reward.gold × 2 × 0.8` 向下取整，再叠加波次奖励和初始金币。
- **Dynamics**：每关卡池必须覆盖稳定清线、控速、高血量处理和多路并发处理。关卡不依赖抽到唯一答案；玩家在资源分配较差但仍部署基础防线的情况下，应能通过出售、升级和补塔纠正局面。
- **Aesthetics**：保留每关主题体验。第1关是教学安全垫，第2关是虫潮密度，第3关是双路优先级，第4关是三路合流调度，第5关是终局混编压迫；难度来自节奏和取舍，不来自资源死局。

### 修改原则

- 先修正敌人引用一致性：敌方黑暗牧师使用 `dark_priest`，避免与玩家祭司 `priest` 混淆。
- 波次奖励必须从 YAML 透传到运行时 `WaveConfig.reward`，否则文档和配置中的资源预算不会真实生效。
- 每个 Boss 前至少提供一次明显补强窗口；终局压力可以高，但不能在多个高威胁单位同时压满所有出生口。
- 每个 Boss 波只能配置 1 个最终 Boss；随机精英候选必须排除 Boss 敌人，避免同一关同时出现“巨型史莱姆”和“精英巨型史莱姆”等双 Boss 体验。
- 后续数值变更需同步更新本节预算口径，并用配置测试校验所有关卡的最坏资源下限。
- 低 HP 高数量敌人的金币不再按“每只都值得一张小额奖励”定价，哥布林/快兵/自爆虫/自爆甲虫等刷钱点下调基础金币；精英怪因为 HP 翻倍且触发 3 选 1，实际金币按基础金币 ×2 发放，保证击杀精英不会变成经济亏损。
- 关卡卡池调整遵守单向累积规则，只追加攻防互补卡。每关初始卡池至少覆盖前排承伤、稳定输出和一种控制/清线手段；精英卡池继续作为补齐短板和高阶解法入口。

### 本轮数值策略

| 关卡 | 初始金币 | 主要调整 | 体验目标 |
|------|----------|----------|----------|
| 1 | 170 | 降低第3-5波重单位数量，提高第4-5波奖励 | 玩家只拿到基础卡也能完成教学关，不因大象/巨人堆血卡死 |
| 2 | 230 | 削减第4-6波蝗虫/自爆虫峰值，提高虫潮中后段奖励 | 保留虫潮密度，但给火塔/毒塔/炮塔成型空间 |
| 3 | 290 | `priest` 改为 `dark_priest`，降低第6-7波巫师/牧师叠压，提高双路奖励 | 玩家能通过优先击杀牧师和分路布防解决压力 |
| 4 | 350 | 加入刺客卡，降低坦克/飞机/巨型机器人多路并发，提高三路图奖励 | 压力来自多路调度，而不是高威胁单位同时封死三路 |
| 5 | 430 | `priest` 改为 `dark_priest`，降低后半段坦克/牧师/无人机锁点，提高 Boss 前资源窗口 | 终局仍高压，但每个 Boss 前都有补强和纠错空间 |

### 2026-06-17 金币与卡池微调

| 项目 | 调整 | 目的 |
|------|------|------|
| 低血刷钱敌人 | 哥布林 5→4、快兵 8→5、自爆虫 12→8、自爆甲虫 15→10 | 避免低 HP 或自爆型敌人成为经济效率异常高的刷钱目标 |
| 精英掉落 | 随机精英实际掉落 = 基础金币 ×2 后再应用随机方差 | 与精英 HP×2 的战斗成本对齐，提高击杀精英的正反馈 |
| 第1关卡池 | 初始卡池加入剑士，精英卡池加入火塔 | 教学关补足近战输出和 AOE/DOT 选择，降低抽到纯防守组件时的卡手感 |
| 第2关卡池 | 初始卡池加入盾卫，精英卡池加入弩塔 | 虫潮关补足前排承伤和高护甲穿透处理 |
| 第3关卡池 | 初始卡池加入弓手、剑士，精英卡池补入弩塔 | 双路低空与高血量敌人并存时，给玩家更多稳定防空和物理输出 |
| 第4关卡池 | 初始卡池加入盾卫，精英卡池加入刺客、淘金热 | 三路合流关补足前排调度和经济补强 |
| 第5关卡池 | 初始/精英卡池加入剑士 | 终局全卡池补齐近战 AOE 输出，保持攻防体系完整 |

## 9. 过关界面（Victory Screen）

### 9.1 概述

每关通关后展示一个 **单层 3 阶段动画序列** 的胜利全屏覆盖层。所有视觉、音频、故事参数由关卡 YAML 的 `victory` 配置节驱动，配合默认值减少重复配置。

过关后记录 `LevelCompletionState` 到存档，与关卡选择界面联动展示通关摘要。

| 阶段 | 时长 | 内容 |
|------|------|------|
| 阶段1：章节句 | 0~1.2s | 全屏只保留主题背景图，棋盘和战斗实体清理；展示 `victory.story.title` 作为章节结尾句，不再显示“恭喜你通关xxxx” |
| 阶段2：仪式特效 | 1.2~2.2s | 胜利模式显示彩带/粒子与星级评定，失败模式使用暗红光效；BGM 切换到胜利/失败曲 |
| 阶段3：剧情与统计 | 2.2s+ | 剧情正文以打字机方式浮现，半透明结算面板展示波次、敌人、使用塔、兵数量、水晶血量、总得分、总伤害；就绪后显示“点击继续/点击返回” |

### 9.2 YAML 配置结构

每个关卡 YAML 新增 `victory` 配置节：

```yaml
victory:
  story:
    title: "标题"
    paragraphs: ["段落1", "段落2", ...]
    summary: "关卡选择界面展示的摘要"
    showFullStoryOnlyFirst: true   # 仅首次通关展示完整故事

  background:
    filter: "rain_to_sunny"       # 预定义背景滤镜
    gradient: { top: "#...", mid: "#...", bottom: "#..." }
    particles:
      - { type: "sparkle", color: "#...", density: 20, speed: 0.5 }

  confetti:
    count: 80                     # 总粒子数
    burst: "top_fall"             # top_fall | bottom_rise | explosion_center | both_sides
    colors: [["#...", "#..."]]   # 彩带颜色组
    shapes: { ribbon: 0.5, petal: 0.35, sparkle: 0.15 }
    duration: 3.0
    spread: 0.8                   # 水平扩散范围 (0-1)

  audio:
    bgm: "bgm/victory_meadow.ogg"
    sfx: "sfx/victory_meadow.ogg"

  typography:
    titleColor: ["#...", "#..."]  # 通关标题渐变
    panelBg: "rgba(..., 0.85)"    # 结算 UI 背景固定按 85% 不透明绘制
    panelBorder: "#..."
    storyColor: "#..."
    accentColor: "#..."
```

### 9.3 背景滤镜类型

| filter 值 | 说明 | 使用关卡 |
|-----------|------|---------|
| `rain_to_sunny` | 灰度80% → 明亮翠绿渐变，太阳光晕亮起 | 第1关 |
| `heat_dissipate` | 棕黄暖色 → 热浪消散，虫族残骸散落 | 第2关 |
| `dawn_break` | 灰暗冷色 → 裂缝白光破开，蝙蝠飞出消散 | 第3关 |
| `eye_close` | 暗红焦黑 → 红色"眼睛夕阳"闭合，绿色光点亮起 | 第4关 |
| `rift_seal` | 深紫 → 裂隙愈合关闭，紫火熄灭，晶簇变蓝白 | 第5关 |
| `gray_tint` | 默认：灰度80%定格，无特殊转场 | 备用 |

### 9.4 发射方式

| burst 值 | 说明 | 隐喻 |
|----------|------|------|
| `top_fall` | 粒子从顶部飘落 | 花瓣、雪花、雨过 |
| `bottom_rise` | 粒子从底部喷射上升 | 热风、能量上升 |
| `explosion_center` | 从屏幕中心向外爆炸 | 爆炸、绽放 |
| `both_sides` | 两侧同时向中间喷射 | 终局庆典 |

### 9.5 各关胜利配置概要

| 关卡 | story.title | background.filter | confetti.burst | confetti 主色 | audio.bgm | summary |
|------|------------|-------------------|----------------|-------------|-----------|---------|
| 1 绿野仙踪 | 雨后初晴 | `rain_to_sunny` | `top_fall` | 绿+金+白 | `victory_meadow` | 草原已平，沙漠风沙扬起。 |
| 2 沙漠虫潮 | 虫群溃散 | `heat_dissipate` | `bottom_rise` | 琥珀+金 | `victory_desert` | 沙漠归于死寂，古堡阴影浮现。 |
| 3 黑暗古堡 | 黎明破晓 | `dawn_break` | `top_fall` | 银白+深蓝+紫 | `victory_castle` | 古堡黎明破晓，废土红光隐现。 |
| 4 末日废土 | 余烬新生 | `eye_close` | `explosion_center` | 橙+灰+铁锈红 | `victory_waste` | 巨眼闭合，焦土中绿芽新生。 |
| 5 深渊裂隙 | 裂隙封印 | `rift_seal` | `both_sides` | 紫金+白金(×2密度) | `victory_abyss` | 裂隙愈合，黎明升起。全部通关。 |

失败结算复用同一覆盖层结构，差异仅为暗红色调、不显示星级和彩带、按钮文案为“点击返回”。失败剧情来自 `victory.defeatStory`，缺失时使用默认失败文本。

### 9.6 配置默认值

YAML 中未指定的字段使用以下默认值，减少配置冗余：

```typescript
const DEFAULT_VICTORY_CONFIG = {
  background: {
    filter: 'gray_tint',
    gradient: { top: '#000000', mid: '#1a1a2e', bottom: '#0a0a14' },
    particles: [],
  },
  confetti: {
    count: 60,
    burst: 'top_fall',
    colors: [['#ffd700', '#ff8c00', '#ffffff']],
    shapes: { ribbon: 0.6, sparkle: 0.4 },
    duration: 3.0,
    spread: 0.7,
  },
  typography: {
    titleColor: ['#ffd700', '#ffffff'],
    panelBg: 'rgba(10, 14, 22, 0.85)',
    panelBorder: '#ffd700',
    storyColor: '#ffffff',
    accentColor: '#ffd700',
  },
};
```

### 9.7 存档联动（LevelCompletionState）

每关通关后存储：

| 字段 | 类型 | 说明 |
|------|------|------|
| `completed` | boolean | 是否已通关 |
| `firstClearedAt` | number? | 首次通关时间戳 |
| `timesCleared` | number | 通关次数 |
| `bestStars` | number | 最高星级 (1-3) |
| `summary` | string | 来自 victory.story.summary |

### 9.8 关卡选择界面联动

- **已通关关卡卡片**：显示星星 + summary 文本 + 通关次数角标
- **刚通关返回**：对应卡片 2 秒金色脉冲动画，提示"刚完成的关卡"
- **视觉衔接**：过关界面的 `background.gradient.bottom` 颜色与关卡选择主题 `gradient.top` 匹配，500ms crossfade 过渡
- **重复通关**：若 `showFullStoryOnlyFirst=true` 且 `timesCleared>0`，跳过阶段3故事叙述，直接显示"继续"按钮
