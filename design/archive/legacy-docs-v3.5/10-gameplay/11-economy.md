# 经济系统（v3.5 单 Run 闭环版）

> ✅ **v3.5 形态级第 1 轮已完成（2026-05-18）**：本文档已从 v3.0.0「三资源（能量/金币/技能点SP）」**整文重写**为 v4.0.0「二资源（能量/金币）+ 人口系统 + 科技树」。
>
> **v3.5 二资源轴**：**能量 E（关内单波）/ 金币 G（本 Run 关间）**。所有资源在 Run 结束时清零，单 Run 闭环无 meta 积累。
>
> **核心变更（v3.4 → v3.5）**：
> - ~~技能点 SP（本 Run 技能树）~~ → **彻底废弃**，整套资源轴删除
> - ~~金币 → SP 单向兑换~~ → **彻底废弃**（v3.4 §3.5 整节删除）
> - ~~SP 驱动的关后技能树节点解锁~~ → **替换为金币驱动的科技树（Crystal 升级 / 卡牌解锁 / 卡牌升级）**
> - ✅ 新增 **人口系统**（关内持续占用资源，详见 §3）
> - ✅ 金币新增用途：**Crystal 升级**（Lv1→Lv3）+ **科技树卡牌升级**（关间节点）

> **数值真理源**：本文档不持有数值，所有数值（能量消耗 / 金币掉落 / 商店价格 / Crystal 升级费用 / 卡牌升级费用 / 人口上限）以 [50-mda](../50-data-numerical/50-mda.md) 为准。本文档仅说明**规则与结构**。

## 1. 二种资源（v3.5 认知边界）

```
┌─────────────────────────────────────────────────────────┐
│  能量 E       ←→  关内单波范围      → 出卡部署            │
│   ↕ 被动恢复 + 波次爆发                                   │
│  金币 G       ←→  本 Run 关间范围    → 商店购买 / 秘境消费 │
│               ↓ Crystal 升级（Lv1→Lv3）                  │
│               ↓ 科技树节点：卡牌解锁 / 卡牌升级            │
│                                                          │
│  Run 结束：二资源全部清零，无 meta 沉淀                   │
└─────────────────────────────────────────────────────────┘
```

> **注意**：人口不在此表中。人口是**关内持续占用型资源**（非消耗型货币），见 §3。

| 资源 | 作用范围 | 持久性 | 来源 | 消耗用途 |
|------|---------|--------|------|---------|
| **能量 E** | 关内单波 | 每波开始部分爆发恢复；**Run 结束清零** | 每波开始 +2E 爆发 + 战斗中 +1E/4s 被动恢复（不超过上限 10） | 战斗中出卡部署消耗 |
| **金币 G** | 本 Run 关间 | 跨波次跨关卡累积；**Run 结束清零** | 击杀敌人 + 波次结算 + 秘境事件 | 关间商店 / 秘境 / Crystal 升级 / 科技树节点 |

**关键设计原则（v3.5 不变式）**：
1. 二种资源**作用范围严格分层**：能量仅用于关内出卡，金币仅用于关间消费
2. **跨货币锁定**：v3.5 不存在任何跨货币兑换通道（v3.4 的 金币→SP 已废弃）
3. **Run 结束清零**：二资源全部归零，无 meta 沉淀通道（10-roguelike-loop §11 INV-01）

## 2. 能量 E（关内出卡资源）

### 2.1 规则

| 维度 | 规则 |
|------|------|
| **上限** | **固定 10**（v3.5 保持 v3.4 锁定，不可永久升级） |
| **每波开始爆发恢复** | **+2 E**（瞬时补充，不超过上限；给玩家波次开始的「爆发感」）|
| **战斗中被动恢复** | **+1 E / 4 秒**（持续滴灌，支撑全波段决策；数值权威见 [50-mda §12.1](../50-data-numerical/50-mda.md#121-能量上限与恢复)）|
| **跨波保留** | 上一波末剩余能量保留进入下一波，受上限约束 |
| **额外恢复** | 部分特殊卡 / 秘境事件可额外回能；商店「能量瓶」道具 40 G/瓶；塔主动技能不消耗能量（见 [22a](../20-units/22a-skill-tree-tower.md)）|

> **v3.4 → v3.5 能量机制不变**：能量系统设计在 v3.4 节奏优化后已稳定，v3.5 无改动。典型一波（约 30s）总能量收入 ≈ 2E（爆发）+ 7E（被动）= 约 9E，玩家每波可出 2-4 张卡。

### 2.2 能量消耗示例（具体数值见 50-mda §12）

| 卡类型 | 典型能量消耗 |
|--------|------------|
| 基础建筑卡（箭塔 L1） | 3 E |
| 高级建筑卡（炮塔 L1 / 激光塔 L1） | 5-6 E |
| 基础单位卡（剑士 / 弓手） | 2-3 E |
| 高级单位卡（祭司 / 召唤师） | 4-5 E |
| 法术卡（火球术 / 减速） | 2-3 E |
| AoE 法术（箭雨） | 4 E |
| 终极召唤 / 全屏冰冻 | 8-10 E |

> **法术卡不占用人口**（能量消耗后效果即生效消散，不产生持久单位）。见 §3.2。

### 2.3 能量与塔的解耦

> 能量塔（重命名「能量水晶」）作为可选建筑卡保留在卡池中，效果是「部署后额外提升被动恢复速率」或临时上限提升（见 50-mda §12），但**不是默认拥有的资源生产链**。

## 3. 人口系统（v3.5 新增）

> **人口是关内「持续占用型」约束资源，不是货币。** 玩家无法通过金币购买更多人口，只能通过升级 Crystal 提升人口上限（关间行为）。

### 3.1 人口规则

| 维度 | 规则 |
|------|------|
| **人口上限** | 由 Crystal 等级决定（数值见 50-mda §NEW-CRYSTAL）；Run 开始固定，关内不可扩大 |
| **人口占用** | 出卡部署持久单位（塔/士兵/陷阱/生产建筑）时，占用 = `CardConfig.energyCost`（与能量消耗值相同） |
| **释放时机** | 单位死亡（或主动回收）后，立即释放对应人口 |
| **法术卡** | **不占用人口**（效果即时消散，不产生持久场上单位） |
| **HUD 显示** | 关内顶部：`人口 占用/上限`（如 `人口 7/12`） |

### 3.2 人口与能量的双重约束

打出一张单位卡需同时满足两个条件：

```
① 当前能量 ≥ CardConfig.energyCost       （能量校验）
② 当前人口占用 + CardConfig.energyCost ≤ 人口上限    （人口校验）
```

两个条件缺一不可。法术卡只做能量校验（跳过人口校验）。

**设计意图**：
- 人口上限防止玩家通过大量堆高费单位无上限强化，使 Crystal 升级成为关键策略决策
- energyCost 复用为人口占用数，简化配置，避免新增 `populationCost` 字段（法术卡除外）

### 3.3 人口上限与 Crystal 等级（关间升级）

| Crystal 等级 | 人口上限 | 卡牌等级上限 | 升级费用（金币） |
|------------|---------|------------|--------------|
| Lv.1（初始） | TBD | 1 | — |
| Lv.2 | TBD | 2 | 见 50-mda §NEW-CRYSTAL |
| Lv.3 | TBD | 3 | 见 50-mda §NEW-CRYSTAL |

> **数值占位 TBD**：具体人口上限数值和升级费用在 50-mda §NEW-CRYSTAL 中定义（待 50-mda 更新）。Crystal 升级为关间行为（商店或专属节点），见 [48-shop-redesign](../40-presentation/48-shop-redesign-v34.md)。

## 4. 金币 G（本 Run 关间货币）

### 4.1 流入途径

| 来源 | 触发时机 | 数值（50-mda §13） |
|------|----------|---------------------|
| **击杀敌人** | 敌人死亡瞬时 | 按敌人 `bounty` 配置（普通 3-8 / 精英 15-25 / Boss 80-150） |
| **波次完成** | 全部敌人死亡 | 基础 20 G + 5 G × 关卡序号 |
| **关卡通过** | 关底 Boss 击败 | 100 G + 50 G × 关卡序号 |
| **秘境事件** | 部分事件选项 | 视事件 0-200 G |
| **生产建筑卡（如金矿）** | 部署后每秒产出 | 见 50-mda §13.2 |

### 4.2 流出途径（仅在关间节点消耗）

| 节点 | 消耗用途 | 价格范围 |
|------|---------|---------|
| **商店单位卡槽**（48 §3 前 4 槽） | 购买单位卡（加入关外卡池） | 30-240 G |
| **商店功能卡槽**（48 §3 后 4 槽） | 能量瓶 / 水晶修复 / 手牌刷新 | 0-80 G |
| **Crystal 升级** | Crystal Lv1→Lv2 / Lv2→Lv3（提升人口上限 + 卡牌等级上限） | 见 50-mda §NEW-CRYSTAL |
| **科技树节点** | 卡牌解锁（进入卡池）/ 卡牌升级（等级提升） | 见 50-mda §NEW-CRYSTAL |
| **秘境** | 部分事件需消耗金币换取效果 | 20-150 G |

> **v3.5 变更**：删除 v3.4 的「金币→SP 兑换」槽（§3.5 整节废弃）。商店第 5 槽改为 Crystal 升级或其他功能，见 [48-shop-redesign](../40-presentation/48-shop-redesign-v34.md)。

### 4.3 关内不能花金币的设计意图

**战斗中金币只攒不花**（v3.5 保持 v3.4 原则）：
- 给玩家明确的「攒钱节奏感」（关卡内只关注出卡 + 击杀）
- 关间一次性决策让「用金币干什么」成为正式策略时刻
- 简化关内 UI（顶部 HUD 仅显示能量 + HP + 人口，金币以累加方式呈现）

### 4.4 初始金币

```
Run 起始金币：0 G（10-roguelike-loop §11 INV-01）
```

### 4.5 Run 结束金币处理

```
Run 结束（胜利 / 失败 / 退出）：
  剩余金币 G → 直接清零，无任何 meta 沉淀
```

## 5. 资源流向总图（v3.5）

```
[关内战斗]
   │ 出卡消耗（能量 + 人口双重校验）
   ▼ ────────────── 能量 E（被动+1E/4s，每波爆发+2E，上限10）
   │                ↕ 人口 占用/上限（单位死亡释放）
   │ 击杀掉落 + 波次结算
   ▼
   ╔═══════════════╗
   ║   金币 G      ║ ← 本 Run 累积（关结束跨关保留，Run 结束清零）
   ╚═══════════════╝
   │
   ├──→ [商店 单位卡槽] ─→ 买卡 → 进关外卡池（自动解锁科技树节点）
   │
   ├──→ [商店 功能卡槽] ─→ 能量瓶 / 水晶修复 / 手牌刷新
   │
   ├──→ [Crystal 升级] ─→ Lv1→Lv2→Lv3（人口上限↑ + 卡牌等级上限↑）
   │
   ├──→ [科技树节点] ─→ 卡牌解锁 / 卡牌升级（受 Crystal 等级上限约束）
   │
   └──→ [秘境] ─→ 部分事件消耗 0-150 G

═══ Run 结束（胜利/失败/退出）═══
   能量 + 金币 + 水晶 HP + 卡池 + 科技树状态 + 人口占用 → **全部清零**
   下一 Run 从初始状态重开（金币 0 / Crystal Lv1 / 人口初始上限 / 起手 4 张基础卡）
```

## 6. 经济节奏（v3.5 设计意图）

| 阶段 | 玩家行为 | 金币预算 | 关键决策 |
|------|---------|---------|---------|
| **关 1-2（开局）** | 金币紧张，仅够商店买 1-2 件 | 80-150 G/关 | 先买卡扩池还是升 Crystal 提人口？ |
| **关 3-5（构筑）** | 金币累计中等，关间选择有重量 | 200-400 G/关 | 确立流派方向，开始把核心卡升级到 Lv2 |
| **关 6-8（成型）** | 金币充裕，Crystal 有望升 Lv3 | 500-1000 G/关 | 卡组成型，Crystal Lv3 解锁高等级牌上限 |
| **终战** | 资源接近峰值，全力一搏 | 累计 1000-2000 G | 终战的张力最大化，无后顾之忧（Run 结束清零） |

> **v3.4 → v3.5 节奏差异**：v3.4 玩家在关间需分配金币和 SP 两条线，v3.5 合并为单一金币决策轴，降低认知负担，但通过 Crystal 升级 vs 卡牌升级 vs 买卡三向竞争维持策略深度。

## 7. 关内回收机制（v3.5 保留 v3.4 原则）

| 维度 | v3.5 规则 |
|------|----------|
| **建造时扣** | 仅能量 E（金币不参与关内消耗） |
| **回收返还** | 能量 E（不返还金币） |
| **回收公式** | `返还能量 = 0.5 × 该卡能量消耗 × (currentHP / maxHP)` |
| **建造冷却** | 3 秒内不可回收 |
| **战斗保护** | 受伤/攻击中 2 秒内不可回收 |
| **回收后人口** | 回收建筑 / 单位后，立即释放对应人口 |

详细公式见 [50-mda §12.3](../50-data-numerical/50-mda.md#123-能量回收公式)。

## 8. 已删除机制（历史）

> 以下机制已完全删除，不再适用：

**v3.0 删除（v1.0 → v3.0 转型）**：
- ~~旧的金币被动生产链（默认拥有金矿/能量塔）~~ → 改为可选卡牌
- ~~旧的人口资源 P~~ → v3.0 改为「场上单位上限 8」固定上限（10-roguelike-loop §11 v3.4-INV-10）
- ~~战斗中花金币建造/升级/释放技能~~ → 全部改用能量

**v3.4 新增删除（v3.0/v3.1 → v3.4 转型）**：
- ~~火花碎片资源轴~~ → v3.4 替换为本 Run 技能点 SP（已废弃）
- ~~Run 结束金币 1:1 转碎片~~ → Run 结束金币直接清零
- ~~主菜单卡池永久解锁机制~~ → 所有卡开局即解锁
- ~~主菜单永久升级~~ → 全部固定锁定
- ~~v3.1 关外永久科技树~~ → v3.4 改为本 Run 内技能树（SP 支付）

**v3.5 新增删除（v3.4 → v3.5 转型）**：
- ~~技能点 SP 资源轴（本 Run 技能树）~~ → **整套废弃**，替换为金币驱动的科技树 + Crystal 升级
- ~~金币 → SP 单向兑换（50 G / 1 SP）~~ → **废弃**（v3.5 无跨货币转换通道）
- ~~关卡通关 SP 奖励（关 N×2 SP）~~ → **废弃**（通关奖励改为关后 3 选 1 节点，见 10-roguelike-loop §3）
- ~~秘境事件 SP 奖励~~ → **废弃**（秘境奖励改为金币或卡牌奖励）
- ~~SP 驱动的技能树节点解锁（3/6/10/15 SP）~~ → **替换**为金币驱动的科技树节点升级
- ~~路径互斥单装备机制~~ → **废弃**，改为线性等级成长（Lv1/Lv2/Lv3）
- ~~场上单位上限 8（INV-10）~~ → **替换**为人口上限机制（INV-11）

> **注意**：v3.5 引入的「人口」与 v3.0 删除的「旧的人口资源 P」是不同机制。
> - 旧的人口资源 P：v1.0 时期的独立资源货币，v3.0 时以「场上单位上限 8」替代后废弃。
> - v3.5 人口系统：基于 `energyCost` 的持续占用约束，与 Crystal 等级上限挂钩，与「旧的 P」语义不同。
