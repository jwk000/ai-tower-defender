# Run 长征循环 ─ v3.5 单 Run 闭环（v3.0.0）

> **本文档是 v3.5 形态的 Run 长征权威。** 在 v2.1.0（v3.4 节奏优化版）基础上进行 v3.5 形态级重构。
>
> **v3.5 形态级变更**（详见 [v3.5-MAJOR-MIGRATION](../v3.5-MAJOR-MIGRATION.md)）：
> 1. **二资源轴**：能量 / 金币（~~技能点 SP 彻底废弃~~）。
> 2. **卡池结构重定义**：关外每种卡只有 1 张；关内从全卡池无限随机抽取（可重复）。
> 3. **手牌机制重设计**：打出 1 张立即抽 1 张补充，始终保持 4 张手牌（~~定时抽牌废弃~~）。
> 4. **人口上限系统**：场上单位持续占用人口（= 卡能量费用），Crystal 等级决定上限（替代固定 8 上限）。
> 5. **科技树替代技能树**：Crystal 升级（金币）提升人口上限 + 卡牌等级上限；卡牌通过金币在科技树升级。
> 6. **关卡内临时升级（instanceLevel）彻底废弃**：精炼术法术卡 + instanceLevel 机制全部删除。

## 1. v3.5 玩法定位

| 维度 | v3.5 形态 |
|---|---|
| 核心循环 | 战斗（手牌 4 张实时补充）→ 关后 3 选 1（商店 / 秘境 / 跳过）→ 下一关 → 终战 |
| 单 Run 时长 | 25-40 分钟，连闯 8 关 + 终战（共 9 关 = W1-W23） |
| 失败代价 | **从第 1 关重开，无任何 meta 留存** |
| 胜利收益 | **本 Run 闭环**，仅记录"通关次数 + 历届最远 + Hero Score"展示型数据 |
| 单 Run 内策略 | 手牌 4 张怎么打、能量 + 人口怎么分配、Crystal 何时升级、商店买什么卡/升什么卡、秘境冒不冒险 |
| 长线驱动 | 历届最远关卡、Hero Score、Run 通关次数（仅展示，不解锁内容） |

### 1.1 三大支柱（v3.5）

1. **手牌随机性** — 从全卡池无限随机抽取；打出即补，始终 4 张，每次打出后的下一张是什么充满期待
2. **关后 3 选 1 抉择** — 商店（稳健：买卡/升卡/升 Crystal）/ 秘境（冒险）/ 跳过（保血）三向博弈
3. **血量继承张力** — 水晶 HP 全程继承，前期失血付出后期代价

### 1.2 受影响的设计哲学（v3.5 锁定）

- ✅ **保留**：一切皆单位、一切皆卡牌、配置驱动、符号优于文字
- ✅ **保留**：关卡随机性（天气、装饰物、波次组成、秘境事件池）
- ✅ **保留**：单 Run 闭环哲学 — 死亡无回报，胜利无 meta 收益
- 🆕 **v3.5 新增**：**手牌即时补充** — 打出立即抽，始终 4 张，无定时抽牌
- 🆕 **v3.5 新增**：**人口上限系统** — 场上单位持续占用人口，Crystal 升级扩展上限
- 🆕 **v3.5 新增**：**科技树** — 金币解锁卡牌 + 升级卡牌，Crystal 升级控制等级上限
- 🛑 **v3.5 删除**：技能点 SP 资源轴、instanceLevel 临时升级、精炼术法术卡
- 🛑 **v3.5 删除**：卡组 / 弃牌堆结构（改为无限随机卡池 + 固定 4 张手牌）

## 2. 卡牌系统

### 2.1 卡牌定义（v3.5）

卡牌是「单位/法术配置 + 打出方式 + (本 Run) 科技树等级状态」的封装。

```typescript
interface CardConfig {
  id: string;
  name: string;
  type: 'unit' | 'spell' | 'trap' | 'production' | 'shop_item';   // shop_item 详见 48 §3
  energyCost: number;          // 出卡消耗能量；同时也是场上单位的人口占用数
  rarity: 'common' | 'rare' | 'epic' | 'legendary';

  // type=unit 时引用现有单位配置
  unitConfigId?: string;

  // type=spell 时引用法术效果
  spellEffectId?: string;
  spellSubtype?: 'damage' | 'control' | 'buff' | 'utility';
  // ↑ v3.5 删除 'buff_instance' 子类型（instanceLevel 机制废弃）

  // 部署/释放约束
  placement: {
    targetType: 'tile' | 'enemy' | 'ally' | 'area' | 'global';
    range?: 'cursor' | 'path' | 'anywhere' | 'left-half' | 'right-half';
  };

  // v3.5 新增：卡牌科技树等级（关外持有，Run 内生效）
  // cardLevel 由 RunManager.cardLevels[cardId] 决定，配置中不直接持有
}
```

> **v3.5 变更**：
> - 删除 `cardLevel` 持久化字段 → 改由 `RunManager.cardLevels` 记录本 Run 卡牌等级
> - 删除 `buff_instance` 法术子类型（instanceLevel 已废弃）
> - `energyCost` 同时作为场上单位的人口占用数（详见 §2.5 人口系统）

### 2.2 卡牌四大分类（v3.5 战斗内）

| 大类 | 子类 | 说明 |
|---|---|---|
| **建筑卡（Tower）** | 防御塔（单体 / AoE / 控制）、生产塔（金矿 / 能量塔）、辅助塔（治疗 / 增益） |
| **单位卡（Soldier）** | 近战（盾兵 / 剑士）、远程（弓手 / 法师）、空中（飞兵）、特殊（祭司 / 工程师 / 召唤师） |
| **法术卡（Spell）** | 伤害 / 控制 / Buff / Debuff / Utility（~~v3.5 删除 buff_instance：instanceLevel 废弃~~） |
| **陷阱卡（Trap）** | 详见 [27-traps-spells-scene §3](../20-units/27-traps-spells-scene.md#3-陷阱) |

> **第 5 类 shop_item 不进战斗手牌**，仅在商店内呈现，详见 [48 §3](../40-presentation/48-shop-redesign-v34.md#3-cardtype-拓展shop_item-为第-5-类)。

### 2.3 卡池结构与手牌机制（v3.5 重设计）

#### 关外卡池（v3.5 新定义）

```
关外卡池 = 玩家本 Run 已解锁的所有卡，每种卡只有 1 张
  - 获得卡（商店购买 / 关卡奖励）→ 进入卡池（唯一，不重复）
  - 每张卡有对应的卡牌等级（1-3 级，受 Crystal 等级上限约束）
  - 卡池大小 = 本 Run 已解锁卡的种类数（非数量）
```

#### 关内手牌与抽卡（v3.5 新机制）

```
关卡开始（第 1 波开始前）：
  从关外卡池随机抽 4 张，充满手牌（4 张）

战斗中（持续进行）：
  1. 能量每 4 秒自动 +1（被动恢复，不超过上限 10）——全波段匀速流入
  2. 每波开始爆发恢复 +2 能量（不超过上限 10）
  3. 玩家从手牌打出 1 张卡：
     a. 消耗能量 E（= 卡的 energyCost）
     b. 打出后：【立即】从全卡池随机抽 1 张补入手牌（保持 4 张满手牌）
     c. 单位卡/建筑卡：生成实体，场上占用人口（详见 §2.5）
     d. 法术卡：立即执行效果
  4. 打出的卡【不进弃牌堆】，下次依然可能被抽到（无限随机，可重复）
  5. **塔主动技能**：玩家可点击场上的塔触发其主动技能（无能量消耗，各塔独立 CD，见 [22a](../20-units/22a-skill-tree-tower.md)）

波次结束：
  1. 结算金币（击杀奖励 + 波次完成基础奖励）
  2. **速清奖励**：清场后 10s 内玩家可点击「提前开始」按钮触发下一波，获得 +5G 速清奖励
  3. 若玩家未手动触发，等待 10 秒后自动进入下一波
  4. 手牌 4 张保留至下一波（波次结束不清空手牌）

关卡通过（最后一波 + 关底 Boss 被击败）：
  1. 发放金币关卡通关奖励（数值由 50-mda §18 持有）
  2. 弹出「3 选 1 单位卡抽卡」面板（详见 §2.6）
  3. 进入「关后 3 选 1 节点」（详见 §3，UI 在 47 §4）
```

> **v3.5 删除**：卡组（deck）/ 弃牌堆（discard）结构、每波定时抽牌（每 8 秒自动抽）、每波开始抽 1 张。

### 2.4 手牌规则（v3.5）

- **手牌固定 4 张**，始终满手牌（打出 1 张立即补 1 张）
- **抽卡来源**：从全关外卡池（本 Run 已解锁的所有种类卡）随机抽取，同一张卡可能被多次抽到
- **不存在"手牌已满"的情况**：由于每次打出立即补充，手牌始终为 4 张
- **无弃牌堆**：不再有卡组消耗 / 弃牌堆洗牌机制

> **设计意图**：玩家始终有 4 张牌可选，不会遭遇"手牌为空"或"卡组耗尽"的困境；每次出牌后的随机补充带来期待感；高稀有度卡的出现频率由池中卡的数量与随机权重决定。

### 2.5 人口系统（v3.5 新增）

人口（Population）是关内持续占用型资源，用于限制场上单位数量。

```
人口上限：由本 Run 的 Crystal 等级决定（详见 §5.3 / [22-skill-tree-overview §CRYSTAL]）

人口占用规则：
  - 打出一张卡，该卡创建的场上单位占用【人口数 = 卡的 energyCost】
  - 例：能量 3 的剑士卡打出后 → 剑士占用 3 人口；剑士死亡 / 被回收 → 3 人口释放
  - 打出卡前检查：当前已占用人口 + 该卡 energyCost ≤ 人口上限；否则无法打出

适用范围：
  - 单位卡（Soldier）：占用 energyCost 人口
  - 建筑卡（Tower）：占用 energyCost 人口（塔也占人口）
  - 法术卡（Spell）：不占人口（打出立即消耗，不创建持久单位）
  - 陷阱卡（Trap）：占用 energyCost 人口（陷阱部署在地格，破坏后释放）

HUD 显示：
  人口：{当前占用} / {人口上限}（如 "人口：7 / 12"）
```

> **v3.5 取代**：原 `v3.4-INV-10`「场上单位固定上限 8」不变式废弃，改由人口上限动态决定。

### 2.6 卡牌升级（v3.5 科技树）

> **v3.5 锁定**：本 Run 内"卡牌升级"由**科技树（金币消耗）**承担，受 Crystal 当前等级约束上限。

- **升级入口**：关后路线图「卡池」按钮 → 显示本 Run 已解锁的卡牌 → 花金币升级（详见 [22-skill-tree-overview](../20-units/22-skill-tree-overview.md)）
- **等级限制**：卡牌等级上限 = Crystal 当前等级（Crystal Lv.1 → 最高 1 级；Lv.2 → 最高 2 级；Lv.3 → 最高 3 级）
- **等级效果**：对应原 22a-22e 各单位技能树 depth=N 节点效果（等级 N = 原 depth=N）
- **关内生效**：卡牌以其当前等级打出，等级效果立即应用

> **v3.5 删除**：~~技能点 SP 消耗~~；~~instanceLevel（关内精炼术法术卡叠加）~~；~~22a-22e 路径互斥单装备机制~~（改为线性等级成长）

### 2.6 关卡通关 3 选 1 单位卡抽卡（v3.5 保留）

每关通关时（W1-W22，终战 W23 例外），在「关后 3 选 1 节点」面板**之前**插入一次单位卡抽卡：

```
关卡 N 通过！选 1 张单位卡加入卡池：

┌──────────┬──────────┬──────────┐
│  弓手    │  炮塔    │  剑士    │
│  Common  │  Rare    │  Common  │
│  E:3     │  E:5     │  E:2     │
└──────────┴──────────┴──────────┘
[ 跳过（不加卡） ]
```

- 池子：当前关外卡池中**未持有**的所有单位卡（按稀有度权重抽 3 张）
- 选中：该卡**加入关外卡池**（自动解锁对应科技树节点），关内手牌即可抽到
- 跳过：不加卡，直接进入关后 3 选 1
- **终战 W23 不触发本机制**（终战是 Run 终点，无关后节点）
- 数值由 [50-mda §18](../50-data-numerical/50-mda.md) 持有（稀有度权重、特殊池规则等）

### 2.7 卡牌死亡 / 移除

- 单位卡战场被击杀 → 该单位实体消失，释放其占用的**人口**（人口回到可用池）
- 建筑卡被摧毁 → 同上，释放人口
- 法术卡打出后 → 立即消耗，不占人口，不影响关外卡池（下次仍可能被抽到）
- 关外卡池中的卡**不会减少**（单次战斗失败不删卡）

### 3.3 秘境节点（v3.4 重设 · 决策 E1c + E2b）

秘境是**怪物火车风格的事件房间**，进入时显示一段情境文字 + 2-3 个选择，每个选择有明确收益和潜在代价。

**v3.4 秘境奖励改造**（决策 E1c）：旧 v1.0 的「碎片奖励」改为**混合奖励池**（金币+技能点+单位卡按权重）。具体奖励组合由 [27-traps-spells-scene §5 秘境事件池](../20-units/27-traps-spells-scene.md#5-秘境事件池) 持有，本文档不重复定义。

**v3.4 新增 30% 高风险事件**（决策 E2b）：在秘境事件池中，约 30% 的事件可能造成**伤害性代价**（损 HP、删卡、减能量上限等），制造单 Run 闭环下的抉择张力。

```
秘境面板示例（v3.4）：

┌──────────────────────────────────────────────────┐
│  🌀 秘境 — 古老的祭坛                                  │
│                                                  │
│  你发现一座血色祭坛，散发着不祥的气息。               │
│  上面刻着三种契约：                                   │
├──────────────────────────────────────────────────┤
│  [A] 献祭 30% 水晶 HP                                │
│      → 获得 5 技能点 + 1 张 Legendary 单位卡         │
│                                                  │
│  [B] 献祭 1 张随机卡                                  │
│      → 水晶恢复 40 HP + 80 金币                      │
│                                                  │
│  [C] 拒绝契约离开                                      │
│      → 不消耗任何资源直接进入下一关                   │
└──────────────────────────────────────────────────┘
```

**v3.4 秘境特性**：
- 每次秘境**至少有一个零成本选项**（如「拒绝契约离开」），玩家可无代价退出
- 事件结果立即生效，无倒计时
- 已遇过的事件本 Run 不重复
- **30% 高风险事件**用色调区分（红边框），强警示玩家代价
- 奖励池替换详细对照表在 [27-traps-spells-scene §5](../20-units/27-traps-spells-scene.md#5-秘境事件池)（第 4 轮一般文档清理时同步重写）

### 3.4 跳过节点（v3.4 新增 · 决策 A2 锁定）

「跳过」分支零成本：不进商店、不触秘境，直接进入下一关。

| 维度 | 跳过节点 |
|---|---|
| 触发 | 关后 3 选 1 选「跳过」 |
| 资源消耗 | 无 |
| 资源获得 | 无（仅水晶 HP / 卡组 / 金币 / 技能点保持现状） |
| 设计意图 | 保血流派、节奏控制、屯资源对终战 |

> 跳过是 v3.3 引入的稳健选项，v3.4 保留并明确"完全零代价"原则。详见 [47 §4](../40-presentation/47-level-map-ui.md#4-胜利结算--3-选-1-一屏面板)。

### 3.5 商店 vs 秘境 vs 跳过的设计意图

| 维度 | 商店 | 秘境 | 跳过 |
|---|---|---|---|
| 不确定性 | 低（明码标价） | 高（事件随机 + 30% 高风险） | 零（确定无变化） |
| 收益期望 | 稳定中等 | 方差大 | 零 |

## 4. 能量系统（关内出卡资源）

### 4.1 能量定义（v3.4 沿用 v1.0）

**Energy (E)** 是关内唯一的出卡消耗资源，与金币、技能点完全分离。

```
能量规则：
  - 上限：10（固定）
  - 初始（每关第 1 波）：5
  - 每波开始爆发恢复：+2（不超过上限）
  - 战斗中被动恢复：+1 E / 4 秒（持续滴灌，支撑全波段决策，见 11-economy §2.1）
  - 跨波保留：上一波末剩余能量保留，受上限约束
  - 部分特殊卡 / 秘境效果可额外恢复能量
  - 塔主动技能不消耗能量（各塔独立 CD，见 22a）
```

> **设计意图**：被动恢复让玩家每 4-5 秒就有一次「要不要出牌」的决策时机，彻底消除「能量耗尽后干等」的节奏死区（详见 [11-economy §2.1](../10-gameplay/11-economy.md#21-规则)）。

> **v3.4 删除**：v1.0 的「永久升级扩展能量上限至 12」机制（随 meta 系统一并废弃）。

## 5. 关卡结构

### 5.1 一次 Run 的完整结构（决策 B3a 保留 8 关 + 终战）

```
[主菜单 → 开始新 Run]
  → [开局抽 12 张卡组]
  → [关卡 1: 平原]（8-10 波 = W1-W3 区段）
      → 水晶 HP 从初始值 100 开始（决策 B1a 跨关继承）
  → [单位卡 3 选 1 抽卡]
  → [关后 3 选 1：商店 / 秘境 / 跳过]
  → [关卡 2: 森林]
  → [单位卡 3 选 1]
  → [关后 3 选 1]
  → ...
  → [关卡 8: 王城]
  → [单位卡 3 选 1]
  → [关后 3 选 1]
  → [关卡 9: 终战 Boss 关 = W23]（单波超长，混合所有敌人类型，无关后节点）
      → 击败 → Run 胜利（详见 §6）
      → 失败或中途任何一关水晶 HP=0 → Run 失败（详见 §6）
```

**Run 总波次**：约 70-90 波 / Run（每关 8-10 波，外加终战）。预计时长 25-40 分钟。

> **关卡映射**：W1-W3 = 关 1，W4-W6 = 关 2，... ，W22 = 关 8，W23 = 终战。详见 [47 §1 世界地图](../40-presentation/47-level-map-ui.md#1-世界地图总览)。

### 5.2 水晶机制（沿用 v1.0 · 决策 B1a 锁定跨关 HP 继承）

> 水晶（Crystal）是玩家方核心防御实体，位于地图左侧路径终点。

#### 5.2.1 定位与外观

- **类型**：特殊单位（`unit_kind=crystal`），固定占用 1 个 tile，不可被部署 / 移除 / 移动
- **视觉**：红色水晶造型，悬浮 + 辉光呼吸。详见 [42-art-assets §13.13](../40-presentation/42-art-assets.md) 与 [44-visual-effects §10](../40-presentation/44-visual-effects.md)

#### 5.2.2 无敌规则（沿用 v1.0）

水晶**免疫一切伤害源**——任何敌人的普攻、技能、AoE、毒、点燃、自爆、Boss 必杀、反弹伤害等**都无法直接扣除水晶 HP**。

- 唯一 HP 消耗途径是**主动秒杀**（见 5.2.3）
- 正向操作仍生效：「治疗水晶」法术卡、商店「水晶 HP 恢复」道具（详见 [48 §5.2](../40-presentation/48-shop-redesign-v34.md#5-后-4-槽随机池4-种功能卡)）、秘境治疗事件等按既定规则结算
- 秘境「献祭 X% 水晶 HP」等主动负向操作仍生效（属于玩家自愿付出的代价）
- 取消旧规则：**敌人抵达终点不再扣 HP**，由「秒杀消耗 1 HP」彻底替代

#### 5.2.3 秒杀攻击（核心战斗机制 · 沿用 v1.0）

水晶把进入其攻击范围的敌人立即秒杀（无视目标的 HP、护盾、无敌、霸体、闪避），每次秒杀**消耗水晶 1 HP**。

| 字段 | 语义 | 默认值 | 真理源 |
|---|---|---|---|
| `crystal.attackRange` | 攻击范围（地格数） | 1 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.attackCooldown` | 攻击间隔（秒） | 0.5 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.hpCostPerKill` | 每次秒杀消耗的 HP | 1 | [50-mda](../50-data-numerical/50-mda.md) |
| `crystal.maxKillsPerTick` | 单次冷却内最大击杀数 | 1 | [50-mda](../50-data-numerical/50-mda.md) |

**目标选择**：范围内多敌人时按「最近距离 > 进入范围时间」排序，每次秒杀 1 个。

**Boss 例外**：终战 Boss（`abyss_lord`）携带 `immune_to_crystal_kill = true`，水晶**无法秒杀 Boss**。Boss 进入水晶范围也不消耗 HP，但 Boss 仍受水晶的无敌保护（无敌规则继续生效）。

#### 5.2.4 水晶 HP 变化（决策 B1a 跨关继承）

```
初始 HP：100（固定，v3.4 删除了 v1.0 的「baseHpBonus 永久升级」机制）

水晶 HP 变化：
  - 秒杀 1 个敌人：-1 HP
  - 商店「水晶 HP 恢复」：+N HP（详见 48 §5.2）
  - 秘境治疗事件：+N HP
  - 「治疗水晶」法术卡（关内）：+N HP
  - 秘境献祭 / 高风险事件：按事件配置 -N HP
  - 关卡通过：HP 跨关继承（决策 B1a），不自动恢复
  - 关后 3 选 1 选「跳过」：HP 跨关继承（决策 B1a）

水晶 HP = 0 → 水晶碎裂动画 → Run 失败
```

> **决策 B1a 锁定**：水晶 HP 全程跨关继承，每关失血在后期付出代价；恢复路径仅限商店购买 / 秘境治疗 / 法术卡。

## 11. 不变式（v3.5 锁定）

> 所有不变式必须在代码 / 配置 / 文档同步遵守。违反任意一条 = BUG。

1. **单 Run 闭环**：Run 结束（无论胜负）所有金币 / 卡池 / Crystal 等级 / 卡牌升级状态 / 水晶 HP 全部清零，无 meta 入账
2. ~~**技能点本 Run 临时**~~ — **v3.5 删除**：技能点 SP 系统整套废弃
3. **死亡从第 1 关重开**：不允许从当前关 / 任一已通关重开
4. **水晶 HP 跨关继承**：每关结束不自动恢复 HP
5. **能量上限固定 10**：无永久升级机制
6. **手牌固定 4 张**：打出 1 张立即补 1 张，始终保持 4 张
7. ~~**所有卡开局即解锁**~~ — **v3.5 修改**：改为「**获得卡 = 自动解锁关外卡池 + 科技树对应节点**」；关内只能抽到已解锁的卡
8. ~~**金币 ↔ 技能点单向**~~ — **v3.5 删除**（技能点废弃）；金币唯一关外消耗：商店购卡 / 升卡 / Crystal 升级
9. **关后 3 选 1 三向互斥**：商店 / 秘境 / 跳过选 1 即进入下一关，不可回头
10. **终战 W23 无关后节点**：终战通关直接进 Run 结算，无 3 选 1，无单位卡抽卡
11. **人口持续占用**：场上单位占用 energyCost 数人口，死亡后释放；不可超过人口上限打出新卡
12. **instanceLevel 不存在**：关内任何机制不能触发 instanceLevel 类型升级（精炼术废弃）
13. **关外卡池每种卡唯一**：同种卡在关外卡池中只有一张（不重复）；关内抽卡可重复抽到同种卡
