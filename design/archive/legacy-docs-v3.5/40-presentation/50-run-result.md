# Run 结算界面设计文档

> Run 结算界面在 Run 胜利（击败终战 Boss）或 Run 失败（水晶 HP 降至 0 或 ESC 主动放弃）后弹出。
> 玩家在此界面查看本次 Run 战绩，然后选择返回主菜单或立即开始下一次新 Run。

## 1. 触发时机

| 触发条件 | 说明 |
|---|---|
| 终战 W23（关 9）胜利 | 击败最终 Boss，水晶存活 → 「Run 胜利」结算 |
| 水晶 HP 降至 0 | 任意关卡水晶血量耗尽 → 「Run 失败」结算 |
| 路线图 ESC 确认退出 Run | 玩家主动放弃，触发 `RunManager.abandonRun()` → 「Run 失败」结算（与正常失败相同面板） |

## 3. 字段定义

### 3.1 标题区

| 结果 | 标题文字 | 图标 | 主色调 |
|---|---|---|---|
| 胜利 | "Run 胜利！" | 🏆 金色皇冠 | 金色 (#ffd700) |
| 失败 | "Run 失败" | 💀 骷髅/碎裂水晶 | 暗红色 (#cc3333) |

### 3.2 战绩统计字段

| 字段 | 来源 | 显示规则 |
|---|---|---|
| 最远到达 | `RunManager.currentLevel`（胜利时为 9，失败时为实际关卡） | "关卡 N（主题名）"，如"关卡 4（雷暴荒原）" |
| 通关时长 / 本次 Run 时长 | `runEndMs - runStartMs` | `mm:ss` 格式 |
| 总击杀 | `runStats.enemiesKilled` | 整数 |
| 最大单波击杀 | `runStats.maxSingleWaveKills`（待实现统计） | 整数 |
| 水晶剩余血量 | `RunManager.crystalHp / RunManager.crystalHpMax` | "N/1000"，胜利时彩色，失败时红色 |
| 共花费金币 | `runStats.goldSpent`（待实现统计） | 整数，金色 |
| 流派标签 | `RunManager.archetypeTag`（本 Run 流派识别，详见 61-save-system §5） | 文字标签，如"法术爆发流"（仅胜利时显示） |
| 关键技能树 | `RunManager.skillTreeState`（已解锁节点列表中权重最高的 2-3 个） | 列举节点名（仅胜利时显示） |

### 3.3 资源清零声明

```
⚪ 本次 Run 所有资源已清零
   金币 / 技能点 / 水晶 HP / 卡组 / 技能树节点
   下一次 Run 从初始状态出发（仅胜利时显示"下一次 Run 从初始状态出发"）
```

- **胜利**：显示完整 3 行（含"下一次 Run 从初始状态出发"）
- **失败**：仅显示前 2 行

## 4. 按钮行为

### 4.1 「返回主菜单」

| 维度 | 规则 |
|---|---|
| 行为 | 关闭结算界面 → 清除 `ongoingRun` 存档 → 显示主菜单 |
| 主菜单状态 | 「继续游戏」按钮灰显（Run 已结束，无进行中存档） |
| 界面跳转 | 结算界面 → 主菜单 |

### 4.2 「立即开始新征程」

| 维度 | 规则 |
|---|---|
| 行为 | 关闭结算界面 → 清除 `ongoingRun` 存档 → `RunManager.initRun()` → 显示关卡路线图（关 1 高亮）|
| 界面跳转 | 结算界面 → 关卡路线图（**跳过主菜单**）|
| 设计意图 | 让"再来一把"的决策路径最短，降低流失 |

## 6. 交互规则

| 行为 | 规则 |
|---|---|
| ESC 键 | **无效**——强制玩家明确选择「返回主菜单」或「立即新征程」，避免误操作 |
| 右键 / 触摸外区点击 | **无效**——面板外区域不响应点击 |
| 点击时机 | 面板入场动画完成后（约 500ms）才响应点击，防误触 |

---

## 7. 入场动画

| 阶段 | 视觉 | 时长 |
|---|---|---|
| 1. 黑色蒙层淡入 | alpha 0 → 0.85 | 300ms |
| 2. 面板缩放弹入 | scale 0.8 → 1.0 + ease-out | 400ms |
| 3. 标题飘入 | y offset -20 → 0 + fade in | 300ms（与 2 同步）|
| 4. 战绩数字跳动（可选） | 数字从 0 计数至最终值 | 800ms（UI 实现时视效果决定）|
| 总计 | 约 700ms 后可交互 | — |

## 8. 存档处理时序

```
Run 结束（胜利/失败）
    │
    ▼
RunController → RunManager.endRun()
    │  · 计算最终统计数据
    │  · 更新 RunHistory（totalRuns / totalVictories / highestLevel…）
    │  · 清除 ongoingRun 存档
    │  · 将 RunHistory 写入 SaveData
    ▼
RunController → 显示结算界面（§2）
    │
    ├─ 点「返回主菜单」→ RunController.returnToMainMenu()
    │    · 显示主菜单（「继续游戏」灰显）
    │
    └─ 点「立即开始新征程」→ RunController.startNewRun()
         · RunManager.initRun()
         · 显示关卡路线图（关 1 高亮）
```
