## 1. 渲染架构

使用 PixiJS WebGL 渲染。场景通过多层 Container 堆叠构成：

```
Stage
 ├── BackgroundContainer（背景层）
 ├── BoardContainer（棋盘容器）
 │    ├── GroundLayer（地面瓦片）
 │    ├── ShadowLayer（阴影投影）
 │    ├── GridLayer（地格层单位）
 │    ├── GroundEntityLayer（地面层单位+装饰物）
 │    ├── LowAirEntityLayer（低空层单位+飞鸟）
 │    ├── SpaceEntityLayer（高空层，预留）
 │    ├── EffectLayer（粒子特效）
 │    └── WeatherLayer（天气粒子）
 ├── ScreenFXContainer（屏幕特效：震动/闪白/暗角）
 └── UIContainer（UI层：HUD/手牌区/面板/弹窗）
```

## 2. 层级系统

### 2.1 两层体系

层级系统分为**逻辑层级**和**表现层级**两个独立维度：

| 维度 | 逻辑层级 | 表现层级 |
|------|---------|---------|
| 所在域 | 棋盘空间高度 | 场景视觉远近 |
| 作用 | 决定攻击目标可达性 | 决定视觉遮挡关系 |
| 类比 | Z轴（上下） | Z-index（前后） |

### 2.2 逻辑层级（棋盘内5层）

```
Space（高空层）    — 最高，预留
LowAir（低空层）   — 飞行单位（蝗虫、蝙蝠、飞机、无人机）
Ground（地面层）   — 行走单位 + 塔 + 建筑 + 场景装饰物
Grid（地格层）     — 地面机关陷阱
Abyss（深渊层）    — 最低，预留
```

**层级交互规则**：

| 规则 | 说明 |
|------|------|
| 低层可攻击高层 | Ground层单位可以攻击LowAir层单位 |
| 高层可攻击低层 | LowAir层单位可以攻击Ground层单位 |
| Grid可攻击Ground | 地刺对路过地面单位造成伤害 |
| 同层可攻击 | 同层单位可以互攻 |
| 弹道穿透 | 弹道飞行不受层级影响，由发射方和目标方层级决定可否命中 |

### 2.3 棋盘渲染Z序（从后到前）

| Z | 层名 | 内容 |
|----|------|------|
| 0 | GroundLayer | 地图网格瓦片 + 主题色 |
| 1 | ShadowLayer | 实体地面投影（半透明椭圆） |
| 2 | Grid | 地格层单位渲染（地刺、捕兽夹、焦油坑、水坑） |
| 3 | Ground | 地面层单位（塔、士兵、敌人、巨石、风扇等）+ 装饰物（树、岩石等），按Y坐标排序 |
| 4 | LowAir | 低空单位（飞行敌人、蝙蝠、飞鸟），按Y坐标排序 |
| 5 | Space | 高空单位（预留） |
| 6 | EffectLayer | 粒子特效（爆炸/命中/建造/金币飞行） |
| 7 | WeatherLayer | 天气粒子（雨/雪/雾/夜） |

### 2.4 棋盘外部层

| 层 | 说明 |
|----|------|
| BackgroundLayer | 天空渐变 + 远景几何（z=-2） |
| ScreenFXLayer | 屏幕震动、闪白、暗角（z=8） |
| UILayer | HUD、手牌区、信息面板（z=9） |
| OverlayLayer | 暂停遮罩、3选1面板、结算面板（z=10） |

## 3. 天气系统

### 3.1 天气类型

| 天气 | ID | 所属关卡 | 说明 |
|------|-----|---------|------|
| 晴天 | Sunny | 第2关 | 默认天气，阳光普照 |
| 下雨 | Rain | 第1关 | 大雨滂沱 |
| 下雪 | Snow | （预留） | 暴雪严寒 |
| 下雾 | Fog | 第4关 | 浓雾弥漫，视野受限 |
| 夜晚 | Night | 第3、5关 | 夜幕降临 |

### 3.2 视觉特效

| 天气 | 特效 |
|------|------|
| 晴天 | 无额外特效，仅太阳光射线 |
| 下雨 | 顶部下落蓝色半透明水滴粒子（#4fc3f7），密度约80粒子/秒，大小3-6px |
| 下雪 | 飘落白色雪花粒子（#ffffff），密度约40粒子/秒，大小4-8px，带正弦漂移 |
| 下雾 | 全屏灰色半透明叠加（#9e9e9e, alpha 0.15）+ 2-4个白色雾团（#ffffff, alpha 0.08）随机缓慢漂移，大小200-400px |
| 夜晚 | 全屏深蓝色半透明叠加（#0d1b2a, alpha 0.3）+ 路径格上散布微光点（#ffe082, alpha 0.4），密度约1点/3格 |

### 3.3 天气对游戏的影响

天气仅影响视觉氛围，**不影响战斗数值**。

> 设计意图：避免玩家需要记忆天气对数值的影响，降低认知负担。天气纯粹用于烘托关卡氛围。

### 4.2 静态装饰物

每个障碍物占用地格，由2-5个几何原语组合而成：

| 关卡 | 装饰物类型 |
|------|-----------|
| 绿野仙踪 | 树、灌木、花丛、巨树、蘑菇、藤蔓 |
| 沙漠虫潮 | 岩石、仙人掌、枯骨 |
| 黑暗古堡 | 残柱、断壁、藤蔓、枯树、蜘蛛网 |
| 末日废土 | 石柱、碎石堆、倒塌的房子、报废汽车、易拉罐、垃圾 |
| 深渊裂隙 | 漂浮石、紫色火焰、晶簇、深渊之眼 |

#### 装饰物几何方案示例

| 装饰物 | 组合方案 |
|--------|---------|
| 树 | 棕色矩形（树干4×10）+ 深绿三角形（树冠14px）+ 浅绿三角形（高光10px，偏移-2px） |
| 灌木 | 2-3个不同大小的深绿/浅绿圆形，随机偏移±3px叠加 |
| 花丛 | 粉色小圆（花心4px）+ 3-5个更小粉圆（花瓣，围绕花心120°均匀分布） |
| 岩石 | 深灰菱形（主体12px）+ 浅灰小三角（高光5px，偏移左上） |
| 仙人掌 | 绿色矩形（主干）+ 2个绿色小矩形（侧臂，45°角） |
| 晶簇 | 浅蓝菱形（主体11px）+ 白色小菱形（核心高光5px）+ 2个浅蓝小三角（侧晶） |

### 4.3 动态环境生物

**飞鸟**（所有关卡共通）：

| 属性 | 值 |
|------|-----|
| 形状 | 简化V形（2条短线相交于中心）+ 小三角头 |
| 大小 | 8-14px翼展 |
| 颜色 | `#ffffff`（白色） |
| 数量 | 2-5只/场景 |
| 运动 | 沿贝塞尔曲线飞行，翅膀正弦拍动（3-5Hz） |
| 行为 | 循环路径：左→右→飞出后从另一侧重入→随机改变高度 |
| 层级 | LowAir层（z=4） |

翅膀拍动周期（1周期=2次拍动/秒）：
- 0-25%：翅膀展开↑，身体+2px
- 25-50%：翅膀收起↓，身体-2px
- 50-75%：翅膀展开↑，身体+2px
- 75-100%：翅膀收起↓，身体-2px
- 翅膀角度：30°（收起）→120°（展开）正弦循环

### 4.4 全屏环境特效

| 类别 | 说明 | 实现方式 |
|------|------|---------|
| 静态装饰物增强 | 微动动画（树叶摇摆、火焰跳动） | 每帧更新少量顶点 |
| 动态环境生物 | 飞鸟、地面小动物 | 独立动画实体 |
| 全屏环境 | 风线、阳光射线、云阴影漂移 | 全屏叠加半透明几何体 |
| 天气粒子增强 | 风、热浪等视觉粒子 | 扩展天气粒子层 |

## 5. 水晶视觉

水晶是玩家的核心防御目标，在地图左侧占据1格。

| 属性 | 值 |
|------|------|
| 形状 | 菱形复合几何体（2个菱形叠加，1主1辉光） |
| 主体色 | `#ff1744` 红色 |
| 辉光色 | `#ff5252`（外层，透明度0.4） |
| 大小 | 主体32px，辉光48px |
| 动画 | 上下浮动（±4px，正弦循环2秒周期） |
| 辉光呼吸 | 透明度从0.2→0.5→0.2循环（3秒周期） |
| 低血量(<30%) | 闪烁红色+呼吸频率加快至1.5秒周期 |
| 秒杀特效 | 水晶向目标方向发出红色光束+目标位置红色爆裂粒子 |
| 碎裂特效 | 水晶分裂为8-12个碎片→向上飘散→透明度渐变到0→完全消失（1.5秒） |

│                                                     │
│                                                     │
│                ┌───────────────┐                    │
│                │               │                    │
│                │   战场区域     │                    │

│  └───┘  └───┘  └───┘  └───┘                       │

### 6.2 顶部HUD

| 元素 | 位置 | 内容 |
|------|------|------|
| 关卡名 | 左上 | 当前关卡名称 |
| 波次进度 | 左上（关卡名下） | "波次 3/7" |
| 金币数 | 右上（左） | 💰图标 + 数值 |
| 水晶HP | 右上（右） | 💎图标 + 当前/最大HP，低血量红色闪烁 |

### 6.3 手牌区

| 属性 | 值 |
|------|------|
| 位置 | 场景底部居中，战场区域正下方 |
| 尺寸 | 与战场同宽（1344px），高140px |
| 卡牌尺寸 | 100×140px |
| 排列 | 水平居中，间距16px |
| 空槽 | 虚线边框灰色占位 |
| 悬停效果 | 卡牌上浮8px + 放大至110% + 发光边框 |

### 6.4 卡牌详情悬浮面板

悬浮手牌中某张卡时显示，尺寸240×320px：

| 行 | 内容 |
|----|------|
| 1 | 卡牌名称（大字） |
| 2 | 类型标签（单位卡/技能卡/奥术卡） + 稀有度 |
| 3 | 攻防血描述（定性："攻击力高/中等/低"） |
| 4 | 技能/机制说明（一句话） |
| 5 | 描述文字 |
| 6 | 额外信息（如"穿透""AOE""减速30%"） |

### 6.5 3选1抽卡面板

- 弹出时游戏暂停，半透明黑色遮罩覆盖全屏
- 面板居中，显示3张候选卡牌（等大并列）
- 每张卡悬停时放大显示详情
- 点击卡牌 = 选择，面板关闭
- 手牌满时额外显示"选择要替换的卡"提示

### 6.6 塔交互

| 操作 | UI反馈 |
|------|--------|
| 点击塔 | 塔高亮边框 + 弹出升级/出售按钮 |
| 升级按钮 | 显示升级费和下一级属性预览 |
| 出售按钮 | 显示回收金币数 |
| 拖卡悬停 | 可放置格绿色边框，不可放置格红色边框 |
| 范围预览 | 拖卡时显示半透明攻击范围圈 |

## 7. 动画系统

### 7.1 单位动画

| 动画类型 | 说明 |
|---------|------|
| 待机 | 微小浮动/呼吸（±2px，1.5-3s周期） |
| 攻击 | 短暂前摇（缩进→弹出）+ 弹道发射 |
| 受击 | 白色闪烁50ms + 微震 |
| 死亡 | 缩小+淡出（300ms），部分单位有特殊死亡动画 |
| 移动 | 匀速平移，无额外动画（简化） |
| 升级 | 金色脉冲光晕从中心扩散（500ms） |

### 7.2 弹道动画

| 弹道类型 | 表现 |
|---------|------|
| arrow | 直线飞行小三角（棕色），速度800px/s |
| beam | 持续激光线条（从塔到目标），颜色随塔类型 |
| missile | 抛物线飞行小圆（红色），速度400px/s，到达后爆炸 |
| aoe | 抛物线飞行大圆（黑色），速度300px/s，落地爆炸粒子 |
| chain | 闪电链（黄色锯齿线），每个弹跳节点闪烁50ms |
| instant | 无弹道，目标受击闪烁 |

### 7.3 粒子效果

| 效果 | 触发时机 | 表现 |
|------|---------|------|
| 命中 | 攻击命中目标 | 小火花粒子（3-5个，颜色随伤害类型） |
| 爆炸 | AOE/自爆 | 10-15个粒子向外扩散（300ms）+ 闪光 |
| 建造 | 部署单位 | 5-8个粒子从中心上升（200ms） |
| 金币 | 敌人死亡 | 金色小圆飞向HUD金币位置 |
| 升级 | 塔升级 | 金色光环扩散+上升粒子 |

## 8. 音效系统

### 8.1 音效分类

| 类别 | 示例 | 数量（预估） |
|------|------|------------|
| UI交互 | 点击、拖卡、选卡、面板打开/关闭 | ~10 |
| 战斗 | 箭矢、激光、爆炸、闪电、火焰 | ~15 |
| 单位 | 部署、死亡、升级 | ~8 |
| 敌人 | 各敌人死亡音效 | ~8 |
| 关卡 | 波次开始、关卡通过、关卡失败 | ~5 |
| BOSS | BOSS登场、技能、死亡 | ~6 |
| 天气 | 雨、雷（可选） | ~3 |
| **合计** | | **~55** |

### 8.2 音效播放规则

- 同类型音效节流：≥50ms间隔，防止播放器过载
- UI音效优先播放（响应感最重要）
- 战斗音效距离衰减（按屏幕位置模拟远近）
