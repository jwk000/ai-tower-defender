# Tower Defender — 游戏概述

> 当前版本：**v3.6（2026-05-19）**
> 定位：卡牌 Roguelike 长征塔防 — 怪物火车 × 月圆之夜 × 塔防融合体。
> 所有游戏数值以 [NUMBERS.md](./NUMBERS.md) 为唯一权威。

---

## 1. 一句话定位

**"把塔防做成卡牌 Roguelike 长征"**

每次游玩（称为一次 **Run**）从卡池出发，连续闯过 **8 关主题关卡（关 1–8，其中关 8 即终战关）**。关与关之间在商店 / 秘境 / 跳过三选一。Run 结束后所有资源清零——**没有 meta 积累，没有永久解锁，死了从第 1 关重开**。

---

## 2. Run 完整流程

```
开始新 Run
  ↓
起始卡池：4 张基础卡（每张卡唯一，初始均为 Lv.1）
  ↓
关卡 1（边境绿野）— 每关 8-14 波
  ┌ 关内循环 ───────────────────────────────────────────┐
  │ 1. 手牌上限 4 张；关卡开始时抽满 4 张                  │
  │ 2. 每 5 秒可手动抽 1 张：                            │
  │    - 手牌未满才可抽                                  │
  │    - 第一次抽到不想要，可立即丢弃并再抽 1 次          │
  │    - 第二次结果必须保留，不能再丢弃                   │
  │ 3. 打出手牌 → 消耗水晶能量（2 秒恢复 1 点，上限 10）   │
  │ 4. 大部分单位攻击兵、不攻击塔；少量单位无攻击力，只沿路走│
  │ 5. 一局可部署的塔总量有限，预期上限约 20 个            │
  └─────────────────────────────────────────────────────┘
  关底首领/强化波被击败 → 获得金币 + 过关奖励卡 / 资源
  ↓
关后成长来源：
  🏪 商店  → 花金币购买新卡 / 升级已有卡 / 删除卡（费用翻倍递增）
  🌀 秘境  → 随机事件，可获得卡、金币、删卡机会或承担代价
  🎁 过关奖励 → 补充新卡或资源，所有新卡均保持唯一
  ↓
关卡 2 → 关卡 3 → … → 关卡 8（异界裂隙，终战）
  ↓
Run 胜利：记录 Hero Score / 通关次数 / 最远关卡（仅展示，不解锁内容）
Run 失败：同样记录战绩，从第 1 关重开，无任何补偿
```

## 3. 二大资源（v3.6）

两种资源的作用范围严格分层，绝不互通：

| 资源 | 作用范围 | 来源 | 消耗用途 |
|------|---------|------|---------|
| **水晶能量 E** | 关内持续 | 自动回复：**每 2 秒 +1E**，上限 10 | 打出手牌部署 / 释放法术 |
| **金币 G** | 本 Run 关间 | 过关奖励 / 商店 / 秘境 / 战斗结算 | 买新卡 / 升级卡 / 删除卡 / 其他关间服务 |

> 技能点 SP 已废弃；当前资源轴为**水晶能量 + 金币**。

**部署总量不是货币，而是本局硬节奏上限：**

| 约束 | 作用范围 | 规则 |
|------|---------|------|
| **塔位总量预期** | 单局全程 | 一局下来可抽到并打出的塔数量有限，**预期最多部署约 20 个塔** |

**Run 结束：二资源全部清零，没有任何东西能带到下一局。**

## 4. 卡牌系统（v3.6）

### 4.1 四类卡

| 类型 | 打出效果 | 打出后去哪 |
|------|---------|----------|
| **建筑卡（Tower）** | 在地格上放置防御塔/生产建筑 | 留在场上直到被摧毁（占用人口） |
| **单位卡（Soldier）** | 召唤我方战斗单位 | 留在场上直到战死（占用人口，死亡释放） |
| **法术卡（Spell）** | 立即触发效果（伤害/控制/Buff） | 效果即时消散（**不占用人口**） |
| **陷阱卡（Trap）** | 布置在路径地格上，触发时生效 | 留在场上（占用人口） |

### 4.2 手牌节奏（v3.6）

- **手牌上限 4 张**
- 关卡开始时抽满 4 张
- **不再是打出立即补牌**，而是改为**每 5 秒允许手动抽 1 张**
- 若本次为一次抽卡动作的**第一次抽取**，玩家可选择：
  - 保留该牌；或
  - 丢弃该牌并立刻再抽 1 张
- **第二次抽取结果必须保留**，不能再次丢弃
- 手牌已满时**不能抽卡**
- 该机制让抽卡本身成为一个明确的节奏决策，而不是自动流水线

### 4.3 开局卡组

- **起手卡池为 4 张基础卡**（固定起手，每 Run 相同）
- 之后通过**商店 / 秘境 / 过关奖励**获得新卡 → 自动进入卡池（同时自动解锁科技树节点）
- **卡池每种卡唯一**，不持有两张同种卡
- 每张卡拥有独立等级上限，设计上为 **Lv.1 / Lv.2 / Lv.3**，但并非所有卡都一定有 3 级；单卡上限可为 **1-3 级**
- 核心成长不是“复制强卡”，而是“扩卡池 + 提单卡等级 + 精简卡池”

## 5. 水晶机制（核心防守目标）

水晶是玩家要保护的防御实体，位于地图终点。

- **无敌**：免疫一切敌方伤害（普攻、技能、AoE、自爆……全免）
- **秒杀**：进入水晶攻击范围的敌人被立即秒杀，每杀 1 个**消耗水晶 1 HP**
- **终战 Boss 例外**：Boss 携带免疫水晶秒杀标记，进入范围不触发秒杀也不消耗 HP

**水晶 HP 全程跨关继承，不自动恢复。** 恢复路径仅限：商店购买 / 秘境治疗事件 / 「治疗水晶」法术卡。

水晶 HP = 0 → 水晶碎裂 → Run 失败，从第 1 关重开。

**Crystal 升级（v3.5 新增）**：在关间花金币升级防御水晶（Crystal Lv1→Lv2→Lv3），每升一级提升人口上限 + 卡牌等级上限（见 [TECH-TREE.md](./TECH-TREE.md)）。

## 6. 关间三选一（Roguelike 决策核心）

| 选项 | 不确定性 | 最适合场景 |
|------|---------|----------|
| **🏪 商店** | 低（明码标价） | 金币充裕、想扩卡/升级/删卡 |
| **🌀 秘境** | 高（随机事件） | 想搏高收益、换删卡机会、处理卡池问题 |
| **🎁 过关奖励** | 中（奖励池可控） | 稳定补充新卡或资源 |

核心成长来源：
- **商店**：买新卡、升级已有卡、删除卡
- **秘境**：随机事件，可能给金币、给卡、删卡、掉血、扣资源
- **过关奖励**：稳定提供一卡 / 一次资源奖励 / 一次成长选择

**删除卡规则**：
- 仅能删除当前已拥有卡池中的卡
- 删除费用使用金币支付
- **每次删除后，下一次删除费用翻倍增加**
- 删卡是重要的 deck-thinning 工具，用于把卡池从“收集”重新压回“成型”

## 7. 科技树 / 卡牌升级（v3.6 本 Run 内成长）

Run 内积累的金币可在关间投入到已有卡牌，提升卡牌等级。

每张卡的等级上限为 **1-3 级**，按卡牌设计决定：

```
卡牌（例：箭塔）
  ├── Lv.1（入手默认）：基础效果，免费
  ├── Lv.2（若该卡支持）：花金币升级
  └── Lv.3（若该卡支持）：花金币升级
```

**关键规则：**
- **获卡 = 自动解锁该卡成长线**（无需额外解锁步骤）
- 升级花金币（goldCost）
- **不同卡牌等级上限可不同**：基础功能卡可以只有 Lv.1，核心卡可有 Lv.2/Lv.3
- **关内不可升级**，仅关间操作
- **科技树本 Run 内有效，Run 结束清零**
- 成长决策的三角关系是：**买新卡 / 升老卡 / 删废卡**

## 8. 单位阵容（"一切皆单位"）

友方单位阵容（全部通过卡牌部署）：
- **塔（7 种）**：箭塔 / 炮塔 / 元素塔 / 电塔 / 激光塔 / 蝙蝠塔 / 导弹塔
- **士兵（6 种）**：盾卫 / 剑士 / 弓手 / 牧师 / 工程师 / 刺客
- **陷阱（9 种）**：地刺 / 冰陷 / 爆炸陷阱 / 减速网等（详见 [UNITS.md](./UNITS.md)）
- **法术（14 种）**：火球 / 箭雨 / 全屏冰冻 / 治疗水晶 / 时停等
- **生产建筑（2 种）**：金矿 / 能量水晶

敌方阵容 20 种，8 关各有主题（边境绿野 → 沙漠虫潮 → 极地暴雪要塞 → 失落丛林神庙 → 沉没港口 → 齿轮蒸汽工厂 → 腐殖孢子菌林 → 异界裂隙）。

## 9. 技术架构四层

```
配置层 (YAML/JSON)        ← 策划维护，驱动一切
    ↓
规则引擎 (RuleEngine)      ← 解析配置，分发生命周期事件，执行 RuleHandler
    ↓
ECS World (bitecs)         ← 所有实体 = Entity + Component，系统 = 纯函数
    ↓
渲染层 (PixiJS WebGL)      ← 几何图形 + 粒子特效 + UI 容器管理
```

技术栈：TypeScript + PixiJS + bitecs，无框架依赖。详见 [TECH.md](./TECH.md)。

## 10. 设计不变式（v3.6，开发前请背下来）

这 14 条是整个设计的底线，违反任意一条 = BUG：

1. **单 Run 闭环**：Run 结束所有金币/卡池/卡等级/水晶能量相关状态全部清零，无 meta 入账
2. **死亡从第 1 关重开**：不允许从任一已通关重开
3. **水晶能量上限固定 10**：无永久升级
4. **水晶能量自动回复固定为 2 秒 1 点**
5. **手牌上限固定 4 张**：满手不能抽
6. **抽卡为手动节奏**：每 5 秒最多抽 1 张，不是打出即补
7. **一次抽卡最多重抽 1 次**：第一次可丢弃，第二次必须保留
8. **关外卡池每种卡唯一**：不持有两张同种卡；获卡=自动解锁成长线
9. **卡牌等级上限按单卡定义为 1-3 级**：不是所有卡都必须有 Lv.3
10. **关间成长来源固定为商店 / 秘境 / 过关奖励**
11. **删除卡使用金币支付，且删除费用逐次翻倍**
12. **一局塔部署总量有限**：预期上限约 20 塔
13. **多数敌方单位不攻击塔**：主压力来自冲线与清兵；少量特种敌或机制例外
14. **存在无攻击力单位**：其价值来自阻挡、治疗、生产、诱导或纯移动机制

## 11. 存档策略（极简）

**存档只保存历史数据，不保存任何本 Run 状态。**

存档内容：历届最远关卡 / Hero Score 榜单 / Run 通关次数 / 成就 / 设置。

Run 进行中意外退出 = Run 丢失（不支持中断续接）。

详见 [TECH.md](./TECH.md)（存档系统章节）。

## 12. 验收标准（摘要）

| 阶段 | 关键门禁 |
|------|---------|
| MVP | 浏览器可完成完整一局 Run（抽卡组→出卡→战斗→关卡通过/失败） |
| 系统完善 | 完整卡牌/能量/金币/关间节点/水晶 HP 继承/AI 规则引擎 |
| 内容填充 | 8 关完整可玩（关 8 为终战），≥30 张卡，流派识别，天气系统 |
| 打磨 | 60fps 稳定，音效完整，响应式布局（1280×720 至 3440×1440） |

体验目标：单局 Run 30-45 分钟；敌人在 60px 尺寸下 2 秒内可识别；核心操作 ≤ 2 步。
