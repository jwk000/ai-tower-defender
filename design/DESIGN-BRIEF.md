# Tower Defender — 核心设计简要说明

> 面向新合作者的 10 分钟读物。当前版本：**v3.5（2026-05-18）**
>
> 需要深入某个子系统？查阅 [`design/README.md`](./README.md) 中的权威文档索引。

---

## 一、一句话定位

**"把塔防做成卡牌 Roguelike 长征"** — 怪物火车 × 月圆之夜 × 塔防的融合体。

每次游玩（称为一次 **Run**）从随机卡池出发，连续闯过 8 关 + 终战 Boss，关与关之间在商店/秘境/跳过三选一。Run 结束后所有资源清零——**没有 meta 积累，没有永久解锁，死了从第 1 关重开**。

---

## 二、单次 Run 的完整流程

```
开始新 Run
  ↓
从全卡池随机抽 4 张起手卡（每次获卡自动解锁科技树节点）
  ↓
┌──────────────────────────────────────────────────────────────────┐
│  关卡 1（平原）                                                    │
│  ┌ 每波循环 ────────────────────────────────────────────────┐    │
│  │ 1. 手牌固定 4 张，打出一张立即随机抽补 1 张               │    │
│  │ 2. 战斗进行中：能量每波爆发 +2E + 持续 +1E/4s（上限 10）  │    │
│  │ 3. 玩家拖卡到地图 → 消耗能量 + 占用人口部署单位/建筑/法术/陷阱│    │
│  │ 4. 敌人沿路径推进，被我方单位拦截攻击                       │    │
│  │ 5. 敌人全灭 → 结算金币 → 下一波                           │    │
│  └───────────────────────────────────────────────────────────┘    │
│  关底 Boss 被击败 → 关卡通关奖励金币                               │
└──────────────────────────────────────────────────────────────────┘
  ↓
关后 3 选 1 节点（选其中一条路走）：
  🏪 商店      →  花本局金币买单位卡（进入卡池）/功能道具/Crystal 升级，稳健补强
  🌀 秘境      →  随机事件，高风险高收益（约 30% 事件可能扣 HP/删卡）
  ⏭ 跳过       →  零代价直接进下一关，保血保资源
  ↓
关卡 2 → 关卡 3 → … → 关卡 8
  ↓
终战 Boss 关（无关后节点，打完直接结算）
  ↓
Run 胜利：记录 Hero Score / 通关次数 / 最远关卡（仅展示，不解锁内容）
Run 失败：同样记录战绩，从第 1 关重开，无任何补偿
```

## 三、二大资源（v3.5 边界最重要）

**两种资源的作用范围严格分层，绝不互通：**

| 资源 | 作用范围 | 来源 | 消耗用途 |
|------|---------|------|---------|
| **能量 E** | 关内单波 | 每波开始 +2E 爆发 + 战斗中 +1E/4s 持续回复，上限 10 | 拖卡出牌部署（塔主动技能不消耗能量） |
| **金币 G** | 本 Run 关间 | 击杀敌人 + 波次结算 + 关卡通关奖励 | 关间商店 / 秘境部分事件 / Crystal 升级 / 科技树节点 |

> **v3.5 注意**：技能点 SP 已彻底废弃，资源轴从三资源（能量/金币/技能点）缩减为二资源（能量/金币）。

**另外：人口不是货币，是持续占用型约束资源。**

| 约束资源 | 作用范围 | 规则 |
|---------|---------|------|
| **人口** | 关内持续 | 出卡部署持久单位（塔/士兵/陷阱/生产建筑）占用 `energyCost` 值；单位死亡释放；受 Crystal 等级控制上限 |

**Run 结束：二资源全部清零，没有任何东西能带到下一局。**

## 四、卡牌系统（v3.5）

### 4.1 四类卡

| 类型 | 打出效果 | 打出后去哪 |
|------|---------|----------|
| **建筑卡（Tower）** | 在地格上放置防御塔/生产建筑 | 留在场上直到被摧毁（占用人口） |
| **单位卡（Soldier）** | 召唤我方战斗单位（盾兵/剑士/弓手等） | 留在场上直到战死（占用人口，死亡释放） |
| **法术卡（Spell）** | 立即触发效果（伤害/控制/Buff） | 效果即时消散（**不占用人口**） |
| **陷阱卡（Trap）** | 布置在路径地格上，触发时生效 | 留在场上（占用人口） |

### 4.2 手牌节奏（v3.5 设计意图）

- **手牌固定 4 张**（始终）
- **打出一张立即随机抽补一张**（不再是波次开始/每 8 秒抽牌，而是打出触发立即补）
- 卡池来自「已拥有的卡」（关外每种卡唯一，卡池随 Run 进行通过商店/奖励不断扩充）

### 4.3 开局卡组

- **起手 4 张基础卡**（固定起手，每 Run 相同）
- 之后通过商店购买、关后奖励、秘境事件获得新卡 → 自动进入卡池（自动解锁科技树节点）
- **卡池每种卡唯一**，不会持有两张同种卡

## 五、水晶机制（核心防守目标）

水晶是玩家要保护的防御实体，位于地图终点。

- **无敌**：免疫一切敌方伤害（普攻、技能、AoE、自爆……全免）
- **秒杀**：进入水晶攻击范围的敌人被立即秒杀，每秒杀 1 个 **消耗水晶 1 HP**
- **终战 Boss 例外**：Boss 携带免疫水晶秒杀标记，进入范围不触发秒杀也不消耗 HP

**水晶 HP 全程跨关继承，不自动恢复。** 恢复路径仅限：商店购买 / 秘境治疗事件 / 「治疗水晶」法术卡。

水晶 HP = 0 → 水晶碎裂 → Run 失败，从第 1 关重开。

**Crystal 升级（v3.5 新增）**：在关间花金币升级防御水晶（Crystal Lv1→Lv2→Lv3），每升一级提升人口上限 + 卡牌等级上限（见科技树章节）。

## 六、关间三选一（Roguelike 决策核心）

每关通关后弹出三向选择，策略博弈发生在这里：

| 选项 | 不确定性 | 最适合场景 |
|------|---------|----------|
| **🏪 商店** | 低（明码标价，8 个商品槽） | 金币充裕、想扩充卡池或升级 Crystal |
| **🌀 秘境** | 高（随机事件 + 30% 高风险） | 资源紧张、需要一次爆发翻盘 |
| **⏭ 跳过** | 零（完全无变化） | 水晶 HP 危险、屯资源冲终战 |

商店 8 槽细节：
- 前 4 槽：随机单位卡，价格 30 / 60 / 120 / 240 G（购买后进入卡池，自动解锁科技树节点）
- 后 4 槽：功能卡随机出现，含「水晶 HP 恢复」「能量瓶」「手牌刷新」「Crystal 升级」等

## 七、科技树（v3.5 本 Run 内单位强化）

Run 内积累的金币可在关间投入到已有单位的科技树节点，提升卡牌等级。

### 结构

每张卡有 **Lv.1 / Lv.2 / Lv.3** 三个线性等级，对应原来的 depth=1/2/3 节点效果：

```
单位卡（例：箭塔）
  ├── Lv.1（入手默认）：基础效果，免费
  ├── Lv.2（进阶）：+1 发投射物，花金币升级（需 Crystal Lv.2）
  └── Lv.3（高阶）：+1 发 + 灼烧，花金币升级（需 Crystal Lv.3）
```

**关键规则：**
- **获卡 = 自动解锁科技树节点**（无需额外解锁步骤）
- 升级花金币（goldCost），无路径互斥
- **卡牌等级上限 = Crystal 当前等级**（Crystal Lv.1 → 卡最高 Lv.1；Crystal Lv.3 → 卡最高 Lv.3）
- Crystal 升级费用较高，形成「先扩阵容 vs 先提 Crystal 上限 vs 直接升核心卡」三向策略取舍
- **关内不可升级**，仅关间节点操作

**科技树只在本 Run 内有效，Run 结束清零。**

## 八、单位设计哲学（"一切皆单位"）

**塔、士兵、敌人、中立机关本质相同**，只是配置不同。所有单位都有：

- 静态属性（HP、攻击、射程、速度…）
- 行为规则（目标选择、攻击模式、移动模式）
- 科技树（仅玩家单位，v3.5 线性 Lv1/2/3）

全部单位配置以 YAML/JSON 驱动，策划直接改配置文件即可调整行为，无需改代码。

友方单位阵容（全部通过卡牌部署）：
- **塔（7 种）**：箭塔 / 炮塔 / 元素塔 / 电塔 / 激光塔 / 蝙蝠塔 / 导弹塔
- **士兵（6 种）**：盾卫 / 剑士 / 弓手 / 牧师 / 工程师 / 刺客
- **陷阱（9 种）**：地刺 / 冰陷 / 爆炸陷阱 / 减速网等
- **法术（14 种）**：火球 / 箭雨 / 全屏冰冻 / 治疗水晶 / 时停等
- **生产建筑（2 种）**：金矿 / 能量水晶

敌方阵容 20 种，8 关各有主题（平原 → 森林 → 沼泽 → 雪山 → 沙漠 → 废墟 → 熔岩 → 王城），越往后难度乘数越高。

## 九、技术架构四层（给程序看）

```
配置层 (YAML/JSON)        ← 策划维护，驱动一切
    ↓
规则引擎 (RuleEngine)      ← 解析配置，分发生命周期事件，执行 RuleHandler
    ↓
ECS World (bitecs)         ← 所有实体 = Entity + Component，系统 = 纯函数
    ↓
渲染层 (PixiJS WebGL)      ← 几何图形 + 粒子特效 + UI 容器管理
```

**关键设计：**
- 单位行为由配置中的 `targetSelection` / `attackMode` / `movementMode` + 生命周期 `RuleHandler` 驱动，不写 if/else 硬编码逻辑
- Pipeline 按 8 阶段固定顺序执行，每帧调度
- 阵营语义统一为 `isHostileTo(a, b)` API，无 `isEnemy` 双轨

**技术栈**：TypeScript + PixiJS + bitecs，无框架依赖。

## 十、设计不变式（v3.5，开发前请背下来）

这 13 条是整个设计的底线，违反任意一条 = BUG：

1. **单 Run 闭环**：Run 结束所有金币/科技树/卡池/水晶 HP 全部清零，无 meta 入账
2. **Crystal 等级本 Run 临时**：不写存档，Run 结束清零（下一 Run 从 Lv.1 开始）
3. **死亡从第 1 关重开**：不允许从任一已通关重开
4. **水晶 HP 跨关继承**：每关结束不自动回复
5. **能量上限固定 10**：无永久升级
6. **手牌固定 4 张，打出立即补**：上限固定，无永久升级，打出一张立即随机抽补一张
7. **关外卡池每种卡唯一**：不持有两张同种卡；获卡=自动解锁科技树节点
8. **关后 3 选 1 三向互斥**：选 1 即进入下一关，不可回头
9. **终战无关后节点**：终战通关直接进 Run 结算
10. **关内不花金币**：战斗中金币只攒不花，金币仅在关间消费
11. **人口持续占用**：单位在场占用 energyCost 人口，死亡释放；受 Crystal 等级上限约束
12. **instanceLevel 不存在**：废弃（v3.5 只有卡牌线性等级 Lv1/2/3）
13. **技能点 SP 不存在**：废弃（v3.5 用金币驱动科技树）

## 十一、存档策略（极简）

**存档只保存历史数据，不保存任何本 Run 状态。**

存档内容：历届最远关卡 / Hero Score 榜单 / Run 通关次数 / 成就 / 设置。

Run 进行中意外退出 = Run 丢失（不支持中断续接）。
