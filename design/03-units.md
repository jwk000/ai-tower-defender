### 1.1 配置驱动约束

运行时单位数值必须优先来自 `src/config/units/*.yaml`，桥接层只允许在配置字段缺失时使用兜底默认值，不能用代码默认值覆盖已配置字段。

- 塔、士兵、敌人的 `stats.damageType` 必须注入运行时攻击组件；`true` 表示真实伤害，绕过护甲和魔抗。
- 塔的 `buildTime`、`cost.atkGrowth`、`cost.rangeGrowth` 必须来自配置；未配置时才使用旧默认值。
- 士兵的 `cost.hpGrowth`、`cost.atkGrowth`、`cost.tauntCapacityGrowth`、`cost.maxLevel` 必须来自配置；未配置时才使用旧默认值。
- 敌人的 `visual.size` 必须决定运行时半径，Boss 不允许被桥接层强制压缩到固定尺寸；`attackAnimDuration` 可由配置覆盖，缺失时再按 Boss/非 Boss 使用默认值。
- 配置中历史遗留的 `damageType: magical` 在加载时归一为 `magic`，运行时不保留新枚举。

## 2. 塔类单位

塔固定在地面层，不可移动，自动攻击射程内敌人。每座塔有升级路径，升级改变外观+提升属性+可能解锁新机制。

### 2.0 低空攻击规则

低空敌人统一使用 `layer: LowAir`。默认地面范围伤害、地面控制和地面爆炸不命中低空层；只有配置为对空白名单的单位会主动选择并命中低空敌人。

吸血蝗虫、吸血蝙蝠、无人机使用群体出生规则：关卡配置中的 `count` 表示群数，每次出生同时生成 4-7 个成员。群体成员使用 Boid 鸟群算法移动，包含分离、对齐、凝聚和路径推进，接近玩家士兵或塔时按普通敌人攻击逻辑停下攻击。

| 单位 | 是否可攻击LowAir | 设计说明 |
|------|------------------|----------|
| 箭塔 | 是 | 基础防空塔，箭矢可锁定低空飞行敌人 |
| 弩塔 | 是 | 直线穿透弩箭可贯穿同一直线上的地面与低空敌人 |
| 激光塔 | 是 | 光束锁定不受地面/低空高度差影响 |
| 蝙蝠塔 | 是 | 蝙蝠本身为低空单位，可追击低空敌人 |
| 电塔 | 是 | 连锁闪电可在地面与低空敌人之间弹跳 |
| 炮塔 | 否 | 炮弹落地爆炸，仅处理地面密集敌人 |
| 导弹塔 | 否 | 战略地面爆炸，不主动选择或伤害低空敌人 |
| 冰塔 | 否 | 地面冰晶控制，不命中低空敌人 |
| 火塔 | 否 | 地面火焰/DOT，不命中低空敌人 |
| 毒塔 | 否 | 地面毒液/DOT，不命中低空敌人 |
| 弓手 | 是 | 士兵防空补位，适合低空教学关和随机卡池兜底 |
| 法师 | 否 | 地面AOE法术，不命中低空敌人 |

关卡配置出现低空波次时，初始卡池或关间成长必须保证至少存在一个稳定 LowAir 对策。详见 [04-levels §1.1](./04-levels.md#11-低空单位与防空对策总览)。

## 3. 士兵单位

士兵放置后可在移动范围内巡逻和战斗。放置点在地面层。

### 3.0 本轮经济与差异化调整

本轮调整目标是让塔成为高价高血量的稳定阵地投资，士兵成为低价、职责明确的战术补位。士兵不再按接近的通用战斗模板设计，而是按前排承伤、远程消耗、爆发斩杀、负面状态、治疗支援、塔修复六类体验拆开。

- 塔：建造价格整体上调，HP 整体上调，玩家每次建塔更像长期投资。
- 士兵：部署价格整体下调，便于在最坏卡序下用低价单位补足防线。
- 盾卫：极高 HP 和护甲，低攻击，只负责吸收火力。
- 弓手：超远射程、低攻击，靠概率 2 倍 / 3 倍暴击制造峰值。
- 刺客：低 HP、高攻击、高移速，可瞬移处决 HP 低于 10% 的普通敌人，不处决精英和 Boss。
- 法师：远程魔法攻击，攻击与周期法术附加 `arcane_vulnerability` 负面 Buff。
- 牧师：由“祭司”改名为“牧师”，治疗我方士兵，同时保留少量攻击力。
- 工程师：无攻击力，只修复我方塔 HP。

## 6. BOSS

> BOSS统一标记 `category: Boss`，拥有专用AI和特殊技能。BOSS不受捕兽夹影响。

### 6.0 Boss通用战斗与表现规则

- 体型：Boss 场景可见尺寸必须超过 1 个地格的基础单位体量，运行时尺寸钳制在 70×70 到 90×90 像素之间；精英放大规则不得作用于 Boss。
- 出场：Boss 首次进入场景时播放 `wave_boss` 音效，并在 Boss 头顶生成红色飘字“xxx出现！”，飘字需要明显高于普通提示。
- 常驻表现：Boss 自带粒子特效，至少包含暗红/主题色光环、环绕粒子或脉冲环，保证即使没有专用贴图也能被识别为 Boss。
- 普攻：Boss 攻击塔时不能移动，使用完整蓄力/挥击动画；普攻伤害按当前目标塔最大 HP 计算下限，保证约 2-3 次攻击可以摧毁一座同级塔。
- 技能：Boss 技能必须有明显的视觉反馈，包括预警、爆发范围、震屏、粒子或地面残留之一；不能只有数值变化。
- 死亡清场：最终 Boss 真正死亡后，场上剩余所有敌方单位立即死亡；巨型史莱姆前两次 HP 归零属于分裂技能，不算真正死亡，只有最后一层小型史莱姆死亡才算 Boss 真正死亡。
- 敌方死亡：所有敌方单位死亡时统一播放“灰飞烟灭”的破碎消散动画，表现为本体灰化、碎片外扩、透明度衰减。

### 6.1 敌人技能完整性规则

敌人配置必须满足以下技能完整性要求：

- 精英怪：所有 `tier: L2` 的敌人视为精英怪，必须配置且仅配置 1 个技能。
- Boss：所有 `isBoss: true` 或 `category: Boss` 的敌人视为 Boss，必须配置 1 个以上技能。
- 技能字段统一使用 `skills` 数组，每项至少包含 `id`、`name`、`cooldown`、`range`、`value`、`description`。
- 当前运行实现中，`EnemySkillSystem` 会自动执行挂载 Boss 组件的 YAML 技能；精英怪技能属于设计与配置要求，若需要战斗中主动释放，后续必须补齐精英技能执行系统或接入生命周期规则。

当前敌人配置审查结论：

- Boss 缺口：`giant_slime`、`queen_beetle`、`lucifer`、`super_robot`、`abyss_lord` 尚未配置 `skills`。
- 精英怪缺口：`heavy`、`mage`、`exploder`、`e_burrow_worm`、`e_giant_beetle`、`e_frost_marauder`、`e_yeti_charger`、`e_blizzard_sprite`、`e_brine_charger`、`e_steam_lancer`、`e_hammergrub_drone`、`e_minecart_juggernaut`、`e_myco_charger`、`e_brood_mother`、`e_blight_spitter`、`e_frost_lancer`、`e_blood_trickster` 尚未配置精英技能。

#### 7.2.1 技能卡粒子特效规范

技能卡运行时特效全部使用程序化粒子和基础几何绘制，不再加载 `public/art/fx/` 下的法术投射物、命中帧或序列帧图片。卡牌插画仍可使用美术图，但战斗场景中的投射物、命中爆发、持续区域表现必须由粒子、光环、冲击波、拖尾和碎屑组成。

伤害飘字时序必须与真实伤害结算帧对齐：投射物或预警阶段只播放视觉引导，不调用伤害结算；命中特效第 1 帧播放核心爆发、箭矢扎地或暴风雪覆盖时，同帧调用伤害结算，伤害观察者生成的飘字从受击敌人上方出现。持续控制类法术的减速效果也在该命中帧生效，后续区域粒子只表达持续寒风，不重复结算首段伤害。

## 8. 数值设计原则

> **注意** ：上述所有数值为设计初值。实际实现时需通过可玩性测试调优。

### 8.1 数值关系参考

| 关系 | 公式/说明 |
|------|----------|
| 塔DPS ≈ 敌人HP | 同等级塔DPS应能处理同等级敌人 |
| 升级价值 | 每级升级费约等于2-3波的金币收入，属性提升30-60% |
| BOSS挑战 | BOSS HP ≈ 同关普通敌人HP×10-15 |
| 精英价值 | 精英HP×2 ATK×1.5，击杀奖励吸引力匹配卡牌价值 |
| 金币节奏 | 每波金币收入应支持升级1-2座塔一次 |
