## 1. Run 长征循环（v3.6）

### 1.1 玩法定位

| 维度 | v3.6 形态 |
|------|----------|
| 核心循环 | 战斗（手动抽卡 + 4 手牌上限）→ 商店 / 秘境 / 过关奖励成长 → 下一关 → 终战 |
| 单 Run 时长 | 25-40 分钟，连闯 8 关（关 8 为终战） |
| 失败代价 | **从第 1 关重开，无任何 meta 留存** |
| 胜利收益 | **本 Run 闭环**，仅记录"通关次数 + 历届最远 + Hero Score"展示型数据 |
| 长线驱动 | 历届最远关卡、Hero Score、Run 通关次数（仅展示，不解锁内容） |

### 1.2 完整 Run 结构

```
[主菜单 → 开始新 Run]
  → [获得 4 张基础卡作为初始唯一卡池]
  → [关卡 1: 边境绿野]
      → [战斗：手牌上限 4，5 秒 1 次手动抽卡，可 1 次重抽]
      → [过关奖励]
      → [关间成长：商店 / 秘境]
  → [关卡 2: 沙漠虫潮]
  → ...
  → [关卡 7: 腐殖孢子菌林]
  → [关卡 8: 异界裂隙（终战，旧日支配者）]
      → 击败 → Run 胜利
      → 水晶防线崩溃 → Run 失败
```

Run 总波次保持 70-90 波量级，目标时长 25-40 分钟。

### 1.3 关内单波战斗循环

```
波次开始
  ↓
1. 结算当前手牌（上限 4，保留上一波结果）
  ↓
2. 敌人按波次配置生成
  ↓
3. 战斗循环（持续直到清场）
   ├─ 每 2 秒：水晶能量 +1（上限 10）
   ├─ 每 5 秒：获得 1 次手动抽卡机会
   │    若手牌未满，可抽 1 张
   │    第一次抽到可丢弃并立刻再抽 1 张
   │    第二次结果必须保留
   ├─ 玩家拖卡部署（随时）
   │    单位/建筑卡：在合法格部署
   │    法术卡：拖到目标区域立即释放
   └─ 少量单位只走位/阻挡/治疗，不承担输出职责
  ↓
4. 全部敌人死亡 → 清场结算金币 / 奖励
```

## 2. 卡牌系统（v3.5）

### 2.1 卡牌定义

```typescript
interface CardConfig {
  id: string;
  name: string;
  type: 'unit' | 'spell' | 'trap' | 'production' | 'shop_item';
  energyCost: number;    // 打出该牌消耗的水晶能量
  rarity: 'common' | 'rare' | 'epic' | 'legendary';
  maxLevel: 1 | 2 | 3;   // 单卡等级上限，按卡牌设计决定
  unitConfigId?: string;
  spellEffectId?: string;
  spellSubtype?: 'damage' | 'control' | 'buff' | 'utility';
  placement: {
    targetType: 'tile' | 'enemy' | 'ally' | 'area' | 'global';
    range?: 'cursor' | 'path' | 'anywhere' | 'left-half' | 'right-half';
  };
}
```

`energyCost` 仅表示出卡时消耗的水晶能量，不再同时承担人口货币语义。

### 2.2 四类卡

| 大类 | 子类 | 说明 |
|------|------|------|
| **建筑卡（Tower）** | 防御塔 / 生产塔 / 辅助塔 | 部署后占用人口直到被摧毁 |
| **单位卡（Soldier）** | 近战 / 远程 / 空中 / 特殊 | 占用人口，死亡释放 |
| **法术卡（Spell）** | 伤害 / 控制 / Buff / Utility | 打出立即消耗，**不占人口** |
| **陷阱卡（Trap）** | 触发式 / 区域式 / 持续式 | 布置在路径地格，占用人口 |

> `shop_item` 为第 5 类，不进战斗手牌，仅在商店呈现（见 UI.md 商店章节）。

### 2.3 关外卡池与手牌机制（v3.6）

**关外卡池**：
- 玩家本 Run 已解锁的所有卡，每种卡只有 1 张
- 获得卡（商店购买 / 秘境奖励 / 过关奖励）→ 进入卡池（唯一，不重复）
- 卡池支持通过金币**删除卡**做精简
- 每张卡拥有自身的 `maxLevel`，上限为 1-3 级

**关内手牌与抽卡**：
- 关卡开始：从关外卡池随机抽牌，直至手牌达到 4 张
- 战斗中：**每 5 秒**获得 1 次手动抽卡机会
- 手牌未满时才可抽
- 第一次抽到若不想要，可立即丢弃并再抽 1 张
- 第二次抽到必须保留
- 同一张卡在单关内可被多次抽到（抽到的是“出牌权限”，不是消耗唯一收藏）
- 波次结束：手牌保留至下一波，不自动清空

> v3.6 核心变化：去掉“打出立即补牌”，改为“定时手动抽卡 + 单次重抽决策”。

### 2.4 关内部署与总量控制

- 打出一张牌时，仅检查**当前水晶能量是否足够**
- **手牌上限**与**抽卡节奏**共同限制单局展开速度
- 单局可部署的塔总量预期控制在 **20 个以内**，避免后期场面失控
- 这不是 UI 上的死板塔位上限，而是由抽卡节奏、能量节奏、关卡时长、卡池结构共同形成的系统结果

### 2.5 过关奖励

每关通过后，玩家获得一次**过关奖励**，用于稳定推进构筑：

- 奖励形态可为：
  - 新卡三选一
  - 金币奖励
  - 指定成长机会（如免费升 1 级、低价删 1 张卡）
- 所有新卡仍遵守**唯一卡**规则
- 过关奖励的目标是提供“低随机、高确定性”的中速成长，作为商店与秘境之间的稳定层

## 3. 经济系统（v3.6 二资源轴）

### 3.1 二种资源

| 资源 | 作用范围 | 来源 | 消耗用途 |
|------|---------|------|---------|
| **水晶能量 E** | 关内持续 | 自动回复：每 2 秒 +1E，上限 10 | 出卡部署 |
| **金币 G** | 本 Run 关间 | 过关奖励 + 战斗结算 + 商店/秘境事件 | 买卡 / 升卡 / 删卡 / 关间服务 |

二资源**作用范围严格分层**，无跨货币兑换通道。Run 结束全部清零。

> 技能点 SP 已于 v3.5 彻底废弃。

### 3.2 水晶能量规则

| 维度 | 规则 |
|------|------|
| 上限 | **固定 10** |
| 自动恢复 | **+1E / 2 秒** |
| 关内用途 | 打出手牌 |
| 额外限制 | 手牌已满时即便有抽卡机会也不能抽 |

该资源的设计目标是：让玩家始终在“现在打”与“攒一下等更关键卡”之间做判断。

### 3.3 金币来源与消耗

**来源**：
- 过关奖励
- 战斗结算
- 商店 / 秘境事件奖励
- 生产建筑或特定成长线

**消耗**：
- 购买新卡
- 升级已有卡
- 删除卡
- 其他关间服务

**删除卡费用曲线**：
- 第 1 次删除：基础费用
- 第 2 次删除：翻倍
- 第 3 次删除：再次翻倍
- 以此类推

删除费用递增的目的，是让删卡成为“昂贵但强力”的后期精炼手段，而不是开局就无脑瘦身。

### 3.4 经济节奏

| 阶段 | 金币预算/关 | 关键决策 |
|------|------------|---------|
| 关 1-2（开局） | 80-150G | 先买卡扩池还是升 Crystal 提人口？ |
| 关 3-5（构筑） | 200-400G | 确立流派，开始把核心卡升级到 Lv2 |
| 关 6-8（成型） | 500-1000G | 卡组成型，Crystal Lv3 解锁高等级牌上限 |
| 终战 | 累计 1000-2000G | 全力一搏，Run 结束清零 |

## 4. 关后成长系统

### 4.1 成长结构

```
关卡 N 通过
  ↓
[过关奖励]
  ↓
[可进入商店 / 触发秘境]
  ↓
进入下一关
```

相比旧版“单一三选一节点”，v3.6 更强调：
- **过关奖励**负责稳定补强
- **商店**负责可控投入
- **秘境**负责风险换收益

### 4.2 商店节点

商店承担三类核心功能：
- **买新卡**：扩充卡池，打开新策略轴
- **升老卡**：集中强化当前核心卡
- **删废卡**：用金币精简卡池，提升抽中关键卡概率

### 4.3 秘境节点

秘境承担高波动成长：
- 可能直接送卡 / 送钱 / 给删卡机会
- 也可能掉血 / 扣钱 / 引入额外风险
- 用于制造 Run 内“赌一把”的记忆点

### 4.4 过关奖励

过关奖励承担稳定保底成长：
- 让玩家即便不进商店，也能持续变强
- 让构筑不会因为单次商店刷坏而完全停滞

## 5. 核心对抗关系

### 5.1 防守目标

- 玩家目标仍然是防住终点，阻止敌人完成冲线
- 但战斗压力主要来自**处理兵线**，而不是传统塔防里双方建筑互拆

### 5.2 敌我攻击关系

- **大多数单位不能攻击塔，但可以攻击兵**
- 这意味着：
  - 塔更偏向稳定输出点
  - 兵更像前线消耗层与拦截层
  - 敌人多数时候是在“冲线 + 清兵”，不是“拆家 + 拆塔”
- 只有少量特种敌人、Boss 机制或特殊事件，才会对塔形成直接威胁

### 5.3 无攻击力单位

部分单位可以**完全没有攻击力**，但仍然有战术价值，例如：
- 纯阻挡
- 纯治疗
- 纯生产
- 纯诱饵
- 纯沿路移动触发机制

这类单位的价值来自位置与功能，而不是 DPS。

## 7. 天气系统

5 种天气，每波结束在天气池中随机切换，1.5 秒渐变过渡。天气**仅影响战斗数值，不影响经济**。

| 天气 | 主要效果 |
|------|---------|
| **晴天 Sunny** | 默认天气，炮塔 ATK +20%，冰冻时间 -0.3s |
| **下雨 Rain** | 炮塔 ATK -30%，电塔 ATK +30% + 链击 +1 跳，敌人移速 -5% |
| **下雪 Snow** | 冰塔 ATK +30%，炮塔 ATK +10%，电塔 ATK -10%，敌人移速 -25% |
| **下雾 Fog** | 所有塔射程 -15%，箭塔攻速 -30%，激光塔 ATK +30%，蝙蝠塔无射程惩罚，敌人移速 -10% |
| **夜晚 Night** | 所有塔射程 -20%，蝙蝠塔 ATK +50%，激光塔射程 +10%，敌人移速 +18% |

## 8. 地图系统

### 8.1 地图规格

- 设计分辨率：1920×1080（横屏 16:9）
- 场景区域：**21×9 格**（1344×576 px），水平垂直居中
- Tile 大小：64×64 px

### 8.2 Tile 类型

| 类型 | 说明 | 行为 |
|------|------|------|
| `empty` | 空地 | 可拖卡部署（需在路径邻格） |
| `path` | 路径 | 敌人行走，不可部署 |
| `trap_path` | 可埋陷阱的路径 | 视觉同 `path`，但可拖陷阱卡部署 |
| `blocked` | 障碍 | 不可通行不可部署 |
| `spawn` | 敌人出生点 | 位于地图右侧 |
| `crystal` | 水晶位置 | 位于地图左侧 |

建造限制：仅可在路径的 8 方向相邻 1 格内的空地上部署；合法格位视觉高亮（浅绿 + 边框）。

### 8.3 PRNG 多流隔离

| 流 | 用途 | 种子派生 |
|---|------|---------|
| `drafterRandom` | 开局抽卡 | `seed ^ 0xDRAFT` |
| `mapRandom` | 地图/天气/装饰物 | `seed ^ 0xMAP` |
| `waveRandom` | 波次内敌人选择 | `seed ^ 0xWAVE` |
| `dropRandom` | 击杀掉落 | `seed ^ 0xDROP` |
| `shopRandom` | 商店商品池 | `seed ^ 0xSHOP` |
| `mysticRandom` | 秘境事件抽取 | `seed ^ 0xMYSTIC` |

禁用 `Math.random()`；使用确定性 PRNG（mulberry32 / xoshiro128++）；同一 seed 在相同操作序列下结果可复现。

## 9. Run 结算

Run 结束（无论胜负）后弹出结算画面：

| 显示项 | 说明 |
|--------|------|
| 结果 | 胜利 / 失败 |
| 最远到达关卡 | 进度（如 7/9） |
| 总击杀 | 累计敌人击杀数 |
| Run 时长 | 用时 |
| 流派标签 | 自动识别（近战墙流/法术爆发流/生产抗压流/远程压制流/混合） |
| 金币结余 | Run 结束清零，无 meta 转换 |

按钮：「返回主菜单」/ 「立即开始新 Run」。

胜利条件：通关终战（W23）Boss 击败。
失败条件：任一关卡水晶 HP ≤ 0，或玩家主动放弃 Run。
