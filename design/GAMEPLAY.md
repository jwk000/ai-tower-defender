## 1. Run 长征循环（v3.5）

### 1.1 玩法定位

| 维度 | v3.5 形态 |
|------|----------|
| 核心循环 | 战斗（手牌 4 张实时补充）→ 关后 3 选 1（商店 / 秘境 / 跳过）→ 下一关 → 终战 |
| 单 Run 时长 | 25-40 分钟，连闯 8 关 + 终战（共 9 关 = W1-W23） |
| 失败代价 | **从第 1 关重开，无任何 meta 留存** |
| 胜利收益 | **本 Run 闭环**，仅记录"通关次数 + 历届最远 + Hero Score"展示型数据 |
| 长线驱动 | 历届最远关卡、Hero Score、Run 通关次数（仅展示，不解锁内容） |

### 1.2 完整 Run 结构

```
[主菜单 → 开始新 Run]
  → [开局抽 4 张起手卡]
  → [关卡 1: 平原]（8 波）
      → [单位卡 3 选 1 抽卡]
      → [关后 3 选 1：商店 / 秘境 / 跳过]
  → [关卡 2: 森林]
  → ...
  → [关卡 8: 王城]（无关间节点，直接接终战）
  → [关卡 9: 终战 Boss 关 = W23]（单波超长）
      → 击败 → Run 胜利
      → 水晶 HP=0 → Run 失败
```

Run 总波次：约 70-90 波 / Run。预计时长 25-40 分钟。

### 1.3 关内单波战斗循环

```
波次开始
  ↓
1. 手牌区保持 4 张（无抽牌步骤，始终满手牌）
  ↓
2. 能量结算：+2E（波次爆发），上限 10
  ↓
3. 出生敌人：按波次配置生成敌人
  ↓
4. 战斗循环（持续直到清场）
   ├─ 每 4 秒：被动恢复 +1E
   ├─ 玩家拖卡部署（随时）
   │    打出 1 张 → 立即从全卡池随机抽 1 张补入（始终保持 4 张）
   │    单位/建筑卡：在合法格部署，作为实体进入战场，占用人口
   │    法术卡：拖到目标区域立即释放
   └─ 玩家点击塔主动技能按钮（随时，各塔独立 CD，不消耗能量）
  ↓
5. 全部敌人死亡 → 清场结算金币
   └─ 速清奖励：清场后 10s 内点击「提前开始」→ +5G
```

## 2. 卡牌系统（v3.5）

### 2.1 卡牌定义

```typescript
interface CardConfig {
  id: string;
  name: string;
  type: 'unit' | 'spell' | 'trap' | 'production' | 'shop_item';
  energyCost: number;    // 出卡消耗能量；同时也是场上单位的人口占用数
  rarity: 'common' | 'rare' | 'epic' | 'legendary';
  unitConfigId?: string;
  spellEffectId?: string;
  spellSubtype?: 'damage' | 'control' | 'buff' | 'utility';
  placement: {
    targetType: 'tile' | 'enemy' | 'ally' | 'area' | 'global';
    range?: 'cursor' | 'path' | 'anywhere' | 'left-half' | 'right-half';
  };
}
```

`energyCost` 同时承担两个角色：出卡时消耗能量 + 场上单位持续占用人口数。法术卡不占人口。

### 2.2 四类卡

| 大类 | 子类 | 说明 |
|------|------|------|
| **建筑卡（Tower）** | 防御塔 / 生产塔 / 辅助塔 | 部署后占用人口直到被摧毁 |
| **单位卡（Soldier）** | 近战 / 远程 / 空中 / 特殊 | 占用人口，死亡释放 |
| **法术卡（Spell）** | 伤害 / 控制 / Buff / Utility | 打出立即消耗，**不占人口** |
| **陷阱卡（Trap）** | 触发式 / 区域式 / 持续式 | 布置在路径地格，占用人口 |

> `shop_item` 为第 5 类，不进战斗手牌，仅在商店呈现（见 UI.md 商店章节）。

### 2.3 关外卡池与手牌机制（v3.5）

**关外卡池**：
- 玩家本 Run 已解锁的所有卡，每种卡只有 1 张
- 获得卡（商店购买 / 关卡奖励）→ 进入卡池（唯一，不重复）
- 每张卡有对应的卡牌等级（1-3 级，受 Crystal 等级上限约束）

**关内手牌与抽卡**：
- 关卡开始（第 1 波前）：从关外卡池随机抽 4 张，充满手牌
- 战斗中：打出 1 张卡 → 立即从全卡池随机抽 1 张补入，始终保持 4 张
- 同一张卡可能被多次抽到（无限随机，可重复）
- **无弃牌堆**，无卡组消耗 / 洗牌机制
- 波次结束：手牌 4 张保留至下一波（不清空）

> v3.5 删除：卡组（deck）/ 弃牌堆 / 每波定时抽牌 / 每 8 秒自动抽。

### 2.4 人口系统（v3.5 新增）

人口（Population）是关内持续占用型约束资源：

```
打出一张持久卡（单位/建筑/陷阱）时，检查：
  ① 当前能量 ≥ energyCost           （能量校验）
  ② 当前已用人口 + energyCost ≤ 人口上限  （人口校验）
两个条件缺一不可。法术卡只做能量校验。

单位死亡 / 建筑被摧毁 / 主动回收 → 立即释放对应人口。
```

人口上限由 Crystal 等级决定（详见科技树章节）。HUD 显示：`人口 占用/上限`。

### 2.5 关卡通关单位卡 3 选 1

每关通过（W1-W22）后，在关后 3 选 1 节点前插入一次单位卡抽卡：

```
┌──────────┬──────────┬──────────┐
│  弓手    │  炮塔    │  剑士    │
│  Common  │  Rare    │  Common  │
└──────────┴──────────┴──────────┘
[ 跳过（不加卡） ]
```

- 从当前关外卡池中**未持有**的单位卡按稀有度权重抽 3 张
- 选中：该卡加入关外卡池（自动解锁对应科技树节点）
- 终战 W23 不触发本机制

## 3. 经济系统（v3.5 二资源轴）

### 3.1 二种资源

| 资源 | 作用范围 | 来源 | 消耗用途 |
|------|---------|------|---------|
| **能量 E** | 关内单波 | 每波 +2E 爆发 + 战斗中 +1E/4s 被动恢复，上限 10 | 出卡部署（塔主动技能不消耗能量） |
| **金币 G** | 本 Run 关间 | 击杀 + 波次结算 + 通关奖励 + 秘境部分奖励 | 商店购卡/功能卡 / Crystal 升级 / 科技树节点 / 秘境部分事件 |

二资源**作用范围严格分层**，无跨货币兑换通道。Run 结束全部清零。

> 技能点 SP 已于 v3.5 彻底废弃。

### 3.2 能量规则

| 维度 | 规则 |
|------|------|
| 上限 | **固定 10**（不可永久升级） |
| 每波爆发恢复 | +2E（瞬时补充，不超上限） |
| 战斗中被动恢复 | +1E / 4 秒（持续滴灌） |
| 跨波保留 | 上一波末剩余能量保留 |
| 回收返还 | `返还能量 = 0.5 × 该卡能量消耗 × (currentHP / maxHP)` |
| 建造冷却 | 3 秒内不可回收 |
| 战斗保护 | 受伤/攻击中 2 秒内不可回收 |

典型一波（约 30s）总能量收入 ≈ 2E（爆发）+ 7E（被动）= 约 9E，玩家每波可出 2-4 张卡。

### 3.3 金币来源与消耗

**来源**（所有数值见 NUMBERS.md）：
- 击杀敌人：按敌人 `bounty` 配置（普通 3-8G / 精英 15-25G / Boss 80-150G）
- 波次完成：基础 20G + 5G × 关卡序号
- 关卡通过：100G + 50G × 关卡序号
- 秘境事件：部分事件奖励 0-200G
- 生产建筑（如金矿）：部署后每秒产出

**消耗**（仅关间节点）：
- 商店单位卡槽：30-240G
- 商店功能卡槽：0-80G
- Crystal 升级：见 NUMBERS.md §NEW-CRYSTAL
- 科技树节点升级：见 NUMBERS.md §NEW-CRYSTAL
- 秘境部分事件：20-150G

**关内不能花金币**——战斗中金币只攒不花，关间一次性决策成为正式策略时刻。

### 3.4 经济节奏

| 阶段 | 金币预算/关 | 关键决策 |
|------|------------|---------|
| 关 1-2（开局） | 80-150G | 先买卡扩池还是升 Crystal 提人口？ |
| 关 3-5（构筑） | 200-400G | 确立流派，开始把核心卡升级到 Lv2 |
| 关 6-8（成型） | 500-1000G | 卡组成型，Crystal Lv3 解锁高等级牌上限 |
| 终战 | 累计 1000-2000G | 全力一搏，Run 结束清零 |

## 4. 关后节点系统

### 4.1 节点结构

```
关卡 N 通过（W1-W22）→ 单位卡 3 选 1 → 关后 3 选 1 节点：

┌─────────────────────────────────────────────────┐
│  关卡 N 通过！选择前往：                           │
├─────────────┬─────────────┬─────────────────────┤
│  🏪 商店    │  🌀 秘境    │  ⏭ 跳过              │
│  确定收益   │  随机事件   │  零成本进入下一关       │
└─────────────┴─────────────┴─────────────────────┘
```

三向互斥：选 1 即进入下一关，不可回头。

### 4.2 商店节点

详见 [UI.md](./UI.md)（商店章节）。边界声明：
- 触发条件：关后 3 选 1 选「商店」时进入
- 入场资源：玩家当前金币余额
- 退出后流程：关闭商店 → 进入下一关战斗准备
- 商店内可购买：新卡（进关外卡池）/ 升级已有卡（金币）/ Crystal 升级（金币）/ 功能性道具

### 4.3 秘境节点

秘境显示一段情境文字 + 2-3 个选择，每个选择有明确收益和潜在代价：

```
秘境面板示例：

┌──────────────────────────────────────────────────┐
│  🌀 秘境 — 古老的祭坛                               │
│  你发现一座血色祭坛，散发着不祥的气息。              │
├──────────────────────────────────────────────────┤
│  [A] 献祭 30% 水晶 HP                              │
│      → 获得 1 张 Legendary 单位卡                  │
│                                                  │
│  [B] 献祭 1 张随机卡                               │
│      → 水晶恢复 40 HP + 80 金币                   │
│                                                  │
│  [C] 拒绝契约离开（零成本）                          │
└──────────────────────────────────────────────────┘
```

秘境特性：
- **每次秘境至少有一个零成本选项**（可无代价退出）
- 约 30% 高风险事件（红边框警示），可能造成损 HP / 删卡等代价
- 已遇过的事件本 Run 不重复
- 奖励池：金币 + 单位卡按权重（v3.5 已废弃 SP 奖励）
- 秘境事件池完整定义见 [UNITS.md](./UNITS.md)（秘境事件池章节）

### 4.4 跳过节点

零成本：不进商店、不触秘境，直接进入下一关。水晶 HP / 卡组 / 金币全部保持现状。
适合：保血流派、节奏控制、屯资源对终战。

## 5. 水晶机制

水晶是玩家要保护的防御实体，位于地图左侧路径终点。

### 5.1 无敌规则

水晶**免疫一切伤害源**——敌人普攻、技能、AoE、毒、点燃、自爆、Boss 必杀、反弹伤害等都无法直接扣除水晶 HP。

正向操作仍生效：治疗水晶法术卡、商店水晶修复道具、秘境治疗事件。
主动负向操作仍生效：秘境「献祭 X% 水晶 HP」（玩家自愿付出的代价）。

### 5.2 秒杀攻击

水晶把进入其攻击范围的敌人立即秒杀（无视 HP、护盾、无敌、霸体、闪避），每次秒杀消耗水晶 1 HP。

| 字段 | 默认值（数值见 NUMBERS.md） |
|------|--------------------------|
| `crystal.attackRange` | 1 格 |
| `crystal.attackCooldown` | 0.5 秒 |
| `crystal.hpCostPerKill` | 1 HP |

目标选择：范围内多敌人时按「最近距离 > 进入范围时间」排序，每次秒杀 1 个。

**Boss 例外**：终战 Boss（`abyss_lord`）携带 `immune_to_crystal_kill = true`，水晶无法秒杀 Boss，进入范围也不消耗 HP。

### 5.3 水晶 HP

```
初始 HP：100（固定，Run 开始时）

变化：
  -1 HP：每次秒杀 1 个敌人
  +N HP：商店「水晶 HP 恢复」道具
  +N HP：秘境治疗事件
  +N HP：「治疗水晶」法术卡（关内）
  -N HP：秘境献祭 / 高风险事件（按事件配置）
  跨关：HP 继承（不自动恢复）

水晶 HP = 0 → 水晶碎裂动画 → Run 失败
```

## 7. 天气系统

5 种天气，每波结束在天气池中随机切换，1.5 秒渐变过渡。天气**仅影响战斗数值，不影响经济**。

| 天气 | 主要效果 |
|------|---------|
| **晴天 Sunny** | 默认天气，炮塔 ATK +20%，冰冻时间 -0.3s |
| **下雨 Rain** | 炮塔 ATK -30%，电塔 ATK +30% + 链击 +1 跳，敌人移速 -5% |
| **下雪 Snow** | 冰塔 ATK +30%，炮塔 ATK +10%，电塔 ATK -10%，敌人移速 -25% |
| **下雾 Fog** | 所有塔射程 -15%，箭塔攻速 -30%，激光塔 ATK +30%，蝙蝠塔无射程惩罚，敌人移速 -10% |
| **夜晚 Night** | 所有塔射程 -20%，蝙蝠塔 ATK +50%，激光塔射程 +10%，敌人移速 +18% |

特殊规则：
- 蝙蝠塔在晴天/下雨/下雪中**休眠**（ATK=0），仅在下雾/夜晚激活（ATK +50%）
- 激光塔在晴天 ATK -30%（强光散射），雨天 ATK +10%

## 8. 地图系统

### 8.1 地图规格

- 设计分辨率：1920×1080（横屏 16:9）
- 场景区域：**21×9 格**（1344×576 px），水平垂直居中
- Tile 大小：64×64 px

### 8.2 Tile 类型

| 类型 | 说明 | 行为 |
|------|------|------|
| `empty` | 空地 | 可拖卡部署（需在路径邻格） |
| `path` | 路径 | 敌人行走，不可部署 |
| `trap_path` | 可埋陷阱的路径 | 视觉同 `path`，但可拖陷阱卡部署 |
| `blocked` | 障碍 | 不可通行不可部署 |
| `spawn` | 敌人出生点 | 位于地图右侧 |
| `crystal` | 水晶位置 | 位于地图左侧 |

建造限制：仅可在路径的 8 方向相邻 1 格内的空地上部署；合法格位视觉高亮（浅绿 + 边框）。

### 8.3 PRNG 多流隔离

| 流 | 用途 | 种子派生 |
|---|------|---------|
| `drafterRandom` | 开局抽卡 | `seed ^ 0xDRAFT` |
| `mapRandom` | 地图/天气/装饰物 | `seed ^ 0xMAP` |
| `waveRandom` | 波次内敌人选择 | `seed ^ 0xWAVE` |
| `dropRandom` | 击杀掉落 | `seed ^ 0xDROP` |
| `shopRandom` | 商店商品池 | `seed ^ 0xSHOP` |
| `mysticRandom` | 秘境事件抽取 | `seed ^ 0xMYSTIC` |

禁用 `Math.random()`；使用确定性 PRNG（mulberry32 / xoshiro128++）；同一 seed 在相同操作序列下结果可复现。

## 9. Run 结算

Run 结束（无论胜负）后弹出结算画面：

| 显示项 | 说明 |
|--------|------|
| 结果 | 胜利 / 失败 |
| 最远到达关卡 | 进度（如 7/9） |
| 总击杀 | 累计敌人击杀数 |
| Run 时长 | 用时 |
| 流派标签 | 自动识别（近战墙流/法术爆发流/生产抗压流/远程压制流/混合） |
| 金币结余 | Run 结束清零，无 meta 转换 |

按钮：「返回主菜单」/ 「立即开始新 Run」。

胜利条件：通关终战（W23）Boss 击败。
失败条件：任一关卡水晶 HP ≤ 0，或玩家主动放弃 Run。
