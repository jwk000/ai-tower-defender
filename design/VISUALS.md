# VISUALS — 视觉表现规范

> 零外部素材 · 100% 程序化几何体 · PixiJS WebGL 渲染 · 1920×1080 横屏

---

## 1. 核心设计原则

| 原则 | 说明 |
|------|------|
| **形状即身份** | 每个单位有独特的复合几何体轮廓，一眼可辨，无需文字标签 |
| **动效即信息** | 攻击动画/待机动画/死亡动画表达单位功能 |
| **色彩即阵营** | 蓝色系=我方，红色系=敌方，黄色/青色=中立 |
| **粒子即反馈** | 命中/死亡/建造/升级用粒子特效传达，不用弹窗文字 |
| **零文字标签** | 场景中不显示单位名称；仅在 UI 面板中按需显示信息 |

### 识别度铁律（P0 硬性要求）

| 测试项 | 标准 |
|--------|------|
| 敌人轮廓识别 | 60px 缩略尺寸下，2 秒内分类 9 种敌人 |
| 我方单位识别 | 40px 缩略尺寸下移动中，2 秒内识别 6 种我方单位 |
| 塔等级识别 | 通过肩章颜色（铜/银/金/白金），1 秒内识别 Lv1–Lv3 |
| 状态识别 | 眩晕/冰冻/嘲讽/燃烧等状态在身上 0.5 秒内可辨 |

**违反铁律 = Bug，必须修复。**

## 2. 渲染层架构

### 2.1 PixiJS Stage 层级（11 层，从下到上）

```
Layer 0   BackgroundLayer   天空渐变 + 远景几何（脏标记触发重绘）
Layer 1   GroundLayer       地图网格、路径、可建区域高亮
Layer 2   DecorationLayer   场景装饰物（树/岩石/冰晶）—— 纯视觉，创建时绘制一次
Layer 3   ShadowLayer       实体地面投影（半透明椭圆）
Layer 4   EntityLayer       所有游戏实体（按 Y 轴排序）+ 头顶 HUD
Layer 5   ProjectileLayer   弹道（箭矢/炮弹/冰晶/闪电/光束/蝙蝠）
Layer 6   EffectLayer       爆炸粒子、命中火花、金币飞行（ParticleContainer）
Layer 7   WeatherLayer      天气粒子（雨/雪/雾/夜光点，独立 ParticleContainer）
Layer 8   ScreenFXLayer     屏幕震动偏移、全屏闪光、晕影暗角
Layer 9   UILayer           HUD、工具栏、选中信息面板
Layer 10  OverlayLayer      暂停遮罩、结算面板、转场黑幕
```

### 2.2 棋盘逻辑层级 vs 渲染 Z-order

棋盘内的**逻辑层级**（空间高度）映射为 EntityLayer 内的渲染 z-order：

| Layer 组件值 | 渲染 z | 典型单位 |
|-------------|--------|----------|
| `BelowGrid` | z=3 | 封印敌人（半透明） |
| `AboveGrid` | z=4 | 地刺陷阱 |
| `Ground` | z=5 | 塔、敌人、我方单位（按 Y 排序） |
| `LowAir` | z=6 | 蝙蝠群、飞行敌（按 Y 排序） |
| `Space` | z=7 | 预留 |

弹道继承来源单位的层级 z。

### 2.3 攻击目标层级约束

| 攻击方 | 可攻击 |
|--------|--------|
| 近战地面单位 | Ground + AboveGrid |
| 远程地面单位 | Ground + AboveGrid + LowAir |
| 低空单位 | Ground + AboveGrid + LowAir |
| 地面陷阱 | AboveGrid + Ground |

## 3. 复合几何体系统

每个单位由三层部件组成：

- **基础部件**（主体形状）：身体、基座、底盘
- **身份部件**（符号特征）：剑、弓、盾、皇冠、翅膀
- **动态部件**（动画变化）：弹道口、翅膀、特效

### 部件类型库

| 部件类型 | PixiJS 绘制方式 |
|----------|----------------|
| `body_rect` | `g.rect()` + fill |
| `body_circle` | `g.circle()` + fill |
| `body_hexagon` | `g.poly()` 6 点 |
| `body_triangle` | `g.poly()` 3 点 |
| `outline` | `g.stroke()` |
| `symbol_sword` | 十字形 rect×2 |
| `symbol_shield` | 半圆弧 + rect |
| `symbol_bow` | arc |
| `symbol_crown` | 3 个三角形 |
| `symbol_wings` | 2 个拉伸三角形 |
| `symbol_halo` | 空心圆 arc+stroke |
| `symbol_crystal` | 六边形 + 内切菱形 |
| `deco_spikes` | N 个三角径向排列 |
| `deco_particles` | N 个小圆环绕 |
| `deco_glow` | 多层半透明圆模拟径向渐变 |

每帧绘制顺序：`g.clear()` → 动态部件 → 基础部件 → 身份部件 → 状态覆盖 → 选中高亮

## 4. 动画状态机

所有实体共用状态机：`SPAWN → IDLE ⇄ ATTACK / HIT → DEATH / SPECIAL`

### 各状态规范

| 状态 | 持续时间 | 关键视觉 |
|------|----------|----------|
| **SPAWN** | 0.3s（Boss 0.5s） | scale 0→1.1→1.0 弹性缓出 + alpha 0→1 + 金色方块粒子上升 |
| **IDLE** | 循环 | 塔：±1px 正弦浮动；我方单位：呼吸摇摆；移动敌人：步行弹跳 ±2px |
| **ATTACK** | 取决于攻速 | 前摇(0~40%)蓄力 → 释放(40~60%)弹道生成 → 后摇(60~100%)恢复 |
| **HIT** | 0.1s | 100ms 闪白 + 微后推 2px + 3~5 粒子飞散 |
| **DEATH** | 0.3s（Boss 更长） | scale 1→0 + alpha 1→0 + 实体颜色粒子爆散；Boss：金色大爆炸 + 全场闪白 |
| **SPECIAL** | 0.3~0.5s | Boss 二阶段：白/红闪烁 + scale 1→1.3 + 全场暂停 0.3s |

### Buff/Debuff 视觉覆盖（与 HIT 叠加）

| 状态 | 视觉修改 | 持续特效 |
|------|----------|----------|
| 减速 | 颜色向 `#4488cc` lerp | 脚下蓝色冰面纹 |
| 冰冻 | 颜色 `#00bcd4` + 外包冰晶轮廓线 | 完全静止（动画暂停） |
| 眩晕 | `#ffd700` 间歇闪烁 | 头顶金色星形旋转 |
| 嘲讽 | 红色描边加粗 2px | 指向嘲讽者的虚线 |

## 5. 单位视觉详细设计

### 5.1 塔（7 种）

| 塔 | 主色 | 核心轮廓 | 待机特效 | 攻击动画 |
|----|------|----------|----------|----------|
| **箭塔** | 蓝 `#4fc3f7` | 高瘦塔身 + 顶部箭头三角 | 塔身 ±1px 浮动 | 塔身后倾→弹回 + 箭头闪蓝 |
| **炮塔** | 橙 `#ff9800` | 矮胖塔身 + 圆炮口 + 铆钉 | 微小左右瞄准漂移 | 炮口后坐 -3px→弹回 + 烟粒子 |
| **冰塔** | 冰蓝 `#80deea` | 六边形冰晶核心缓慢旋转 | 冰晶旋转 + 小冰晶绕转 | 冰晶加速旋转→闪光+冰粒子爆发 |
| **电塔** | 金 `#ffd54f` | 线圈环 + 放电针 | 线圈间歇电弧跳动 | 电弧加速→针尖闪光+闪电生成 |
| **激光塔** | 青 `#4dd0e1` | 晶体柱多层辉光呼吸脉动 | 辉光呼吸 alpha 0.2~0.5 | 晶体亮度至峰→光束→回落 |
| **蝙蝠塔** | 紫 `#7b1fa2` | 蝙蝠翼 + 哥特塔尖 | 休眠：翼收起灰色；激活：翼展开+暗影粒子 | 翼完全展开→蝙蝠影飞出 |
| **导弹塔** | 红 `#e53935` | 大型弹体 24~32px + 橙黄尾焰 | 呼吸灯（CD 完成后红色辉光脉冲 1.2s/周期） | 抛物线弹道→红色冲击波环+蘑菇云+棋盘震动 |

**塔等级肩章（所有塔通用）**：V 形 chevron + 星标，铜色(Lv1) / 银色(Lv2) / 金色(Lv3)

**蝙蝠群成员**（独立飞行单位，LowAir 层）：
- 身体圆 + 左右翅膀三角，深灰 `#2d2d2d`，size 10~14px
- 待机：翅膀拍动（30°~70° 正弦，3~6 Hz）+ 上下浮动 ±5~15px
- 攻击四阶段：锁定蓄力(0~20%) → 俯冲弧线(20~60%) → 撕咬命中(60~75%) → 回弹归巢(75~100%)
  - 俯冲：贝塞尔弧线，arcHeight 15~30px，拖尾暗影粒子
  - 撕咬：獠牙红色三角 + 命中紫色粒子爆散 + 绿色吸血光从目标回流
  - 伤害时机：phase=0.65（撕咬瞬间），不是攻击启动时

## 7. 粒子特效清单

### 战斗反馈

| 特效 | 触发 | 视觉表现 |
|------|------|----------|
| 命中 | 弹道到达目标 | 弹道颜色小圆点飞散 6~10 个 |
| 暴击 | 暴击命中 | 加大粒子 + 轻微白色闪光 |
| 爆炸 | 炮塔/自爆虫 | 红橙圆点 15~25 个 + 快速缩放消失 |
| 死亡 | 单位 HP 归零 | 单位颜色粒子收缩→爆散 8~12 个 |
| Boss 死亡 | Boss HP 归零 | 金色大爆炸 + 全场 0.3s 闪白 + 慢动作 |

### 状态反馈

| 特效 | 视觉表现 |
|------|----------|
| 眩晕 | 头顶黄色星形符号旋转 |
| 减速 | 目标颜色向蓝色渐变 + 脚下蓝色冰面纹 |
| 冰冻 | 目标变亮青色 + 外包冰晶轮廓线 + 完全静止 |
| 嘲讽 | 被嘲讽敌人红色描边变粗 + 指向盾卫的虚线 |
| 受击 | 100ms 纯白 tint（闪白） |
| 飞行无效 | 头顶 ⊘ 红色禁止符号（0.3s alpha 渐隐） |

### 经济反馈

| 特效 | 视觉表现 |
|------|----------|
| +金币 | 金色小圆点从尸体位置飞向 HUD 金币区（贝塞尔曲线） |
| +能量 | 蓝色小圆点飞向 HUD 能量区 |
| 建造完成 | 金色方块粒子从基座上升 |
| 升级完成 | 白色+金色光芒粒子上升 + 建筑短暂放大 |
| 回收 | 灰色方块向下消散 + 金币粒子飞回 HUD |

### 天气粒子

| 天气 | 粒子 | 生成速率 | 速度 |
|------|------|----------|------|
| 晴 | 无持续粒子；背景暖色太阳辉光 | — | — |
| 雨 | 蓝色细长矩形 2×12px | 80~120/s | 600~800 px/s ↓；地面溅射小圆 |
| 雪 | 白色圆 r=2~4 | 30~50/s | 80~150 px/s ↓ + 横向摆动 |
| 雾 | 白色半透明大圆 r=40~80，alpha 0.05~0.1 | 5~8 团 | 30~60 px/s 横向 |
| 夜晚 | 黄白微光点 r=2~3，路径上方静止明灭 | 15~25/s | 静止 |

**天气切换过渡**：旧粒子 alpha 逐帧递减（1.5s 到 0），新粒子递增（1.5s 到 1），全屏色调叠加渐变。

## 8. 场景装饰（动态环境）

> 纯视觉层，不参与碰撞/寻路/伤害，帧预算 ≤ 2ms

### 8.1 静态装饰物（复合几何升级）

| 装饰物 | 几何构成 |
|--------|----------|
| 树 | 棕色矩形（树干）+ 绿色三角（树冠）+ 小三角（层次感） |
| 岩石 | 2~3 个重叠灰色菱形/圆，不同深浅灰 |
| 冰晶 | 中心大菱形 + 上下左右 4 个小菱形，白色渐变 |

## 10. 水晶（基地）视觉

### 辉光呼吸

| HP 区间 | 周期 | alpha 范围 | 辉光半径 |
|---------|------|------------|---------|
| >70% | 3.0s | 0.6~1.0 | 50px |
| 30%~70% | 2.0s | 0.5~0.9 | 40px |
| <30% | 0.8s | 0.3~0.7 | 30px + 脉冲圆环 |

HP<30%：每 0.8s 从水晶中心向外扩散红色描边圆（半径 30→80px，alpha 1→0，2px 描边）。

### 悬浮动画

- 本体浮动：`y = baseY - 6 * sin(t/2.4 * 2π)`，周期 2.4s，幅度 ±6px
- 环绕碎片：公转半径 35px，周期 4.0s，角速度恒定

### 攻击光束（秒杀触发）

4 阶段共 0.6s：蓄力(100ms，辉光 alpha 强制 1.0) → 射出(150ms，细线+模糊宽光束) → 命中(200ms，红色菱形粒子 8~12 个+目标销毁) → 回弹(150ms，本体 scale 0.9→1.1→1.0)

Boss 若 `immune_to_crystal_kill === true` 则整套特效不触发。

### 死亡碎裂时序

```
0.0s   HP=0；水晶剧烈震颤；辉光红白闪烁（每 50ms 切换）
0.3s   本体替换为 12~18 个碎片容器；底座符文熄灭
0.3-2.0s 碎片飞散渐隐（angular_velocity ±360°/s）
2.0s   碎片销毁；进入 Run 失败结算
```
