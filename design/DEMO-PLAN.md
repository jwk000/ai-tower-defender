# Demo 开发计划

> 状态：进行中
> 目标：把当前可玩原型推进到可展示、可试玩、可持续迭代的 Demo 版本。
> 执行规则：后续开发按本清单推进；每完成一个逻辑任务，必须立即在本文档对应条目标记完成，并按 AGENTS.md 要求立刻提交代码。
> 更新依据：已按 `design/*.md` 最新设计与当前代码现状对齐（2026-05-19）。本文聚焦**设计目标 vs 当前落地差距**，已完成项仅保留确实在代码中落地的内容。

---

## 0. 里程碑定义

### Demo 验收目标

- 一局时长约 20~40 分钟
- 有 1 条完整可通关路线
- 有 1 个正式 Boss
- 有 2~3 个可感知流派
- 能让试玩者明确感受到“卡牌 Roguelike 塔防”的特色
- 关内节奏符合 v3.6：手牌上限 4、每 5 秒手动抽 1 张、单次可重抽 1 次、水晶能量 2 秒回 1、场上塔总量预期约 20 个

### 当前实现基线（代码现状）

- 已有 8 关 YAML 与关卡地图面板，终点节点已作为 `boss` 节点接入
- 已有普通战 / 精英战 / 商店 / 神秘事件 / 宝箱 / 休整 / Boss 的路线节点类型与基础跳转
- 已有关后三段奖励闭环：卡牌三选一 → 金币三选一 → 卡牌升级三选一
- 已有商店、神秘事件、卡池面板、卡牌升级、Run 存档恢复基础闭环
- 仍大量保留旧版本结构：技能点 `sp` / `skilltree`、旧塔名 `elemental_tower`、起始牌组含重复卡、存档仍保存进行中 Run
- 内容与文档不一致处较多，本文默认这些都属于待收敛项

### 标记规则

- `[ ]` 未开始
- `[-]` 进行中
- `[x]` 已完成
- 完成任务时：
  1. 先完成代码、测试、验证
  2. 立刻把本文件对应项改为 `[x]`
  3. 如有必要补充简短备注（日期/影响范围）
  4. 立刻提交 Git commit
