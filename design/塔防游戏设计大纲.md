# 塔防游戏设计大纲

## 核心玩法
这是一个卡牌+塔防+rougelike游戏。
玩家的游戏目的是守护我方水晶，地图上每隔一段时间会出现一波敌人，敌人沿着固定路线从出生点向我方水晶移动，进入我方水晶位置后对水晶造成敌人剩余血量的伤害。
水晶血量为0死亡，玩家游戏失败。所有波次的敌人都死亡则玩家游戏胜利。
玩家进入关卡后获得4张手牌，放入手牌区域，玩家可以把手牌拖放到场景格子上变成真实的单位。
每个波次会出现一个精英怪，击杀精英怪可以获得一次3选一的抽卡机会，选择的卡放入手牌区域，手牌区域共4个卡槽，如果卡槽满了抽卡之后无法获得卡牌。
玩家单位击杀敌人可以获得金币，金币可以给塔升级，塔满级后无法继续升级。
塔可以卖掉换成金币，但只能回收造价的50%金币。

## 关卡
玩家进入游戏看到5个关卡，前面的关卡通关可以解锁后面的关卡。每个关卡有不同的主题不同的敌人。每个关卡有5~12个波次，最后一波出现boss。

### 1.关卡主题和敌人

- 绿野仙踪。场景是绿色草原，天气是雨天，敌人是怪兽族。哥布林（普通速度）、疯狂野猪（快速）、铁甲大象（慢速高血低攻）、草原巨人（慢速高血高攻），boss是巨型史莱姆（血量高攻击低），死亡后分裂成2个中型史莱姆（血量中攻击中），每个中型史莱姆死亡分裂成2个小型史莱姆（血量低攻击高）。共5个波次敌人，第5波出现boss。
- 沙漠虫潮。场景是黄色沙漠，天气是晴天，敌人是虫族。沙漠黑虫（速度慢数量多血量低）、钻地甲虫（每隔3秒钻到地下2秒，钻地后移动速度提升）、吸血蝗虫（低空单位，速度慢，数量多，血量低，攻击中）、自爆甲虫（会寻找塔发动自爆攻击，伤害高，数量少，血量低，不能被嘲讽），boss是虫族女王，技能是塔不能攻击它，只能用士兵单位或技能攻击。共6个波次，第6波出现boss。
- 黑暗古堡。场景是森林古堡，天气是夜晚，敌人是黑暗族。狼人、吸血蝙蝠、巫师、牧师（复活）、弗兰肯斯坦。boss是路西法，可以召唤骷髅大军。共7个波次。
- 末日废土。场景是废弃的城市，天气是红雾：巨大红色夕阳在天空右上角，像一只眼睛；暗红色云/火山灰快速移动；远方黑色山脉山顶有红色闪光。敌人是机器人。飞机（低空单位）、坦克（高攻高防低速）、油罐车（死亡会爆炸）、机器狗（快速低攻低血）、巨型机器人（慢速高攻高防）、无人机（快低空单位，快速低攻低血）。boss是超级机器人，每隔10秒发射远程导弹攻击任意位置造成大范围伤害（攻击位置按评分规则计算）。共8个波次。
- 深渊裂隙。场景是黑暗地下，天气是夜晚，敌人是之前所有关卡出现的敌人混合出现，包括之前关卡的boss，多个出生点。boss是深渊领主，血量高，移动速度慢，每隔5秒释放一次黑暗吞噬技能，周围的所有单位（塔、敌人、士兵等一切单位）会被吞噬（死亡并从场景中移除）每吞噬1个单位恢复2%血量。共12个波次。

| 属性     | 值                                                             |
| -------- | -------------------------------------------------------------- |
| **形状** | 简化的 V 形（2 条短线相交于中心）+ 小三角头                    |
| **大小** | 8-14px 翼展                                                    |
| **颜色** | `#ffffff`（白色，所有主题统一）                                |
| **数量** | 2-5 只/场景，随机                                              |
| **运动** | 沿贝塞尔曲线飞行路径，翅膀正弦拍动（频率 3-5 Hz）              |
| **行为** | 循环路径飞行：从左到右 → 飞出后从另一侧重新进入 → 随机改变高度 |
| **层**   | 在实体层之上（LayerVal.LowAir）                                |

**动画分解**：

```
翅膀拍动周期（1 个完整周期 = 2 次拍动/秒）：
  帧 0-25%:  翅膀展开（↑）→ 身体微升 +2px
  帧 25-50%: 翅膀收起（↓）→ 身体微降 -2px
  帧 50-75%: 翅膀展开（↑）→ 身体微升 +2px
  帧 75-100%:翅膀收起（↓）→ 身体微降 -2px

形状变化：翅膀角度从 30°（收起）到 120°（展开）正弦循环
```


## 层级系统

> 逻辑层级（棋盘空间）与表现层级（场景视觉）的映射关系、渲染排序规则、攻击目标层级约束

## 2. 两层体系：逻辑层级 vs 表现层级

两者是**不同维度**的概念，不能混淆：

| 维度             | 逻辑层级                            | 表现层级           |
| ---------------- | ----------------------------------- | ------------------ |
| **所在域**       | 棋盘内的空间高度                    | 场景的视觉远近     |
| **作用**         | 决定攻击目标可达性、碰撞            | 决定视觉遮挡关系   |
| **用户看到的是** | 单位在棋盘上的空间位置（地面/低空） | 由远到近的渲染顺序 |
| **类比**         | Z 轴（上下）                        | Z-index（前后）    |

```
┌────────────────────────────────────────────────────────────────┐
│                        表现层级（场景视觉）                       │
│                                                                  │
│  远景背景 ───────────────────────────────────── 最远             │
│  中景背景 ─────────────────────────────────────                  │
│  ╔══════════════════════════════════════════╗                    │
│  ║           棋盘层 (Board Layer)           ║                    │
│  ║  ┌──────────────────────────────────┐   ║                    │
│  ║  │    逻辑层级（空间高度）            │   ║                    │
│  ║  │                                  │   ║                    │
│  ║  │  高空层  Space       ─ 最高       │   ║                    │
│  ║  │  低空层  LowAir      ─ 飞行单位   │   ║                    │
│  ║  │  地面层  Ground      ─ 行走单位   │   ║                    │
│  ║  │  地格层  Grid        ─ 机关陷阱   │   ║                    │
│  ║  │  深渊层  Abyss       ─ 最低       │   ║                    │
│  ║  └──────────────────────────────────┘   ║                    │
│  ╚══════════════════════════════════════════╝                    │
│  近景层（UI/特效叠加）───────────────────── 最近                  │
│                                                                  │
└────────────────────────────────────────────────────────────────┘
```

**关键认知**：逻辑层级只存在于棋盘内。对于整个场景来说，棋盘是一个平面，逻辑层级通过**棋盘内的渲染 z-order**来表达——高层级单位画在低层级单位之上。


### 3.1 棋盘外部层（不参与游戏逻辑）

| 层号 | 名称              | 说明                  | 渲染策略       |
| ---- | ----------------- | --------------------- | -------------- |
| L0   | `BackgroundLayer` | 天空渐变 + 远景几何   | 脏标记触发重绘 |
| L8   | `ScreenFXLayer`   | 屏幕震动、闪白、暗角  | 覆盖全屏       |
| L9   | `UILayer`         | HUD、工具栏、信息面板 | 覆盖棋盘上方   |
| L10  | `OverlayLayer`    | 暂停遮罩、结算面板    | 最顶层         |

### 3.2 棋盘内部层（游戏逻辑发生的区域）

棋盘内部 = 从地图瓦片到弹道特效，所有层在一个平面对齐。

```
棋盘渲染 z-order（从后到前 / 从远到近）：

  z=0   GroundLayer         地图网格瓦片 + 主题色
  z=1   ShadowLayer         实体地面投影（半透明椭圆）
  z=2   Grid                地格层 —— 地面机关陷阱（地刺等）
  z=3   Ground              地面层 —— 行走单位 + 塔 + 建筑 +场景装饰物（树/岩石/冰晶）（按 Y 排序）
  z=4   LowAir              低空层 —— 飞行单位（按 Y 排序）
  z=5   Space               高空层（预留，当前无单位）
  z=6   EffectLayer         粒子特效（爆炸/命中/建造/金币飞行）
  z=7   WeatherLayer        天气粒子（雨/雪/雾）
```

## 卡牌系统
卡牌分类：单位卡、技能卡、奥术卡。

单位卡：可以拖放到场景中变成单位，包括所有塔、所有士兵、所有机关。具体内容参考单位系统。
拖放单位的时候可以预览攻击范围，单位可以放置的格子显示绿色，不能放置的格子显示红色。
每个格子可以放置一个地格层单位、一个地面层单位、一个低空层单位，如果格子所在的层已经放置了单位，无法放置新的单位。

技能卡：可以拖放到场景中释放技能，技能和场景有关，拖放的时候可以预览攻击范围。
- 火球术。对2x2区域的格子造成少量伤害；
- 剑雨。对3x3区域的格子造成伤害。
- 暴风雪。对3x3区域的格子造成伤害，持续5秒；
- 炸弹。对2x2区域的格子造成大量伤害，放置后2秒爆炸；

奥术卡：抽卡选择奥术卡后立即生效，不放入手牌。
- 紧急防护（水晶10秒内无敌）
- 提升箭塔伤害
- 提升盾卫血量



卡牌上的信息应该包括：名字、图标、等级、阵营、技能描述、攻防血的描述（不需要具体数值，但要说明数值特点）

## 单位系统

塔、兵、机关、敌人都是单位，它们的本质相同——都有一组属性、一组行为规则、一组视觉表现。
所有单位通过配置驱动逻辑，逻辑和表现都是可以替换的组件，不同单位只是配置不同。
单位只能有一个技能，减少理解成本和实现成本。

 单位属性分类

| 分类        | 说明                                               | 谁有               |
| ----------- | ------------------------------------------------- | ------------------ |
| **生存属性** | HP、护甲、魔抗                                     | 所有单位           |
| **战斗属性** | ATK、攻速、射程、伤害类型                           | 有攻击能力的单位   |
| **机动属性** | 移速、移动范围                                     | 可移动单位 |
| **经济属性** | 造价、回收价、升级费                                | 玩家可部署的单位   |
| **视觉属性** | 形状轮廓、颜色、尺寸、符号特征、待机动画、弹道样式     | 所有单位           |
| **分类属性** | 所属阵营、类别、层级                                | 所有单位           |

### 塔类单位：

箭塔（攻击低频率高）升级路径：普通箭塔（单发） | 双重箭塔（同时射出 2 支箭，可锁不同目标） | 三重箭塔（同时射出 3 支箭）
弩塔（攻击高频率低）升级路径：普通弩塔（箭头发出后直线攻击所有路径上的敌人）|高级弩塔（攻击提升）|超级弩塔（攻击提升频率增加）
炮塔（AOE）升级路径： 普通炮塔（AOE 范围伤害） | 重炮塔（AOE + 概率眩晕命中目标） | 超级炮塔（AOE + 眩晕 + 击退）
激光塔（单体持续高伤）升级路径：稳压激光塔（在攻击范围内持续锁单体，不切换目标） | 充能激光塔（蓄能机制：**持续锁同一目标越久伤害越高**，目标离开攻击范围 / 死亡后清零）
蝙蝠塔（召唤）升级路径：普通蝙蝠塔（3 蝙蝠） | 中级蝙蝠塔（4 蝙蝠 + ATK↑） | 高级蝙蝠塔（5 蝙蝠 + ATK↑） 
导弹塔（大范围伤害）升级路径：普通导弹塔（单发） | 双联导弹塔（同一地格同时发射 2 个导弹，单发伤害↓，总伤害↑） | 集束导弹塔（同一地格同时发射 3 个导弹，单发伤害再↓，总伤害再↑）
冰塔（减速）升级路径：冰塔（命中减速） | 冰冻塔（命中按概率触发"冰冻"硬控） | 霜冻塔（冰冻 + 命中目标攻击力 Debuff）
火塔（灼烧）升级路径：火塔（命中附加灼烧 DOT） | 真火塔（灼烧 + 低概率直接击杀非boss单位）
毒塔（中毒）升级路径：毒塔（命中附加中毒 DOT） | 病毒塔（中毒 + **传染**：跳数 ≤3、半径 ≤80px、每跳伤害衰减 50%） 
电塔（连锁攻击）升级路径：普通电塔（1 次弹跳） | 热电塔（2 次弹跳） | 核电塔（3 次弹跳） | 闪电塔（3 次弹跳 + **概率触发"全屏闪电"**：随机攻击全图敌人一次，CD ≥10s，单次伤害 ≈ 1.5× 普攻） 

### 士兵单位：

盾卫（高血高防低攻击带嘲讽）
弓手（远程低血高攻）
法师（远程低血AOE技能）
牧师（远程低血治疗技能）

### 机关单位：

地刺：放在敌人前进路径上，位于地格层，每秒对地面层单位造成少量伤害
捕兽夹：放在敌人前进路径上，位于地格层，地面单位会被陷阱困住1秒，对boss无效
焦油坑：放在敌人前进路径上，位于地格层，地面单位移动速度降低20%
巨石：放在敌人前进路径上，位于地面层，会阻挡地面单位前进，可以被攻击，死亡后消失，敌人可以继续前进；
风扇：放在路径两侧地格，位于地面层，风扇前方3格的单位会减速20%；可以被敌人攻击；
水坑：放在路径两侧地格，位于地格层，敌人掉入水坑后50%概率会死亡，如果敌人没有死亡就自动寻路回到路径；
拳击套：放在路径两侧地格，位于地面层，有朝向，可以把前面1格的敌人朝前方推动1格距离，敌人会自动寻路回到路径继续前进；
机械臂：放在路径两侧地格，位于地面层，有朝向，可以把前面2格的敌人朝后方拉近1格距离，敌人会自动寻路回到路径继续前进；

### 单位的AI

士兵以放置点为圆心有移动距离，士兵自身为圆心有攻击距离。
我方士兵AI：在移动范围内随机游荡 -> 移动范围内发现敌人 → 追击敌人 → 进入攻击距离后攻击；
敌人士兵AI：如果被我方士兵攻击，需要停止移动攻击我方士兵；

### 单位的阵营

所有单位分为4个阵营：正义、邪恶、混乱、中立。玩家是正义阵营，敌人是邪恶阵营，场景设计可能有巨龙之类的单位是混乱阵营，场景中可能有生命之泉之类的中立单位是中立阵营。
正义或者邪恶阵营内部的单位互为友方，可以治疗不能攻击。
正义、邪恶、混乱三个阵营互相视为敌方，可以攻击。
所有单位不能攻击中立阵营的单位，中立阵营的单位把所有阵营的单位视为友方。
混乱阵营的单位可以攻击任何非中立阵营的单位，包括混乱阵营的单位。boss路西法是混乱阵营。


## buff系统

除了技能，大部分效果都是buff。
buff通常有时间限制，相同buff可以叠层，buff效果可以叠加。
冰冻、中毒、灼烧、加速、眩晕、治疗等效果都是buff。
数值变化的效果也都是通过buff实现。
以后可能会增加场景词条、遗物词条，本质上都是buff。
