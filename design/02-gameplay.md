## 1. 地图与网格

### 1.1 网格规格

| 参数 | 值 |
|------|-----|
| 设计分辨率 | 1920×1080（横屏16:9） |
| 场景区域 | 21列×9行 |
| 单格大小 | 64×64px |
| 场景总尺寸 | 1344×576px |
| 场景位置 | 水平居中（offsetX=288），垂直居中 |

### 1.2 格子类型

| 类型 | 说明 | 可否部署单位 | 可否通行 |
|------|------|:---:|:---:|
| `empty` | 空地 | ✅ | ✅ |
| `path` | 敌人行走路径 | ❌ | ✅（仅敌人） |
| `blocked` | 障碍物 | ❌ | ❌ |
| `spawn` | 敌人出生点 | ❌ | ✅ |
| `crystal` | 水晶位置（1格） | ❌ | ❌ |

### 1.3 水晶

水晶是玩家要守护的核心目标，位于地图左侧。

| 属性 | 说明 |
|------|------|
| 位置 | 地图左侧固定1格，每关地图定义 |
| HP | 关卡设定（约500-2000，随关卡递增） |
| 受损 | 敌人到达水晶位置后，按该敌人 ATK 对水晶造成伤害；所有敌人 ATK 至少为 1 |
| 结算次数 | 普通敌人每次进入水晶只结算一次伤害，随后播放进入水晶动画并销毁 |
| 归零 | 关卡失败 |
| Boss到达 | 任意 Boss 进入水晶后立即判定关卡失败 |

> 水晶在场景中显示为悬浮的红色菱形几何体，带有呼吸辉光动画。详见 [05-presentation §2.3](./05-presentation.md#23-水晶视觉)

### 1.4 建造限制

- 塔只能放置在 `empty` 格上，且必须位于路径**相邻1格**内的空地（8方向）
- 士兵只能放置在 `path` 格上，并且永远无视地格占用；同一格可叠放任意数量士兵，也可与机关同格
- 机关只能放置在 `path` 格上；每个格子最多1个机关，机关死亡或被移除后释放该格机关占用
- 塔占用地面层（Ground），塔死亡或被移除后释放该格塔位占用
- 装饰物占用地面格，塔不能放置在装饰物所在格
- 地格层（Grid）单位：每格最多1个（地刺、捕兽夹、焦油坑、水坑）
- 地面层（Ground）单位：每格最多1个（塔、巨石、风扇、拳击套、机械臂、装饰物）
- 低空层（LowAir）单位：每格最多1个（仅飞行单位）
- **同一格可以同时有地格层+地面层+低空层各1个单位**
- 拖卡时：可放置格显示绿色高亮，不可放置格显示红色

### 1.5 路径

- 每关有1条或多条固定路径（由 `path` 格连通）
- 敌人沿路径从 `spawn` 向 `crystal` 移动
- 路径在关卡配置中以格子坐标序列定义
- 如果敌人被推离/拉离路径（拳击套/机械臂），会自动寻路回到最近的路径格
- 敌人与士兵不发生位置碰撞或挤开，可以停留在同一坐标；攻击、受击、寻路回归仍按位置距离独立判定
- 敌人图片的左右朝向必须与当前路径段的水平前进方向一致；当敌人切换到向左/向右的新路径段时，渲染朝向必须同步翻转。纯竖直移动段不改变既有左右朝向。

## 2. 波次系统

### 2.1 基本规则

| 参数 | 说明 |
|------|------|
| 每关波数 | 5-12波（见 [04-levels](./04-levels.md) 每关定义） |
| 最后一波 | 必为BOSS波（BOSS+少量随从） |
| 波间间隔 | 上一波全部敌人死亡后2秒自动开始下一波 |
| 触发方式 | 自动，无需玩家手动启动 |
| 每波精英 | 每波包含1个精英怪（属性强化的普通敌人） |
| 胜利判定 | 最后一波全部出怪完成后，必须场上所有敌人死亡才算关卡胜利 |

调试面板必须提供“直接进入最后一波”按钮，仅战斗中可用。点击后当前关卡立刻切换到最后一波并按正常波次流程开始出怪，用于测试 Boss 技能；该调试行为不发放被跳过波次的奖励，也不自动清理场上已有单位。

调试面板“查看全部卡牌”仅战斗中可用，用于快速试用任意卡牌。打开列表后每点击一张卡牌，都按从左到右的顺序替换当前手牌槽位；替换到最右侧后下一次点击重新从最左侧开始。该调试行为只改变手牌内容，不改变金币、波次、卡池解锁或关卡进度。

### 2.2 波次构成

每波由以下内容组成：

```
波次配置 = {
  敌人编组: [{敌人ID, 数量, 出生间隔(秒)}],  // 多个编组可叠加
  精英怪: {敌人ID（精英版）, 属性倍率},
  波次奖励: 金币数量
}
```

- 同波内多个敌人编组可以按不同出生间隔并发产出
- 敌人出生间隔控制密度节奏
- 波内敌人数量随波次递增

### 2.3 难度曲线

关卡内采用4阶段难度递增：

| 阶段 | 波次比例 | HP倍率 | ATK倍率 | 描述 |
|------|---------|--------|---------|------|
| 开局 | 前20%波 | 0.8× | 0.8× | 让玩家有时间布局 |
| 建设 | 20%-50%波 | 1.0× | 1.0× | 标准难度 |
| 压力 | 50%-80%波 | 1.3× | 1.3× | 考验防线质量 |
| 高潮 | 最后20%波 | 1.5× | 1.5× | BOSS战前压力 |

关卡间也有递增（第2关敌人基础属性高于第1关）。具体数值见 [04-levels](./04-levels.md)。

### 2.4 精英怪

| 属性 | 说明 |
|------|------|
| 外观 | 同普通敌人+体型增大20%，名称显示为黄色 |
| 属性 | 普通版的2.0倍HP、1.5倍ATK |
| 标识 | 头顶金色精英标记 |
| 奖励 | 击杀后触发3选1抽卡 |
| 每波数量 | 1个 |

## 3. 卡牌系统

### 3.1 卡牌分类

| 类型 | 使用方式 | 是否进手牌 | 是否进战场 |
|------|---------|:---:|:---:|
| **单位卡** | 拖到场景空格 → 生成单位 | ✅ | ✅ |
| **技能卡** | 拖到场景目标区域或点击自施法 → 立即释放技能 | ✅ | ❌（效果后消失） |

> 详细卡牌清单见 [03-units §7](./03-units.md#7-卡牌完整列表)

### 3.2 手牌区

| 参数 | 值 |
|------|-----|
| 手牌上限 | 5张（v5.0 固定，出牌后自动补满） |
| 初始手牌 | 开局从已解锁卡池随机抽5张 |
| 补牌方式 | 出牌后自动从卡池随机补1张（手牌始终满） |
| 卡池随机权重 | 每局每张卡初始权重20；任何成功抽入手牌的卡会使自身权重降低1点，最低降到5。后续随机补牌按当前权重加权抽取，降低已频繁出现卡牌的重复概率 |
| 连续抽牌限制 | 随机补牌时不能连续两次抽到同一张卡，出牌后补牌也不能立刻补回刚打出的同一张卡；若当前只有被限制卡是唯一合法候选，则允许兜底抽取 |
| 抽牌限制 | 兵、塔、法术三类各自在当前手牌中最多出现2张；同一张卡仍最多出现2张。开局抽牌、手动抽牌和出牌后补牌都必须遵守 |
| 使用后 | 消耗对应金币 → 单位卡场上产生单位实体；技能卡释放效果；卡牌自动补牌 |
| UI位置 | 场景底部居中，水平排列 |

### 3.3 卡牌金币费用

v5.0 起，每张卡牌出牌需消耗金币。不同卡牌费用不同，按强度分级：

| 费用档 | 金币 | 典型卡牌 |
|---|---|---|
| T1 廉价 | 30-40 | 地刺、焦油坑、箭塔、冰塔、火球术 |
| T2 标准 | 45-65 | 剑士、弓手、捕兽夹、毒塔、火塔、法师、牧师、巨石 |
| T3 高级 | 70-100 | 电塔、炮塔、蝙蝠塔、激光塔、弩塔 |
| T4 战略 | 120-150 | 导弹塔 |

> 详细费用见 [03-units §7](./03-units.md#7-卡牌完整列表) 和 YAML 配置 `src/config/cards/`。

| 信息 | 说明 |

| 名称 | 单位/技能名 |

## 4. 经济系统

### 4.1 金币来源

| 来源 | 数额 |
|------|------|
| 击杀普通敌人 | 敌人配置的 `killReward` |
| 击杀精英怪 | 2×普通版 `killReward` |
| 击杀BOSS | 固定大额奖励（关卡配置） |
| 波次通关 | 波次配置的奖励金币 |
| 金手指按钮 | 战斗 HUD 临时调试入口，点击随机增加1-2金币 |

### 4.2 金币用途

| 用途 | 说明 |
|------|------|
| **出牌** | v5.0 新增：打出卡牌消耗金币（不同卡牌费用不同，见 §3.3） |
| 升级塔 | 花费金币提升塔等级（每座塔独立升级） |
| 升级士兵 | 花费金币提升士兵等级 |
| （塔/士兵满级后不可再升级） | |

> v5.0 金币用途扩展为：出牌 + 升级。出牌费用是主要金币消耗渠道。

### 4.3 塔升级费用

| 塔类型 | L1→L2 | L2→L3 | L3→L4（如有） |
|--------|--------|--------|--------------|
| 各塔费用详见 [03-units](./03-units.md) 每座塔的升级路径表 | | | |

### 4.4 塔出售

- 任何时刻可以出售塔
- 回收金币 = 该塔累计投入金币 × 50%
- 出售后塔消失，格子恢复为空格
- 出售塔上的升级金币也按50%回收

### 4.5 金币不跨关

- 金币仅在本关内有效
- 通关后金币不保留
- 清除设计意图：避免"前期攒金币后期碾压"的滚雪球问题

## 5. 战斗系统

### 5.1 伤害公式

```
实际伤害 = 攻击力ATK × (1 - 伤害减免率)
```

- **物理伤害**：伤害减免率 = 护甲 / (护甲 + 100)
- **魔法伤害**：伤害减免率 = 魔抗 / (魔抗 + 100)
- **真实伤害**：无视护甲和魔抗，伤害减免率 = 0

### 5.2 攻击流程

```
单位每 attackSpeed 秒执行一次攻击：
  1. 选择目标（按目标选择规则）
  2. 检查目标是否在射程内
  3. 发射弹道/立即造成伤害
  4. 目标HP -= 实际伤害
  5. 目标HP ≤ 0 → 触发 onDeath → 死亡
```

### 5.3 攻击速度

- `attackSpeed` = 每秒攻击次数
- 攻击间隔 = 1 / attackSpeed 秒
- 示例：attackSpeed=2 → 每0.5秒攻击一次
- 攻速可被Buff修改

### 5.4 伤害类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `physical` | 物理伤害，受护甲减免 | 箭塔、弩塔、炮塔 |
| `magic` | 魔法伤害，受魔抗减免 | 激光塔、冰塔、火塔、毒塔、电塔、法师 |
| `true` | 真实伤害，无视护甲魔抗 | 导弹塔（部分）、秒杀技能 |

### 5.5 范围伤害（AOE）

- AOE伤害对范围内的所有合法目标造成伤害
- 主目标：100%伤害
- 范围内其他目标：该单位AOE系数 × 100%伤害

### 6.1 四大阵营

| 阵营 | 成员 | 敌对关系 |
|------|------|---------|
| **正义** | 玩家部署的所有单位 | 邪恶+混乱 |

## 7. AI与目标选择

### 7.1 我方单位AI

#### 塔

- 固定位置，不移动
- 在攻击范围内自动选择目标攻击
- 默认目标选择：**最近路径进度**的敌人（走在最前面的）

#### 士兵

四状态AI模型：

```
IDLE（游荡）
  │ 移动范围内随机走动
  ▼
ALERT（警戒）
  │ 发现敌人（进入alert_range）
  ▼
CHASE（追击）
  │ 向目标移动
  ▼
COMBAT（战斗）
  │ 进入攻击距离 → 攻击
  │ 超出移动范围 → 返回IDLE
  │ 目标死亡 → 返回ALERT
```

| 士兵类型 | alert_range | attack_range | 移动范围 |
|---------|-------------|-------------|---------|
| 盾卫 | 80px | 40px（近战） | 120px |
| 弓手 | 250px | 200px | 100px |
| 法师 | 200px | 150px | 80px |
| 牧师 | 200px | 150px（治疗） | 80px |

> 所有具备攻击能力的单位，运行时有效攻击距离不得低于当前关卡 1 格（`map.tileSize`）。配置表可保留低于 1 格的近战表现数值，但战斗寻敌、进入战斗和出手判定必须按至少 1 格执行。

- 士兵移动范围以放置点为圆心
- 士兵与士兵、士兵与敌人不发生位置碰撞或挤开，可以重叠站位
- 多士兵不扎堆攻击同一弱敌（优先分配未受攻击的目标）

#### 召唤物（蝙蝠塔的蝙蝠）

- 在塔周围飞行游荡
- 自动攻击范围内最近的敌人
- 蝙蝠HP=0时死亡，塔重新召唤（有冷却）

### 7.2 敌方AI

#### 普通敌人

- 沿路径向水晶移动
- 如果被攻击且有攻击能力：停下反击（优先攻击正在攻击自己的单位）
- 如果路径被巨石阻挡：攻击巨石直到破坏；近战敌人与巨石碰撞接触时，即使中心点距离略大于攻击范围，也必须把巨石视为可攻击目标
- 如果被推离路径：自动寻路回到路径

#### 特殊敌人

| 敌人 | 特殊AI |
|------|--------|
| 钻地甲虫 | 每隔3秒钻入地下，潜地后进入 BelowGrid 层并沿路径前进3格再钻出；潜地期间攻击能否命中完全遵循棋盘层级限制，地面只显示随其移动的翻土尘土特效 |
| 吸血蝗虫 | 低空飞行，沿路径前进 |
| 自爆甲虫 | 检测攻击范围内的塔 → 向其移动并自爆（不能被嘲讽），否则沿路径前进 |
| 巫师 | 攻击+可复活周围死亡敌人 |
| 飞机/无人机 | 低空飞行单位 |

## 8. Buff系统

### 8.1 Buff概念

Buff是可以挂载在单位上的临时效果，几乎所有非直接伤害/治疗的效果都通过Buff实现。

| Buff类型 | 示例 |
|----------|------|
| DOT（持续伤害） | 灼烧、中毒 |
| 控制 | 冰冻、眩晕、减速 |
| 属性修改 | 攻击力↑、移速↑、攻速↓ |
| 特殊状态 | 嘲讽、无敌 |

### 8.2 Buff规则

| 规则 | 说明 |
|------|------|
| 时效 | Buff通常有时间限制（如"持续5秒"） |
| 叠层 | 同类型Buff可叠层（最多3层） |
| 独立计算 | 不同来源的Buff独立计算、同时生效 |
| 单位死亡 | 身上的Buff全部清除 |
| 优先级 | 眩晕 > 嘲讽 > 减速 > DOT > 增益 |
| 嘲讽 | 敌人仅在嘲讽来源处于自身攻击范围内时停止路径移动并锁定攻击嘲讽来源；嘲讽来源死亡或离开可维持条件后效果消失；盾卫嘲讽为自动光环，同一盾卫同一时间只维持 1 个敌人 |

### 8.3 Buff来源

- 塔的攻击效果（冰塔→减速、火塔→灼烧、毒塔→中毒）
- 士兵技能（牧师→治疗Buff）
- 技能卡（暴风雪→全屏减速、炸弹→伤害）
- 技能卡（暴风雪→全屏减速、炸弹→伤害、淘金热→金币）
- 场景效果（焦油坑→减速）

### 8.4 Buff系统实现方式

Buff在ECS中以 `Buff` 组件 + `BuffInstance` 实体实现。系统管线中的 `PHASE_MODIFIERS` 阶段统一处理Buff的添加/更新/过期/移除。详见 [06-technical §4](./06-technical.md#4-系统管线)。
