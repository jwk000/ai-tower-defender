# Tower Defender — 关卡设计

> 本文档是 Run 长征 8 关（关 1–8）+ 终战主题、敌人阵容、数值签名、关卡蓝图的权威。
> 具体数值（HP/伤害/速度数字）以 [NUMBERS.md](./NUMBERS.md) 为准。
> 关卡 YAML 配置的落地以本文档为准。

---

## 1. 设计原则

### 1.1 三条铁律

1. **8 关主题、敌人机制、数值体验三者全部不重复**——拒绝"换皮关"
2. **每关只引入 1 个新敌人 + 1 个新机制**（KR 黄金法则）——避免认知过载
3. **每关有自己的 signature flavor**（5 大波次类型选 1 主导，60–70% 波次符合主导 flavor）

### 1.2 五大波次 Flavor

| Flavor | 特征 | 考验玩家能力 |
|--------|------|------------|
| **A. RUSH（潮涌）** | 高数量/低HP/高速/单类型/紧密间隔 | AoE 覆盖，持续输出 |
| **B. SIEGE（攻坚）** | 低数量/极高HP/低速/带护甲 | 单体爆发，反甲，优先级管理 |
| **C. SWARM（饱和）** | 极高数量/低HP/多类型/多路/宽间隔 | 全图 DPS，战场覆盖率 |
| **D. ELITE-SPIKE（精英突袭）** | 1 个高HP目标 + 5-10 个护卫 | 目标优先级，微操 |
| **E. GAUNTLET（车轮战）** | 长波/多子组/3+类型/无停歇 | 经济管理，卡池灵活度 |

修饰子：F. SUMMONER（敌人会生敌人）/ G. BOSS（单一支配性 Boss）

### 1.3 8 关 Flavor 序列

```
关1: A (RUSH)     — 教学，建立核心循环
关2: C (SWARM)    — 钻地+空袭，与 RUSH 的"覆盖压力"区分
关3: B (SIEGE)    — 护甲战，节奏放缓，喘息空间
关4: D (ELITE)    — 护卫+精英，强制优先级管理
关5: A→C 切换     — 多类型海陆空乱斗，节奏复合
关6: E (GAUNTLET) — 车轮战，经济/卡池成型检验
关7: F (SUMMONER) — 产卵者递归，杀根才有效
关8: G (BOSS)     — 多形态终战 Boss，所有 flavor 浓缩
```

### 1.4 Boss 出场节奏

- 关 1–3：无独立 Boss（关末出强化波），让玩家建立卡池与节奏
- 关 4：首个 Mini-Boss（单形态，可用既有工具击败）
- 关 5–7：正式 Boss（二形态，每形态反制一种卡池流派）
- 关 8：终战 Boss（三形态，依次 invalidate 三种最强流派）

## 3. 各关详细设计

### 关 1 — 🌅 边境绿野（Verdant Marches）

**一句话定调**：王国边境的青翠平原，远处雾中藏着被苔藓吞噬的废弃古墓。

**场景**：暖黄绿草地 + 浅棕土路 + 远处紫雾古墓彩蛋；摇曳麦穗、风车、栅栏、散养羊（动态环境生物）

**天气池**：`[Sunny, Rain]`（明亮温和，教学关）

**数值签名**：
| 字段 | 档位 |
|------|------|
| 平均敌人 HP | 极低 |
| 平均敌人速度 | 中（含 1 只"快"狼骑） |
| 战场密度 | 1.0 敌/秒 |
| 类型熵 | 三类混合 |
| 抗性分布 | 100% 物理 / 0% 飞行 / 0% 隐形 |
| 节奏档 | Pulse（波间 25s） |
| 波数 | 8 |
| 关卡 Twist | 能量回收 +20%（教学加成） |
| Boss 强度 | 无（关末强化波代替） |

**敌人阵容**：
| ID | 名称 | 角色定位 | 核心机制 |
|----|------|---------|---------|
| `e_goblin_scout` | 哥布林斥候 | 基础杂兵（A 主导） | 标准 grunt，无特殊机制 |
| `e_wolf_rider` | 狼骑兵 | 加速冲刺单元 | 速度档"快" |
| `e_orc_brute` | 兽人战士 | 中血量近战 | 护甲档"低" |
| `e_grave_crawler` 🆕 | 墓爬者 | 战斗中破甲 | `onHit`：首次受击永久移除自身护甲（引导玩家主动开打） |
| `e_goblin_chief` 🆕 | 哥布林酋长 | 关末告别精英 | 强化兽人，无多形态 |

**波次大纲（8 波）**：
教学心智地图：A(单)→A(单)→A(混)→A(快)→A(混)→A(密)→A(新机制)→A+告别

**场景**：赭黄沙地 + 暗紫红虫巢肉膜 + 半埋方尖碑、骆驼骨架、风蚀岩柱

**天气池**：`[Sunny, Sandstorm, Fog]`

**数值签名**：
| 字段 | 档位 |
|------|------|
| 平均敌人 HP | 低 |
| 平均敌人速度 | 快（多高速兵种 + 一个慢坦克反差） |
| 战场密度 | 2.0 敌/秒（本 Run 最高） |
| 类型熵 | 四类混合（地面+飞行+钻地+远程） |
| 抗性分布 | 60% 物理 / 30% 飞行 / 20% 钻地段无敌 |
| 节奏档 | Sustained（波间 15s，持续压力） |
| 波数 | 10 |
| 关卡 Twist | AoE 地面塔对钻地段无效，强制混搭探测塔 |
| Boss 强度 | 关末强化精英波（母虫单形态，3 次召唤后死亡） |

**敌人阵容**：
| ID | 名称 | 核心机制 |
|----|------|---------|
| `e_burrow_worm` 🆕 | 钻地虫 | 每 5s 进入地下 3s（`isTargetable=false`，仅探测塔/AoE 可命中） |

### 关 4 — 🏛️ 失落丛林神庙（Sunken Glyph）

**一句话定调**：被翠绿丛林吞噬的玛雅风神庙，藤蔓缠绕石像，危险祭坛机关遍布。

| 字段 | 档位 |

### 关 8 — 👁️ 异界裂隙（The Veil's Edge）

**一句话定调**：克苏鲁风格的虚空裂隙——紫黑虚空、几何扭曲、传送门随机改向路径。

**地图特殊**：25×11 格，路径更长

**新引入机制**：
- `e_void_blinker`：虚空闪烁者——瞬移 + 死亡传送门（死后在前方 N 格重生一次）
- `e_tower_crusher`：摧塔之眼——专门摧毁玩家防御塔

**终战 Boss — 旧日支配者（三形态）**：

**数值签名**：
| 字段 | 档位 |
|------|------|

## 5. 关卡地图蓝图规范

### 5.1 全局规范

- **坐标系**：21 列（X: 0–20）× 9 行（Y: 0–8），原点左上 `(0,0)`，X 向右增、Y 向下增
- **像素映射**：64 px/格 → 1344×576 战场
- **例外尺寸**：关 6（21×11）/ 关 8（25×11）
- **ASCII 图例**：
  - `.` 空地（不可建不可走）
  - `#` 障碍
  - `P` 路径（敌人与士兵的可行进格）
  - `S` 出生点
  - `C` 水晶锚点
  - `T` 可建塔区（语义上等价于非路径可建格）

### 5.2 YAML 与战场表现的一致性约束（v3.7 新增）

关卡 YAML 中以下字段必须被战斗渲染/交互层消费，而不是仅作静态描述：

| YAML 字段 | 作用 | 强制要求 |
|----------|------|---------|
| `sceneDescription` | 场景设定文本 | 用于约束背景与环境表现方向 |
| `map.tiles` | 棋盘底图与语义格 | 决定路径格/空地/出生点/基地/特殊地块 |
| `map.tileColors` | 默认地块色 | 至少作为渲染基础色来源 |
| `map.pathGraph` | 敌人主路径 | 决定敌人合法行进路线 |
| `weather.pool` / `weather.initial` | 天气池 | 决定天气表现与初始天气 |

### 5.3 战斗规则与 tile 语义绑定

- `path` / `spawn` / `base` 语义格属于**路径体系**。
- 士兵与陷阱的落点判定基于路径体系。
- 塔与生产建筑的落点判定基于非路径可建格。
- 敌人的移动判定基于 `pathGraph` 与路径体系，不允许自由穿越非路径格。
- 若关卡存在 `ice_tile`、`water`、`portal` 等特殊地块，必须在表现层给出独立颜色或纹理区分。
`~` 可建格（玩家放塔位置）
  - `=` 主路径地块
  - `+` 路径分叉/交汇节点
  - `*` 隧道/传送门入口
  - `@` 水域
  - `B` 水晶坐标（终点）
  - `S0..S3` spawn 点（最多 4 个）
  - `W` 空投点（关 5 专用）

### 5.2 波次时间线 Schema

每关的波次时间线必须按以下格式定义（`src/config/levels/level-0{1…8}.yaml`）：

| 列 | 含义 |
|----|------|
| 波 | 波次编号（1 起） |
| t | 该波相对开战 0:00 的开始时间（秒） |
| spawn 流 | `Δt: enemy_id × count @S{n} fmt={编队} [interval={s}]` |
| 下一波 trigger | `auto_after_clear` / `auto_at(t)` / `victory` / `manual` |
| flavor | 5 大类之一 |

**速清奖励**：清场后 10s 内点击「提前开始下一波」→ +5G，10s 后自动触发（无奖励）。

**默认波间冷场**：10s（关 6 GAUNTLET 强制 5s；关 8 单超长波三阶段无波间冷场）。

### 5.3 编队枚举

| 名称 | 含义 |
|------|------|
| `single` | 单只孤兵 |
| `column` | 纵队（沿路径方向单列，spawnInterval ≥ 0.5s） |
| `wedge` | 楔形（1+2+3+...） |
| `phalanx` | 方阵（横向 3 列纵向 N 行，多 spawn 点同步） |
| `swarm` | 蜂群（spawnInterval ≤ 0.3s，飞行层乱序） |
| `convoy` | 护送队（1 高价值目标 + N 护卫） |
| `pincer` | 钳形（≥2 个 spawn 点同步开火） |
| `boss_solo` | Boss 单刷（前后 ≥10s 空窗） |
| `boss_escort` | Boss + 护卫 |
