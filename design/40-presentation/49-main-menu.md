# 游戏主界面（主菜单）设计文档

> 主菜单是游戏的第一个界面，也是每次 Run 开始/结束后的回归点。设计原则：**极简聚焦**——玩家只需做一个决定："重新开始还是继续？"

## 2. 按钮行为定义

### 2.1 新的征程

| 维度 | 规则 |
|---|---|
| 触发条件 | 任何时候均可点击 |
| 有进行中 Run 存档时 | 弹出确认弹窗（§2.4），告知"将丢弃当前 Run 进度，是否确认？" |
| 无进行中 Run 存档时 | 直接初始化新 Run |
| 初始化流程 | `RunManager.initRun()` → 清除 `ongoingRun` 存档 → 显示关卡路线图（关 1 高亮）|
| 界面跳转 | 主菜单 → 关卡路线图 |

### 2.2 继续游戏

| 维度 | 规则 |
|---|---|
| 触发条件 | 存在 `ongoingRun` 存档（`SaveSystem.hasOngoingRun() === true`）|
| 无存档时 | 按钮灰显，点击无响应 |
| 恢复流程 | `RunManager.loadRun(ongoingRun)` → 恢复 `currentLevelIdx` / `gold` / `sp` / `crystalHp` / 卡组 → 显示关卡路线图（按存档进度高亮）|
| 界面跳转 | 主菜单 → 关卡路线图（定位到存档中断节点）|

### 2.3 离开游戏

| 维度 | 规则 |
|---|---|
| 触发条件 | 任何时候均可点击 |
| 行为 | Web 环境：关闭当前 Tab / `window.close()` + 停止 Pixi ticker；桌面端：调用 OS 退出 API |
| 存档处理 | 退出前自动保存当前设置（若有）；进行中 Run 的 `ongoingRun` 已在关间自动存档，无需额外处理 |
| 确认弹窗 | **不弹确认**，直接退出（离开游戏是用户明确意图，无需二次确认）|

### 2.4 丢弃存档确认弹窗（「新的征程」触发）

```
┌──────────────────────────────────────┐
│         ⚠ 开始新的征程                │
├──────────────────────────────────────┤
│  当前已有进行中的 Run 记录：           │
│  · 已通过关卡 3                       │
│  · 金币 240    技能点 6               │
│                                      │
│  开始新征程将永久丢弃以上进度。        │
│                                      │
│      [取消]        [确认放弃]         │
└──────────────────────────────────────┘
```

| 元素 | 锚点 | 说明 |
|---|---|---|
| 弹窗背景 | `anchor: middle-center, offset(0, 0)`, `size: 480 × 260` | 居中 |
| 标题 | 弹窗内 `offset(0, 30)` | "⚠ 开始新的征程"，金色 |
| 存档摘要 | 弹窗内 `offset(0, -10)` | 显示已通关数 / 金币 / 技能点 |
| 说明文字 | 弹窗内 `offset(0, 50)` | 灰色，12px |
| 取消按钮 | 弹窗内 `bottom-left, offset(60, -30)` | 灰色，关闭弹窗 |
| 确认放弃按钮 | 弹窗内 `bottom-right, offset(-60, -30)` | 红色，执行新 Run 初始化 |

## 3. 进入主菜单的来源场景

| 来源 | 进入时机 | 「继续游戏」状态 |
|---|---|---|
| 游戏首次启动 | `bootstrap()` 完成后 | 取决于本地存档是否有 `ongoingRun` |
| Run 结算结束（胜利/失败）后点「返回主菜单」 | `RunResultPanel` 关闭 | 胜利 Run 无存档（Run 已完结）；失败 Run 无存档（数据清零）|
| Run 结算后点「立即开始新 Run」 | 直接跳路线图，**不经过主菜单** | — |
| 路线图 ESC 退出 Run 确认后 | `RunManager.abandonRun()` → 清除存档 → 主菜单 | 灰显（存档已清除）|

## 4. 存档与按钮同步规则

主菜单显示时，UI 层必须**重新读取存档状态**以确定「继续游戏」按钮是否高亮：

```typescript
// 伪代码：主菜单进入时同步存档状态
function onMainMenuEnter(): void {
  const hasSavedRun = SaveSystem.hasOngoingRun();
  mainMenu.refresh({ hasSavedRun });
}
```

- **不缓存**按钮状态，每次显示主菜单都重新查存档
- 存档读取失败（损坏/版本不匹配）→ 视为无存档，「继续游戏」灰显 + 控制台 warn

## 5. 背景画面规范

| 元素 | 规范 |
|---|---|
| 主色调 | 深蓝→深紫渐变（顶部 `#0d1b2a` → 底部 `#1a0a2e`）|
| 粒子效果 | 轻微漂浮白点（30-50 个），alpha 0.3-0.6，缓慢随机漂移，不干扰主按钮视觉 |
| 远景装饰 | 可选：极淡的山脉轮廓（alpha 0.1），纯装饰 |
| 音频 | 循环播放主题背景音乐（音量 0.3），参见 [46-audio](./46-audio.md) |

## 6. 验收清单

- [ ] 主菜单全屏铺满，标题居中显示
- [ ] 「新的征程」按钮常亮可点，点击后检测存档
- [ ] 有进行中 Run 存档时，点「新的征程」弹出确认弹窗（§2.4 结构）
- [ ] 无进行中 Run 存档时，点「新的征程」直接初始化新 Run 并跳路线图
- [ ] 「继续游戏」按钮：有存档时绿色边框可点；无存档时灰显不可点
- [ ] 点「继续游戏」恢复路线图，节点状态与存档一致（已通关/当前/未解锁）
- [ ] 「离开游戏」直接退出，无确认弹窗
- [ ] 主菜单每次显示时重新读取存档状态（§4 同步规则）
- [ ] 弹窗中显示当前 Run 存档摘要（已通关关卡数 / 金币 / 技能点）
