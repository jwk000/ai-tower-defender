# 士兵 AI 行为设计

> 我方可移动单位（统称"士兵"）的 AI 行为规格：警戒 / 追击 / 战斗 / 游荡 四状态机。
>
> **实现路径（v3.4）**：四状态机由规则引擎在 AttackSystem / MovementSystem 中实现，状态字段挂在士兵 ECS 组件上（`SoldierAI.state: enum`），状态转换由帧内条件检测触发（圈范围 query + cooldown 计时）。

---

## 1. 三圈模型

```
           ┌──────────────────────────────┐
           │         移动范围 (moveRange)   │
           │    ┌──────────────────┐       │
           │    │  警戒范围 (alert)  │       │
           │    │  ┌─────────┐     │       │
           │    │  │攻击范围  │     │       │
           │    │  │(attack) │     │       │
           │    │  │  🗡️    │     │       │
           │    │  └─────────┘     │       │
           │    │                  │       │
           │    └──────────────────┘       │
           │                               │
           │            🏠 放置点           │
           └──────────────────────────────┘
```

| 范围 | 定义 | ECS 字段 | 行为 |
|------|------|----------|------|
| **移动范围** (`moveRange`) | 士兵离开放置点的最大距离 | `Movement.moveRange` + `Movement.homeX/Y` | 任何移动都不超出此范围（绳索约束） |
| **警戒范围** (`alertRange`) | 能发现敌人的最远距离 | `Attack.alertRange` | 范围内出现敌人 → 进入 ALERT |
| **攻击范围** (`attackRange`) | 能攻击到敌人的距离 | `Attack.range` | 范围内有敌人 → 进入 COMBAT |

**范围大小关系**：`moveRange > alertRange > attackRange`

**远程兵抖动防护（硬约束）**：`alertRange ≥ attackRange × 1.1 + 20px`

| 士兵类型 | alertRange 建议 |
|----------|----------------|
| 近战（盾卫/剑士） | `attackRange × 4~5` |
| 远程（弓手） | `attackRange × 1.1 + 20px` |
| 治疗（祭司） | `attackRange × 1.5~2` |
| 爆发（刺客） | `attackRange × 5~8` |

## 2. 状态机

### 2.1 状态优先级

每帧强制按以下优先级重新评估，不允许"状态保留惯性"：

```
COMBAT > ALERT > RETURN > IDLE
```

| 状态 | 触发条件 |
|------|---------|
| **COMBAT** | 当前目标存活 + 在 `attackRange` 内 |
| **ALERT** | `alertRange` 内有合法敌人（且不满足 COMBAT） |
| **RETURN** | 与 home 距离 > 10px 且无敌人在 `alertRange` 内 |
| **IDLE** | 以上均不满足 |

### 2.2 状态转换图

```
                          ┌──────────────────┐
        无敌人             │                  │
    ┌─────────────────────►       IDLE        │
    │                     │    (游荡+等待)     │
    │                     └────────┬─────────┘
    │                              │
    │                    警戒范围发现敌人
    │                              │
    │                     ┌────────▼─────────┐
    │                     │                  │
    │  敌人全灭/全离开     │      ALERT       │
    │  警戒范围            │   (追击+红!)      │
    │                     └────────┬─────────┘
    │                              │
    │                    进入攻击范围
    │                              │
    │                     ┌────────▼─────────┐
    │   敌人死亡/          │                  │
    │   离开攻击范围        │     COMBAT       │
    └─────────────────────│    (攻击+追击)     │
       回到IDLE            └────────┬─────────┘
                                   │
                         目标移动到移动范围外
                         或超出警戒范围
                                   │
                          ┌────────▼─────────┐
                          │                  │
                          │     RETURN       │
                          │   (返回home)      │
                          └────────┬─────────┘
                                   │
                            回到home位置
                                   ▼
                                 IDLE
```

### 2.3 各状态详解

#### IDLE — 空闲

- 在 `moveRange` 半径内随机游荡
- 每 2-4 秒随机选移动范围内一个点，50% 速度移动，抵达后停顿 1-3 秒
- 视觉：正常色，无感叹号

```yaml
wander:
  speed_ratio: 0.5
  pick_interval: [2, 4]      # 秒
  pause_interval: [1, 3]     # 秒
  jitter_radius: 40          # px
```

#### ALERT — 警戒

- 锁定 `alertRange` 内最近的敌人（首次锁定，后续目标稳定）
- 以 100% 速度追击，不超出 `moveRange`（超界返回 RETURN）
- 视觉：头顶**红色感叹号闪烁**（亮 0.3s → 灭 0.3s 循环）
- 音效：`SFX_SOLDIER_ALERT`（每士兵最多 1 次/3秒）

#### COMBAT — 战斗

- 攻击当前锁定目标；目标死亡后自动在 `alertRange` 内重选
- 近战：`HoldPosition`；远程：`ChaseTarget` 微调
- 视觉：红叹号常亮，攻击动画

#### RETURN — 返回

- 以 80% 速度向 `homeX/homeY` 移动
- 中途在 `alertRange` 内发现新敌人 → 立即切换到 ALERT
- 视觉：红叹号 0.3s 渐变消失

### 2.4 玩家操控优先级

| 优先级 | 来源 | 行为 |
|--------|------|------|
| 1（最高） | 玩家拖拽 | `PlayerControllable.targetX/Y` 驱动，覆盖 AI 状态 |
| 2 | 战斗 | COMBAT 中 AI 自动攻击 |
| 3 | 警戒/返回 | ALERT/RETURN 由 AI 自动处理 |
| 4（最低） | 空闲 | IDLE 自动游荡 |

> 玩家拖拽松手后，AI 根据当前位置重新判断状态。

## 3. 多士兵协同

### 3.1 目标分配（防扎堆）

| 规则 | 说明 |
|------|------|
| **目标负载追踪** | 每个敌人维护 `attackerCount`（当前正在攻击该敌的己方士兵数） |
| **目标选择加权** | `score = distance + attackerCount × 80` |
| **解除超饱和** | `attackerCount ≥ 3` 且其他敌人在范围内 → 该敌排除候选 |
| **高威胁例外** | 法师/萨满/虫母等目标阈值放宽到 5 |
| **目标重选触发** | ① 当前目标死亡；② 当前目标超出 max_range；③ 战斗中每 2 秒检查一次更高优先级目标 |

### 3.2 碰撞与抖动防护

| 规则 | 说明 |
|------|------|
| **软碰撞半径** | 同阵营士兵间保持 `personalSpace = unitRadius × 1.5` |
| **推挤力** | 两兵距离 < `personalSpace` 时互相施加垂直偏移力（每帧 ≤ 4px） |
| **到位门槛** | 距离 < 5px 即视为到位，停止位移 |
| **攻击格位锁** | 进入 COMBAT 时占据一个攻击锚点，其他士兵选不同锚点 |
| **死锁解除** | 2 秒内位置变化 < 2px 且非 COMBAT → 强制随机偏移 8-16px |

### 3.3 士兵死亡协同

| 事件 | 处理 |
|------|------|
| 士兵 A 死亡 | 目标 `attackerCount` 自减；解除攻击格位锁 |
| 士兵全灭 | 敌人 `attackerCount` 归零，恢复路径移动 |
| 复活/重新部署 | 回到 IDLE 状态，从 home 点开始游荡 |

## 4. 敌方反击规则

### 4.1 触发条件

```
敌人受到士兵伤害 → 将伤害来源设为当前攻击目标 → 停止路径移动 → 开始攻击
```

### 4.2 反击规则

| 规则 | 说明 |
|------|------|
| **反击门槛** | 仅携带 `Attack` 组件的敌人才有反击能力 |
| **仇恨优先级** | 受到伤害 > 距离最近 |
| **仇恨持续** | 3 秒；3 秒内未再次受到同一士兵伤害 → 重置为距离驱动 |
| **仇恨切换** | 另一士兵造成更多伤害 → 切换目标 |
| **停止移动** | 反击时 `moveMode = HoldPosition` |
| **恢复移动** | 目标死亡或仇恨超时后，恢复 `FollowPath` |
| **Boss 特殊** | Boss 反击但不中断路径移动 |

### 4.3 各敌人反击能力

| 敌人 | 携带 Attack？ | 会反击？ | 备注 |
|------|-------------|----------|------|
| 小兵 | ❌ | ❌ | 纯路径移动 |
| 快兵 | ❌ | ❌ | 速度型，挨打不还手 |
| 重装兵 | ✅ | ✅ | 被攻击后转向近战 |
| 法师 | ✅ | ✅ | 被攻击后远程还击 |
| 自爆虫 | ❌ | ❌ | 死亡爆炸，不反击 |
| 指挥官(Boss) | ✅ | ✅ | 反击但不终止路径 |
| 攻城兽(Boss) | ✅ | ✅ | 反击但不终止路径 |
| 热气球 | ✅ | ⚠️ | 不因士兵攻击改变目标（优先轰炸建筑） |
| 萨满 | ✅ | ✅ | 被攻击后还击（低伤害） |
| 幽鬼 | ❌ | ❌ | 渗透型，挨打不还手 |
| 虫母 | ❌ | ❌ | 靠召唤，本体不反击 |
| 擂鼓手 | ✅ | ✅ | 被攻击后还击 |

## 6. 嘲讽机制

### 6.1 核心规则

| 字段 | 定义 |
|------|------|
| `tauntCapacity` | 嘲讽源同时能吸引的敌人数量（`Attack.tauntCapacity`） |
| `attackerCount` | 当前以该单位为目标的敌人计数（引用计数维护） |
| 嘲讽源 | `tauntCapacity > 0` 的单位 |
| 饱和嘲讽源 | `attackerCount >= tauntCapacity` |

### 6.2 敌方目标选择优先级

按 `(tauntGroup, distSq)` 排序，组内距离最近优先：

| tauntGroup | 含义 | 优先级 |
|-----------|------|--------|
| 0 | 可用嘲讽源（未饱和） | 最高 |
| 1 | 非嘲讽源（`tauntCapacity === 0`） | 中 |
| 2 | 饱和嘲讽源 | 最低 |

**防抖**：若敌人当前目标已是该嘲讽源，视为"可用"（防止饱和瞬间脱锁频繁切换）。

### 6.3 引用计数维护

所有 `Attack.targetId` 写入必须经过统一接口：

- **`setEnemyTarget(eid, newTargetId)`**：旧目标 `releaseTaunt`（自减）+ 新目标 `acquireTaunt`（自增）；仅对嘲讽源操作
- **`LifecycleSystem.onDeath`**：单位死亡时遍历所有敌人，把指向死者的 `targetId` 清零，同步释放嘲讽引用

### 6.4 数值

| 单位 | `tauntCapacity` | 升级变化 |
|------|----------------|---------|
| 盾卫 | Lv1=2 / Lv2=3 / Lv3=4 | `upgradeTauntCapacityBonus: [1, 1]` |
| 剑士 | 固定 1 | 升级不变 |

## 7. AOE 溅射（剑士）

### 7.1 规则

| 字段 | 值 |
|------|-----|
| `splashRadius` | 96px（覆盖 3×3 = 9格，含对角） |
| 溅射中心 | 攻击者自身坐标 |
| 溅射系数 | 0.6（周围目标承受 60% 主目标伤害） |
| 主目标 | 不重复计算溅射伤害 |

### 7.2 覆盖证明

格子大小 = 64px，对角距离 = √2 × 64 ≈ 90.51px，取 `splashRadius = 96px`，余量 5.49px，确保对角格不被裁掉。

### 7.3 单位配置

| 单位 | `splashRadius` | AOE |
|------|---------------|-----|
| 剑士 | 96 | ✅ 含对角 |
| 盾卫 | 0 | 无 |

## 9. 验收标准

### 9.1 我方士兵 AI

- [ ] IDLE：在移动范围内随机游荡，无敌人时不攻击
- [ ] ALERT：警戒范围内出现敌人，头顶显示红色闪烁感叹号
- [ ] ALERT → 追击：向最近敌人移动，不超出 `moveRange`
- [ ] COMBAT：敌人进入攻击范围，感叹号常亮，开始攻击
- [ ] COMBAT → ALERT：敌人离开攻击范围但仍在警戒范围，恢复追击
- [ ] RETURN：无敌人时感叹号消失，返回 home
- [ ] RETURN → IDLE：回到 home 后恢复游荡
- [ ] 玩家拖拽优先级最高
- [ ] 目标切换：当前目标死亡后自动重选警戒范围内最近敌人
- [ ] 远程兵抖动防护：`alertRange ≥ attackRange × 1.1 + 20`
- [ ] 超界保护：追击到 `moveRange` 边界立即转 RETURN，不卡死边缘

### 9.2 敌方反击

- [ ] 无 Attack 组件的敌人被攻击后不反击
- [ ] 有 Attack 组件的敌人被士兵攻击后，伤害来源成为当前目标
- [ ] 反击时停止路径移动；目标死亡后恢复
- [ ] Boss 反击但不终止路径移动
- [ ] 热气球被士兵攻击后不改变目标
- [ ] 3 秒未受同一士兵伤害 → 清除仇恨

### 9.3 视觉反馈

- [ ] 红叹号 ALERT 时闪烁，COMBAT 时常亮，IDLE/RETURN 时不可见
- [ ] 返回时感叹号 0.3s 渐变消失

### 9.4 回归验证

- [ ] 现有塔攻击逻辑不受影响
- [ ] 现有敌人路径移动逻辑不受影响
- [ ] 现有士兵部署/拖拽逻辑不受影响
